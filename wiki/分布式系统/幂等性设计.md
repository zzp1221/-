## 核心定义

幂等性（Idempotency）是指一个操作执行一次和执行多次的效果完全相同。在分布式系统中，由于**网络超时重试**、**消息重复投递**、**用户重复点击**等原因，同一操作可能被执行多次。幂等性设计确保这些重复执行不会导致数据不一致。

**为什么需要幂等性**：
- **At-Least-Once语义**：消息队列保证消息至少投递一次，但可能重复
- **网络超时重试**：客户端发送请求后超时，不知道服务端是否已处理，只能重试
- **用户重复提交**：用户点击"提交"按钮多次

**实现方式**：

**唯一请求ID（幂等键）**：每个请求携带唯一的请求ID，服务端检查该ID是否已处理过。
- 请求ID通常由客户端生成（如UUID）
- 服务端维护已处理请求ID的记录
- 相同ID的请求直接返回之前的结果

**数据库唯一约束**：利用数据库的唯一约束防止重复插入。
- 如订单号设为唯一索引，重复插入会违反约束
- 简单但只适用于插入操作

**乐观锁（版本号）**：每次更新时检查版本号，版本号不匹配则拒绝更新。
- `UPDATE table SET value = ?, version = version + 1 WHERE id = ? AND version = ?`
- 适用于更新操作

**状态机**：业务状态只能单向流转，重复操作会被状态机拒绝。
- 如订单状态：待支付 → 已支付 → 已发货
- 重复支付请求会因状态不匹配被拒绝

## 关键结论

- 幂等性是分布式系统的**基本要求**——在At-Least-Once语义下，不保证幂等会导致数据不一致
- **幂等键**是最通用的实现方式，适用于所有操作类型
- 幂等键的存储需要**持久化**和**过期清理**——否则重启后丢失记录，或存储无限增长
- 幂等性可以分层实现：**网关层**（请求去重）、**服务层**（业务幂等）、**数据层**（唯一约束）
- **CAS（Compare-And-Swap）**操作天然幂等——相同条件相同时多次执行效果一致

## 易错点

1. **只在客户端做幂等**：客户端生成幂等键后，如果服务端没有检查，重复请求仍会被处理。必须在服务端检查
2. **幂等键存储未持久化**：如果幂等键存储在内存中，服务重启后记录丢失，重复请求会被处理
3. **忽视幂等键的过期问题**：长期存储幂等键会导致存储无限增长。需要设置合理的过期时间

## 例题

**题目**：一个支付系统需要保证扣款操作的幂等性。用户通过API网关发起支付请求，请求经过网关到达支付服务。由于网络超时，用户重复点击了3次。请设计幂等方案。

**解答**：

**多层幂等方案**：

**第一层：API网关层**
- 网关为每个请求生成唯一的`request_id`（如果客户端未提供）
- 网关维护一个短时间窗口（如5秒）的请求去重缓存
- 相同`request_id`的重复请求直接返回缓存结果
- **过滤大部分重复请求**

**第二层：支付服务层**
- 检查`request_id`是否已处理（查询幂等表）
- 如果已处理，直接返回之前的结果
- 如果未处理，执行支付逻辑

**第三层：数据库层**
- 支付流水号设为唯一索引
- 即使前两层失败，数据库唯一约束也能防止重复扣款

**流程**：
```
用户点击 → API网关（去重） → 支付服务（幂等检查） → 数据库（唯一约束）
   ↓              ↓                    ↓
 重复请求      缓存命中             幂等表命中
   ↓              ↓                    ↓
 返回结果      返回缓存结果         返回之前结果
```

**幂等表设计**：
```sql
CREATE TABLE idempotency_key (
    request_id VARCHAR(64) PRIMARY KEY,
    response TEXT,
    created_at TIMESTAMP,
    expires_at TIMESTAMP  -- 过期时间，如24小时
);
```

**幂等检查代码**：
```python
def process_payment(request_id, amount):
    # 1. 检查幂等表
    existing = db.query("SELECT response FROM idempotency_key WHERE request_id = ?", request_id)
    if existing:
        return existing.response  # 直接返回之前的结果
    
    # 2. 执行支付逻辑
    try:
        result = do_payment(amount)
        # 3. 记录幂等键
        db.insert("INSERT INTO idempotency_key (request_id, response) VALUES (?, ?)", 
                  request_id, result)
        return result
    except Exception:
        # 记录失败状态，防止重试时重复执行
        db.insert("INSERT INTO idempotency_key (request_id, response) VALUES (?, ?)", 
                  request_id, "FAILED")
        raise
```

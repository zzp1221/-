## 核心定义

片元着色器（Fragment/Pixel Shader）是渲染管线中对每个片元执行的可编程阶段，负责计算该片元的最终颜色。片元着色器接收光栅化阶段插值后的顶点属性（颜色、法线、纹理坐标等），执行纹理采样、光照计算、阴影查询等操作，输出一个或多个颜色值（MRT，Multiple Render Targets）。片元着色器可以执行 discard 操作丢弃片元（如用于树叶的 alpha test）。在片元着色器执行之前，GPU 可能会执行 Early-Z 深度测试来跳过被遮挡的片元，但当片元着色器修改了深度值或使用了 discard 时，Early-Z 会被禁用。

深度测试（Depth Test）是逐片元操作阶段的核心环节。深度缓冲（Z-Buffer/Depth Buffer）为每个像素存储当前最近的深度值。新片元的深度值与深度缓冲中的值比较（默认使用 `GL_LESS`），只有更近的片元才会通过测试并更新深度缓冲。深度值在光栅化阶段通过重心坐标线性插值得到，范围通常为 $[0, 1]$（OpenGL）或 $[0, 1]$（Vulkan/DirectX）。深度测试之后还有模板测试（Stencil Test）、混合（Blending）等操作。混合用于实现透明效果：$C_{result} = C_{src} \cdot F_{src} + C_{dst} \cdot F_{dst}$，其中 $F_{src}$ 和 $F_{dst}$ 是混合因子。透明物体通常需要关闭深度写入并按从后到前的顺序渲染，或者使用 OIT（Order-Independent Transparency）技术。

## 关键结论

- 深度缓冲的精度是非线性的：透视投影后 $z$ 值在近平面处精度最高，远处精度急剧下降。这是 z-fighting 的根本原因
- Early-Z 优化需要片元着色器不修改深度值且不使用 discard，否则硬件无法提前执行深度测试
- 透明物体渲染必须关闭深度写入（`glDepthMask(GL_FALSE)`），并按从后到前排序。不排序会导致错误的混合结果
- 多重渲染目标（MRT）允许片元着色器同时输出多个颜色到不同的纹理，是延迟渲染的基础
- 深度预通道（Depth Pre-pass）先用简化着色器写入深度，然后正常渲染时启用 Early-Z，避免复杂着色器对被遮挡片元的无效计算

## 易错点

1. **透明物体深度写入问题**：渲染透明物体时如果不关闭深度写入，透明物体的深度值会遮挡其后面的物体，导致后面的物体"消失"
2. **深度比较函数理解错误**：OpenGL 默认 `GL_LESS`（更小的深度值通过测试），但在某些投影设置下深度方向可能反向，需要调整为 `GL_GREATER`
3. **混合顺序依赖**：alpha 混合的结果依赖于渲染顺序（$A \oplus B \neq B \oplus A$），混合的物体必须按深度排序。半透明物体与不透明物体的深度测试交互需要特别注意

## 例题

**题目**：屏幕上一个像素已经渲染了一个不透明物体，深度缓冲中存储的深度值为 0.5。现在要渲染一个透明物体，其在该像素处的深度值为 0.3，颜色为 $(1, 0, 0, 0.6)$（RGBA），已有颜色为 $(0, 0, 1, 1)$。使用标准 alpha 混合，求最终颜色。

**解答**：

第一步，深度测试：透明物体深度 0.3 < 不透明物体深度 0.5，深度测试通过。

第二步，alpha 混合（使用标准混合模式 $F_{src} = \alpha_{src}$, $F_{dst} = 1 - \alpha_{src}$）：
$$C_{result} = C_{src} \cdot \alpha_{src} + C_{dst} \cdot (1 - \alpha_{src})$$
$$= (1, 0, 0) \cdot 0.6 + (0, 0, 1) \cdot 0.4$$
$$= (0.6, 0, 0) + (0, 0, 0.4) = (0.6, 0, 0.4)$$

最终颜色为 $(0.6, 0, 0.4)$，呈现紫色调。

注意：如果先渲染透明物体再渲染不透明物体，由于透明物体关闭了深度写入，不透明物体的深度测试会通过，最终颜色为 $(0, 0, 1)$（蓝色），透明物体被完全覆盖。

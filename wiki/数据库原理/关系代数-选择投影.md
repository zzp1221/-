## 核心定义

关系代数（Relational Algebra）是一组以关系为运算对象的代数系统，是关系数据库查询语言（SQL）的理论基础。关系代数运算分为两大类：传统的集合运算（并、差、交、笛卡尔积）和专门的关系运算（选择、投影、连接、除）。选择（Selection）运算记为σ_F(R)，其中σ是选择运算符，R是关系名，F是选择条件（逻辑表达式，由比较运算符和逻辑运算符组成）。选择运算是从关系R中选出满足条件F的元组，属于横向的行过滤操作——结果仍然是一个关系，模式（属性数及名称）与R相同。投影（Projection）运算记为π_A(R)，其中π是投影运算符，A是属性列表（R的若干属性组成的列集合）。投影运算是从关系R中选出指定的若干属性列，并去除结果中可能产生的重复元组（因为关系定义要求元组不重复），属于纵向的列过滤操作。选择和投影是SQL查询语句中SELECT-WHERE子句的直接对应——σ对应WHERE条件，π对应SELECT的属性列表（去重集）。选择和投影是所有复杂查询的基础运算。

## 关键结论

- 选择σ_F(R)的形式化定义：σ_F(R) = { t | t∈R ∧ F(t)=True }，其中F是合法的逻辑表达式（由属性名、常量、比较运算符{>, ≥, <, ≤, =, ≠}和逻辑运算符{∧, ∨, ¬}组成）。F不能包含聚合函数、不能包含子查询结果（那是SQL的扩展能力超出基础的关系代数）
- 投影π_{A1,A2,...,Ak}(R)的形式化定义：对于R中的每个元组t，取出t的k个分量组成新的元组，然后消去重复元组（去重是因为在关系中元组是集合不能有重复）。投影不仅减少了列数，还减少了行数（因为投影后多个原有不同的元组可能变成相同而被合并）
- 选择和投影的执行顺序可以互换吗？通常情况下：π(σ(R)) = σ(π(R))成立的前提是σ条件中涉及的属性全都包含在π的属性列表中。如果条件F涉及未在投影列表中的属性，则必须先做选择再投影——在逻辑查询优化中这叫"谓词下推"，尽量在投影前行完成过滤
- 选择条件和投影列表中的属性使用关系模式中的属性名，不能使用"=常量+列别名"的表达式（SQL支持的关系代数并不完全对应选代数——SQL还额外支持如+ - * / 等算术表达式）
- 选择和投影的复合操作π_A(σ_F(R)) ≈ SQL语句SELECT A FROM R WHERE F --即SQL最基本的查询结构

## 易错点

1. **投影去重是关系代数的要求而非SQL的要求**：关系代数中的投影π自动去重（因为结果必须是关系即元组集合不包含重复元组），但SQL中的SELECT默认不去重——必须使用SELECT DISTINCT显式要求去重。这是关系代数和SQL的一个重要区别。

2. **选择条件的逻辑定律**：σ_{F1∧F2}(R) = σ_{F1}(σ_{F2}(R)) = σ_{F2}(σ_{F1}(R)) ——串联式的条件结合等价于分别应用选择（处理谓词交换律）。而σ_{F1∨F2}(R) = σ_{F1}(R) ∪ σ_{F2}(R) 如果F1和F2相容（同属性的互斥条件）则是将结果的并集——即选择中OR条件可以拆分为并集操作。

3. **投影操作可能丢失信息的不可逆性**：投影是不可逆的——不能从π_A(R)反推R。因为一些属性被投影丢弃后信息丢失（除非被丢弃的属性函数依赖于投影属性，此时可通过自然连接恢复——这是关系分解理论/无损分解的核心）。

4. **关系代数选择中的比较运算局限**：关系代数只能处理属性与常量或属性与属性之间的简单比较，不支持集合成员判断（IN）、子查询、LIKE模式匹配等SQL丰富的表达式。这些扩展超越了纯关系代数，属于关系完备性之上的增补功能。

## 例题

**例题1**：已知学生关系Student(sno, sname, ssex, sage, sdept)和选课关系SC(sno, cno, grade)。写出以下查询的关系代数表达式：
(1)查询计算机系(CS)全体学生信息。
(2)查询计算机系学生的姓名和学号。
(3)查询选修了C01课程且成绩高于80分的学生学号。

**解答**：(1) σ_{sdept='CS'}(Student)。(2) π_{sname,sno}(σ_{sdept='CS'}(Student))。(3) π_{sno}(σ_{cno='C01' ∧ grade>80}(SC))。通过选择C01且成绩>80的SC行后投影到只有sno列，即得所要求的学号。

**例题2**：设有关系R(A, B, C, D)和数据若干行。说明表达式σ_{A>10}(π_{A,C}(R)) 与 π_{A,C}(σ_{A>10}(R)) 是否等价？分两种情况讨论。

**解答思路**：需要根据选择条件中属性是否都在投影属性集内来判断。情况一：这两个表达式不等价，因为前一个表达式——先投影再选择(π_{A,C}(R)得到仅有A,C列的新关系，再用A>10选择)中，因为A列在投影集合内，所以σ_{A>10}(π_{A,C}(R))合法且可用谓词。后一个表达式——先选择再投影(σ_{A>10}(R)保留所有四列后用π_{A,C}投影到两列)。两者结果相同（π_{A,C}(σ_{A>10}(R)) = σ_{A>10}(π_{A,C}(R))前提是选择条件涉及的所有属性都在投影属性集内。此例中仅条件涉及A而在投影集的{A,C}中，等式成立）。

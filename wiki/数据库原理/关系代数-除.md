## 核心定义

除运算（Division）是关系代数中最复杂但也最强大的运算，用于处理蕴含"所有"或"全部"语义的查询。设关系R有属性集{X, Y}（将属性分为两组X和Y），关系S定义在属性集Y上（S的属性集必须完全被包含在R的属性集中）。R除以S记为R ÷ S，结果是在X上的投影，包含所有满足以下条件的X值：这个X值对应的Y属性上的值集包含S在Y上的所有值。形式化定义：R ÷ S = { t[X] | t∈R ∧ ∀ s∈S ( (t[X], s[Y]) ∈ R ) }，即"对于R中的每个X值，如果它与S中所有Y值的每个组合都在R中出现了，那么该X值属于除的结果"。除运算常用来回答"至少满足所有某条件"或"全部包含"这类查询，相当于关系代数中对全称量词（∀）的处理方法。由于SQL中没有直接的除法操作符，除运算需要通过NOT EXISTS嵌套子查询或GROUP BY - HAVING COUNT技巧来间接实现。

## 关键结论

- 除运算的语义等价："找出在R中与S的所有Y值都关联的X值"。典型应用——查询选修了所有课程的学生学号（R=SC(sno,cno)，S=Course(cno)），查询使用了全部零件的工程号（R=SPJ(Sno,Pno,Jno)，S=Part(Pno)）
- 除运算的推导式解法：R ÷ S = π_X(R) - π_X( (π_X(R) × S) - R ) ——这个恒等式揭示了除法的本质：(1)生成X值所有可能的全组合π_X(R) × S（每个X值×所有Y值=本应具有的全部配对）；(2)找出实际缺失的组合——即(π_X(R) × S) - R ——期望的全组合中实际没有出现过的（缺失）；（3)从所有的X值中去掉那些"有缺失"的X值——剩下就是"一个都不缺（全都有）"的X值→除的结果
- 除运算的应用条件：S的属性集必须是R属性集的子集（通常S的所有属性都在R中），且S的属性与R的属性有相同域语义。被除结果中X既可以是单个属性也可以是属性组合
- 除运算与"全称"和"存在"量词：选择σ表达了存在量词∃（至少选了一门课——存在某条SC记录匹配），而除运算÷表达了全称量词∀（选修了所有课——所有课程在SC中都有对应条目）。多数复杂查询正是在全称量词与存在量词的组合中需要除运算
- 除运算在实际SQL查询优化中的等价转化：由于SQL无直接除法操作符，通常将R ÷ S转化为 NOT EXISTS ( SELECT * FROM S WHERE NOT EXISTS ( SELECT * FROM R ... ) )的双层嵌套查询（全称量词的逻辑等价转换），或GROUP BY + HAVING COUNT(DISTINCT ...) = (SELECT COUNT(*) FROM S)

## 易错点

1. **除运算要求X和Y是R的属性集的划分**：R的属性集被明确地分为{X集}和{Y集}，且S的属性集是Y集（完全落在R的属性Y范围内）。S不能包含X集的属性也不能包含R没有的属性。

2. **除运算结果不包含S中的Y值信息**：除的结果是X集上的投影，结果模式仅包含{X集合}不包含Y（因为除运算隐含了"与S的所有Y值都关联"的信息而不再需要Y列的显式表示）。这与初学者期待"除的结果还带S的所有值"有所差异。

3. **除运算与差运算(÷ 和 - )注意符号不可混淆**：÷除号(关系除法)和减法(差)是完全不同的操作。关系差运算R-S是从R中去掉同时出现在S中的元组——与除法毫无关系。÷是一个较少使用的操作符，需小心辨别笔误和语义。

4. **除运算不等于数字的除法**：关系除法不涉及数值计算，是无算术意义的关系运算。"R ÷ S" = 在R中找出与S的全部Y值都配对的X值，是一个逻辑运算。这个命名只是为了表明它是对笛卡尔积的某种"逆运算"——在处理R = Q × S的特殊情况下，R ÷ S = Q成立（正好是一种逆关系）。

## 例题

**例题1**：已知关系SC(sno, cno, grade)包含学生选课记录，关系Course(cno, cname, credit)包含课程信息。写出"查询选修了所有课程的学生的学号"的关系代数表达式。

**解答**：将SC按属性划分为X={sno}, Y={cno}；Course投影到{cno}（因为只关系课程号而无关课程名）。除运算：π_{sno,cno}(SC) ÷ π_{cno}(Course)。SC中每个学生对应多个cno值的组合。如果某学号的{cno}值集包含了Course中全部cno的每一个值，除法结果中就包含这个学号。这就是"选修了全部课程"的语义——全称量词÷表达。

**例题2**：使用推导式π_X(R) - π_X((π_X(R) × S) - R) 求出除运算R ÷ S的结果，并解释该公式的两个步骤。

**解答思路**：第一步——计算缺的组合：(π_X(R) × S) - R，这是找出"每个X值与所有S值的全组合中，哪些组合不在R中（缺失）"。π_X((π_X(R) × S) - R) 投影出所有"有缺失"的X值。第二步——排除有缺失的X：π_X(R) - π_X(缺失的X) = 所有X值中去掉有缺失的X值即为结果。这就是先找有缺失的X集合，然后从全体X集合中排除它们——剩下的是全都不缺的X（也就是每个X与S全部配对都在R中出现过的X）。这正是两个步骤：找坏（缺）→ 排除坏（好）。公式演示了除法的实践求解方法。

## 代码示例

```sql
-- 查询"选修了所有课程的学生学号" —— 除运算的SQL等价写法
-- 方法一：双重NOT EXISTS（最直接的逻辑等价）
SELECT DISTINCT sno FROM Student S1
WHERE NOT EXISTS (
    SELECT * FROM Course C
    WHERE NOT EXISTS (
        SELECT * FROM SC
        WHERE SC.sno = S1.sno AND SC.cno = C.cno
    )
);

-- 方法二：GROUP BY + HAVING计数法（更常用和更直观）
SELECT sno
FROM SC
GROUP BY sno
HAVING COUNT(DISTINCT cno) = (SELECT COUNT(*) FROM Course);
-- 要求：SC中每学号的不重复课程数 = 课程表总课程数
```

## 核心定义

码（Key）是关系模型中用来唯一标识关系中每一个元组的属性或属性组合。按照标识能力和扮演的角色，码分为以下几种类型：超码（Superkey）——能唯一标识关系中每个元组的属性集合，超码可能包含多余的属性（即去除某些属性后仍然是超码）；候选码（Candidate Key）——最小的超码，即不能再去掉任何一个属性而仍然保持唯一标识能力的超码；主码（Primary Key）——从候选码中选定的一个作为关系的主标识码（一个关系只有一个主码）；外码（Foreign Key）——关系R中的属性（集），其取值要么为空，要么等于另一个关系S中某元组的候选码的值，用于实现关系之间的引用联系；全码（All-Key）——关系的全体属性集合才构成候选码（即任何一个属性或属性子集都不足以唯一标识元组）。包含在任意候选码中的属性称为主属性（Prime Attribute），不包含在任何候选码中的属性称为非主属性（Non-Prime Attribute）。

## 关键结论

- 超码与候选码的关系：候选码是包含最少属性的超码。例如关系Student(sno, sname, id_card, age)中，{sno}和{id_card}是候选码，{sno, sname}是超码但不是候选码（因为去掉sname后{sno}仍是超码）
- 候选码的两个性质：唯一性（所有元组的候选码取值唯一不重复）、最小性（候选码的任何真子集不再是超码——即去掉任意一个属性后无法保证唯一标识）。这是判定候选码的关键标准
- 外码的三种取值情况：（1）外码取值为空（NULL）——表示该元组当前没有与之关联的参照实体；（2）外码取值等于被参照关系中某个元组的候选码值——表示存在关联；（3）外码取值不等于任何被参照的候选码值——违反参照完整性，DBMS应拒绝此操作
- 全码关系的典型场景：关系中的任何属性组合都不是候选码时（即所有属性组合在一起才能唯一区分每个元组）。例如汽车司机-公车-用车纪录关系(Car, Driver, Date)可能三个人共同成为候选码——因为同一辆车在同一日期可被同一司机多次使用的业务规则可能因列数已全覆盖而构成全码

## 易错点

1. **主键(主码)与外键(外码)是两个完全不同的概念**：主键是关系内部唯一标识，外键是跨关系引用。一个属性可以既是一个关系的主键成员又同时作为另一个关系的外键（例如选课表SC的sno是外键引用Student的主键sno，同时sno+cno组合是SC的候选码/主键）。

2. **候选码不一定只有一个属性**：多个属性的组合也可以构成候选码，只要这些属性的组合满足唯一性和最小性。例如选课表SC的属性组合{sno, cno}的候选码（学号和课程号组合唯一），单独的sno或cno都不能唯一标识一行。

3. **超码和候选码的判定需要基于语义（现实世界约束）而非当前数据实例**：不能仅因为目前表中某一列没有重复就说它是候选码。候选码具有语义性质——在关系所表达的语义中必然不能重复。例如身份证号在中国公民中是唯一的（语义保证），而姓名可能在当前数据中不重复但未来可能重复（不是语义保证所以不是候选码）。

4. **外键不是必须指向主键**：外键可以指向任何候选码（自然也可以是主键以外的其他候选键）。实践中为简化通常选择主键作为外键的被参照属性。这一点容易被"外键总是引用主键"的习惯性思维固化。

## 例题

**例题1**：关系模式R(A, B, C, D)，已知函数依赖集F={A→B, B→C, D→A}。求R的所有候选码。

**解答**：候选码是能函数决定所有属性的最小属性集。从依赖图分析：D→A→B→C，所以D可以决定A, A决定B, B决定C——得出D→ABCD。D的属性闭包D+= {D, A, B, C} = ABCD（全部属性集），且D是单个属性（显然不能再减小），因此D是候选码。此外检查其他：A+ = {A, B, C}（缺D）；B+ = {B, C}；C+ = {C}。额外的候选码有吗？A+和B+都不是全属性，任何包含BC的属性集合不增加闭包大小。候选码仅{D}一个。所求关系只有一个候选码D。非主属性为A, B, C（因为它们都不含在任何候选码中——D是唯一的码且A/B/C不是其组成部分）。

**例题2**：外码(外键)的级联操作(CASCADE vs SET NULL vs RESTRICT)分别适用哪些场景？给出SQL示例。

**解答思路**：DELETE级联操作：删除被参照表的主键行时自动删除参照表中所有关联的外键行——适用于强从属关系（如订单删除时删除其下所有订单行）。SET NULL：将被参照主键删除时，参照表外键设为NULL——适用于可选从属关系（如部门解散，员工所属部门置空而不删除员工）。RESTRICT/NO ACTION：如果参照表中有关联行，拒绝删除被参照行——适用于保护主实体（如还有学生属于某系，拒绝删除该系）。SQL定义示例：
```sql
CREATE TABLE Employee (
    emp_id INT PRIMARY KEY,
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES Department(dept_id)
    ON DELETE SET NULL ON UPDATE CASCADE
);
```

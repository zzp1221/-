## 核心定义

握手定理（Handshaking Lemma/Theorem）是图论中最基本的定理之一。对于任意无向图 G = (V, E)，所有顶点的度数之和等于边数的两倍：∑_{v∈V} deg(v) = 2|E|。其中顶点度数 deg(v) 表示与 v 相关联的边数（自环计两次）。这一定理得名于"握手"的比喻：在一个聚会中，总握手次数等于每人握手次数之和的一半（因为每次握手涉及两个人）。握手定理的一个直接且重要的推论是：**在任何无向图中，奇度顶点（度数为奇数的顶点）的个数必为偶数**。这是因为度数总和为偶数（2|E|），若奇度顶点的个数为奇数，总和将为奇数，产生矛盾。对于有向图，有类似的定理：所有顶点的入度之和 = 所有顶点的出度之和 = 边数，即 ∑_{v∈V} deg⁺(v) = ∑_{v∈V} deg⁻(v) = |E|。

## 关键结论

- 所有顶点的度数和必为偶数，这约束了可能的度数序列——一个序列是某个简单图的度数序列的必要条件是度数和为偶数
- 奇度顶点（Odd Degree Vertex）的个数必为偶数——这是图论证明中常用的"奇偶论证"工具
- 对任意简单图，若图中有度数为 n-1 的顶点（即该顶点与其他所有顶点均相邻），则不可能同时存在度数为 0 的顶点（孤立点）
- 度数序列（Degree Sequence）的可图性判定由 Erdos-Gallai 定理给出充要条件
- 握手定理还可以推广到超图（Hypergraph）中

## 易错点

1. 计算度数时忘记自环计算：在无向图中，自环为其端点贡献度数为 2（而非 1），因为自环的两个端点重合但计数两次
2. 混淆有向图的入度和出度：入度之和 = 出度之和 = 边数，但某个顶点的入度和出度可以完全不同
3. 误认为握手定理可以证明图的唯一性：握手定理只提供度数和的必要条件，远非充分条件

## 例题

**例题1**：是否存在一个具有 5 个顶点、度数分别为 4, 4, 3, 3, 2 的简单图？

**解答**：度数和 = 4+4+3+3+2 = 16，为偶数（满足握手定理）。但有度数为 4 的顶点（与其他 4 个顶点都相邻），故每个顶点的度数至少为 1（通过与该顶点相邻），然而序列中没有度数为 0 或 1 的顶点，所以这是可能的。进一步检验 Erdos-Gallai 条件，该序列确实可图。

**例题2**：某次聚会有 9 人参加，已知其中 4 人各握了 1 次手，3 人各握了 2 次手，2 人各握了 3 次手。问这个描述是否合理？

**解答**：总握手次数 = (4×1 + 3×2 + 2×3) / 2 = (4+6+6) / 2 = 8。但奇度顶点的个数：4 人握 1 次（奇度数），共 4 人（偶数），满足推论。计算合理。

## 代码示例

```python
def handshaking_check(degrees):
    """检查度数序列是否满足握手定理必要条件"""
    total = sum(degrees)
    odd_count = sum(1 for d in degrees if d % 2 == 1)
    
    if total % 2 != 0:
        return False, "度数和不为偶数"
    if odd_count % 2 != 0:
        return False, "奇度顶点个数不为偶数"
    return True, "通过握手定理检查"

# 测试
seq1 = [4, 4, 3, 3, 2]
seq2 = [3, 3, 3, 2, 2]  # 3个奇度顶点
seq3 = [5, 4, 3, 2, 1]

for seq in [seq1, seq2, seq3]:
    ok, msg = handshaking_check(seq)
    print(f"{seq}: {msg}")

# 计算一个图的总度数
def total_degree(graph):
    return sum(len(neighbors) for neighbors in graph.values())

graph = {0: [1, 2], 1: [0, 2, 3], 2: [0, 1, 3], 3: [1, 2]}
E = sum(len(v) for v in graph.values()) // 2
print(f"度数和={total_degree(graph)}, 2|E|={2*E}, 相等={total_degree(graph)==2*E}")
```

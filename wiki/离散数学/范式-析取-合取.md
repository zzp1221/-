## 核心定义

范式（Normal Form）是命题逻辑中复合命题的标准化表示形式。**析取范式**（Disjunctive Normal Form, DNF）是若干合取式的析取，即形如 (A₁₁∧A₁₂∧...) ∨ (A₂₁∧A₂₂∧...) ∨ ...，其中每个 Aᵢⱼ 是原子命题或其否定。**合取范式**（Conjunctive Normal Form, CNF）是若干析取式的合取，即形如 (A₁₁∨A₁₂∨...) ∧ (A₂₁∨A₂₂∨...) ∧ ...，其中每个 Aᵢⱼ 同样是原子命题或其否定（称为文字，Literal）。DNF 中的每个合取项称为**极小项**（Minterm），CNF 中的每个析取项称为**极大项**（Maxterm）。主析取范式（PDNF）要求每个合取项包含所有原子命题恰好一次（每个变量必出现，或为其本身，或为其否定）；主合取范式（PCNF）有类似的完备性要求。任何非永假的命题都可以等价地转化为 DNF，任何非永真的命题都可以等价地转化为 CNF——这是命题逻辑的**范式存在定理**。

## 关键结论

- 任何命题公式都存在与之等价的 DNF 和 CNF（对于永真式只能用特殊方式处理 CNF，对于永假式只能用特殊方式处理 DNF）
- 从真值表可直接读出 DNF：考察所有使公式为真的行，每行构造一个合取项（极小项），所有合取项做析取即得 DNF
- 从真值表可直接读出 CNF：考察所有使公式为假的行，每行构造一个析取项（极大项），所有析取项做合取即得 CNF
- 析取范式与合取范式是最常用的两种范式，在数字电路设计（DNF对应SOP，CNF对应POS）和SAT求解中应用广泛
- 3-CNF（每个子句恰好3个文字）是SAT问题的经典形式，Cook-Levin定理证明了3-SAT是NP完全的

## 易错点

1. 混淆析取范式与合取范式的结构：DNF 是"和的积"（∨ of ∧），CNF 是"积的和"（∧ of ∨），记忆口诀：DNF = Disjunction of Conjunctions, CNF = Conjunction of Disjunctions
2. 在构造主范式时遗漏变量：主范式要求每个子句包含所有变量，若某变量未出现，需要通过 p ≡ (p ∧ (q ∨ ¬q)) 的方式引入
3. 误以为合取范式和析取范式是唯一的：非主范式不唯一，只有主析取范式（PDNF）和主合取范式（PCNF）才是唯一的（在原子命题顺序固定的前提下）

## 例题

**例题1**：将 p → q 转化为析取范式和合取范式。

**解答**：
p → q ≡ ¬p ∨ q （蕴含等价式）
¬p ∨ q 本身既是 DNF（一个析取式，每个析取项是单个文字的"合取"），也是 CNF（一个合取式，每个合取项是单个文字的"析取"）。

主析取范式（PDNF）：p → q 在 p=T,q=F 时为假，其余三行为真。故：
PDNF = (p∧q) ∨ (p∧¬q) ∨ (¬p∧q) [不必要，此处略] 实际上 PDNF = (p∧q) ∨ (¬p∧q) ∨ (¬p∧¬q)

主合取范式（PCNF）：p → q 在 p=T,q=F 时为假，仅一行。故 PCNF = ¬p ∨ q（恰好包含所有变量，即为PCNF）。

**例题2**：通过真值表法将 (p ∨ q) → r 转化为合取范式。

真值表中为假的指派：(p=T,q=T,r=F), (p=T,q=F,r=F), (p=F,q=T,r=F)，对应极大项：(¬p ∨ ¬q ∨ ¬r), (¬p ∨ q ∨ ¬r), (p ∨ ¬q ∨ ¬r)。

PCNF = (¬p ∨ ¬q ∨ ¬r) ∧ (¬p ∨ q ∨ ¬r) ∧ (p ∨ ¬q ∨ ¬r)

## 代码示例

```python
import itertools

def to_dnf(formula, variables):
    """从真值表生成析取范式 (DNF)"""
    terms = []
    for values in itertools.product([True, False], repeat=len(variables)):
        assign = dict(zip(variables, values))
        if formula(assign):  # 该行使公式为真
            literals = []
            for v in variables:
                if assign[v]:
                    literals.append(v)
                else:
                    literals.append(f"¬{v}")
            terms.append("(" + " ∧ ".join(literals) + ")")
    
    if not terms:
        return "FALSE"
    return " ∨ ".join(terms)

# 测试: p → q
def implies(assign):
    return (not assign['p']) or assign['q']

print(to_dnf(implies, ['p', 'q']))
# 输出类似: (p ∧ q) ∨ (¬p ∧ q) ∨ (¬p ∧ ¬q)
```

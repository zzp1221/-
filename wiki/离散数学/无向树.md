## 核心定义

树（Tree）是图论中最重要的特殊图类，其定义有多种等价方式。一个无向树是一个**连通且无圈**（Acyclic）的无向图；**森林**（Forest）是无圈的（不一定连通的）无向图。对于 n 个顶点的树 T，以下性质彼此等价：(1) T 是连通且无圈的；(2) T 是连通图且有恰好 n−1 条边；(3) T 是无圈图且有恰好 n−1 条边；(4) T 中任意两个顶点之间有且仅有一条路径；(5) T 是极小连通图——删除任意一条边都会使图不连通；(6) T 是极大无圈图——添加任意一条边都会产生唯一的圈。树中**度为 1** 的顶点称为**叶子**（Leaf），度 > 1 的顶点称为**分支点**（Internal Vertex / Branch Node）。任意至少有两个顶点的树至少有两个叶子——这是树的基本性质。在数据结构中，树被广泛用于表示层次化数据（如文件系统、HTML DOM、组织结构图等）。

## 关键结论

- n 个顶点的树恰有 n−1 条边——这是树的最基本数量特征
- 树中任意两顶点间恰有一条简单路径，保证了树形结构在路由和搜索中的高效性
- 每个至少有两个顶点的树至少有两个叶子，这是通过在树中取最长路径的端点证明的
- 树的补图（在完全图意义下）一般不是树，因为完全图 Kₙ 有 n(n−1)/2 条边，减去 n−1 后边数远大于 n−1（n≥5 时）
- 森林的连通分量数 = 顶点数 − 边数；若森林有 k 棵树，则边数 = n − k
- Cayley定理：n 个有标号顶点的不同树有 n^(n−2) 个（Prufer编码证明）

## 易错点

1. 认为树一定有根：无向树（自由树）是无根的结构，根树（Rooted Tree）才是选择一个顶点作为层次根的有向（或有层次关系）变体
2. 混淆树的中点与重心：树的中心是偏心距最小的顶点，重心是以该点为根时最大子树规模最小的顶点
3. 误认为所有无圈图都是树：无圈但不连通的图是森林而非树

## 例题

**例题1**：证明任何有 n 个顶点和 n−1 条边的连通图一定是树。

**解答**：反证法。若 G 不是树，则 G 连通但含圈。从圈中删除一条边，G 仍连通，边数变为 n−2。重复此操作直到无圈为止，得到一棵树 T，T 有 n 个顶点，边数 < n−1。但任何 n 个顶点的树恰有 n−1 条边，矛盾。故 G 无圈，即为树。

**例题2**：一个树有 5 个度为 3 的顶点、3 个度为 2 的顶点，其余全为叶子。求叶子数和总顶点数。

**解答**：设叶子数（度=1）为 ℓ，总顶点数 n = 5+3+ℓ = 8+ℓ。由握手定理：5×3 + 3×2 + ℓ×1 = 2(n−1) = 2(7+ℓ)。即 15+6+ℓ = 14+2ℓ，21+ℓ = 14+2ℓ，ℓ = 7。总顶点数 n = 15。

## 代码示例

```python
def is_tree(graph):
    """判断无向图是否为树: 连通且无圈"""
    vertices = list(graph.keys())
    n = len(vertices)
    if n == 0:
        return True
    
    # BFS检查连通性
    visited = set()
    stack = [vertices[0]]
    while stack:
        v = stack.pop()
        if v not in visited:
            visited.add(v)
            for u in graph[v]:
                if u not in visited:
                    stack.append(u)
    
    if len(visited) != n:
        return False  # 不连通
    
    # 计算边数: 树应有 n-1 条边
    edge_count = sum(len(graph[v]) for v in graph) // 2
    return edge_count == n - 1

# 一棵简单的树
tree = {0: [1, 2], 1: [0], 2: [0, 3, 4], 3: [2], 4: [2]}
print(f"是树: {is_tree(tree)}")  # True (5顶点, 4条边)

# 含圈的图
cyclic = {0: [1, 2], 1: [0, 2], 2: [0, 1]}
print(f"是树: {is_tree(cyclic)}")  # False (3顶点但3条边)
```

## 核心定义

欧拉路径与哈密顿路径是图论中的两类经典路径问题。**欧拉回路**（Eulerian Circuit/Cycle）是经过图中每条边**恰好一次**并回到起点的闭迹；包含欧拉回路的图称为**欧拉图**（Eulerian Graph）。欧拉路径（Eulerian Trail）是经过每条边恰好一次但不要求回到起点的迹。欧拉图定理提供了简洁的判定条件：一个连通无向图是欧拉图当且仅当所有顶点的度数均为**偶数**；存在欧拉路径（但不闭合）当且仅当图恰有两个奇度顶点（这两个奇度顶点分别作为起点和终点）。**哈密顿回路**（Hamiltonian Cycle）是经过图中每个顶点**恰好一次**并回到起点的圈；**哈密顿路径**是经过每个顶点恰好一次但不要求回到起点的路径。包含哈密顿回路的图称为**哈密顿图**（Hamiltonian Graph）。与欧拉图不同，哈密顿图的判定没有简单而漂亮的充要条件——哈密顿回路的存在性问题是**NP完全的**。充分的判定条件有：狄拉克定理（Dirac）：若 n ≥ 3 且最小度 δ ≥ n/2，则图为哈密顿图；奥尔定理（Ore）：若对任意不相邻顶点 u, v 有 deg(u)+deg(v) ≥ n，则图为哈密顿图。

## 关键结论

- 欧拉回路问题（Konigsberg七桥问题）是图论的历史起源，欧拉在1736年的论文中解决了该问题
- 欧拉回路的存在性可在多项式时间（O(V+E)）内判定和构造（Fleury算法或Hierholzer算法）
- 哈密顿回路的存在性判定是**NP完全**的，无法在多项式时间内解决（除非 P=NP）
- TSP（旅行商问题）：在加权完全图中寻找最小权哈密顿回路，是著名的 NP 难优化问题
- 中国邮递员问题：在带权图中找经过每条边至少一次的最小权闭通路（欧拉图特例下恰为欧拉回路）
- 哈密顿图的必要条件：删除任意 k 个顶点后，连通分量数不超过 k

## 易错点

1. 混淆欧拉回路与哈密顿回路：欧拉回路重"边"（每条边恰好一次），哈密顿回路重"顶点"（每个顶点恰好一次）
2. 认为哈密顿回路一定包含所有边：哈密顿回路仅要求经过所有顶点恰好一次，不要求经过所有边
3. 将 Dirac/Ore 定理的必要性误用：这两个只是充分条件而非必要条件——一个图完全可能是哈密顿图但 δ < n/2

## 例题

**例题1**：判断完全图 K₅ 是否为欧拉图和哈密顿图。

**解答**：K₅ 每个顶点度数为 4（偶数），连通，故为欧拉图。n=5, δ=4 ≥ 5/2=2.5，由 Dirac 定理，K₅ 为哈密顿图。事实上所有 n≥3 的完全图都是哈密顿图。

**例题2**：证明若连通无向图有 2k 个奇度顶点（k>0），则该图可以分解为 k 条边不相交的欧拉路径。

**解答**：添加 k 条"虚边"连接配对奇度顶点，使所有顶点度数为偶数，得到一个欧拉图，存在欧拉回路。删去 k 条虚边后，欧拉回路被断开为 k 条欧拉路径，每条路径以一对奇度顶点为端点。

## 代码示例

```python
def hierholzer(graph):
    """找欧拉回路 (前提：图是连通的且所有顶点度数为偶)"""
    from collections import defaultdict
    adj = defaultdict(list)
    for u, neighbors in graph.items():
        adj[u] = list(neighbors)
    
    stack = [next(iter(graph))]
    circuit = []
    
    while stack:
        v = stack[-1]
        if adj[v]:
            u = adj[v].pop()
            adj[u].remove(v)
            stack.append(u)
        else:
            circuit.append(stack.pop())
    
    return circuit[::-1]

# K_5的一条欧拉回路示例 (每个顶点的度都是4=偶数)
K5 = {0: [1,2,3,4], 1: [0,2,3,4], 2: [0,1,3,4], 
      3: [0,1,2,4], 4: [0,1,2,3]}
circuit = hierholzer(K5)
print(f"欧拉回路长度: {len(circuit)-1} 条边")
```

## 核心定义

外部排序是指待排序的记录数量太大，不能一次性全部装入内存，需要借助外存（磁盘、SSD）进行的排序。外部排序通常采用归并排序的思想，分为两个阶段：(1) 生成初始归并段——将数据分批读入内存，用内部排序（如快排/置换选择排序）生成若干有序段写回磁盘；(2) 多路归并——将多个归并段合并为更大的有序段，反复直到只剩一个有序段。多路归并的瓶颈在于每一趟归并都需要大量磁盘 I/O。K 路归并共需 log_k(m) 趟（m 为初始段数）。为了减少趟数，可增大归并路数 k，此时需用"败者树"或"堆"来维护 k 个输入流中的最小值。最佳归并树（哈夫曼树）通过不等长归并段来减少总 I/O。

## 关键结论

- 外部排序的总时间 = 内部排序时间 + 磁盘 I/O 时间（主要瓶颈）
- 初始归并段越多，归并趟数越多，I/O 越大
- 置换选择排序（Replacement Selection）可生成平均长度 2*M 的归并段（M 为内存容量）
- 败者树（Tournament Tree）是 k 路归并的高效选最小值数据结构，每次选 min 只需 O(log k)
- 最佳归并树：以段长为权值构造哈夫曼树，较短的段先合并，使总 I/O 最小

## 排序复杂度对比

| 排序算法 | 最好 | 平均 | 最坏 | 空间 | 稳定性 |
|----------|------|------|------|------|--------|
| 冒泡排序 | O(n) | O(n^2) | O(n^2) | O(1) | 稳定 |
| 插入排序 | O(n) | O(n^2) | O(n^2) | O(1) | 稳定 |
| 选择排序 | O(n^2) | O(n^2) | O(n^2) | O(1) | 不稳定 |
| 希尔排序 | O(n) | O(n^1.3) | O(n^2) | O(1) | 不稳定 |
| 快速排序 | O(nlogn) | O(nlogn) | O(n^2) | O(logn) | 不稳定 |
| 归并排序 | O(nlogn) | O(nlogn) | O(nlogn) | O(n) | 稳定 |
| 堆排序 | O(nlogn) | O(nlogn) | O(nlogn) | O(1) | 不稳定 |
| 基数排序 | O(dn) | O(dn) | O(dn) | O(n+r) | 稳定 |

## 易错点

1. 外部排序 ≠ 某一种算法，而是利用内部排序+多路归并在外存上完成排序的框架
2. 败者树 vs 胜者树：败者树记录失败者，重构只需向上比较到父结点而非根，效率更高
3. 口诀记忆：选"不稳定"的有快选堆希（快排、选择、堆、希尔）；稳定的可记"归并插冒基"（归并、插入、冒泡、基数）

## 代码示例

```cpp
// 败者树（简化版：用于 k 路归并）
class LoserTree {
    vector<int> leaves;   // k 个叶子（各归并段的当前值）
    vector<int> losers;   // 败者树节点
    int k;
    
    void adjust(int s) {
        int parent = (s + k) / 2;
        while (parent > 0) {
            if (s != -1 && (losers[parent] == -1 ||
                leaves[s] > leaves[losers[parent]])) {
                swap(s, losers[parent]);  // s 是胜者，原败者继续向上
            }
            parent /= 2;
        }
        losers[0] = s;  // 最终胜者
    }
    
public:
    void build(vector<int>& initValues) {
        k = initValues.size();
        leaves = initValues;
        losers.assign(k, -1);
        for (int i = k - 1; i >= 0; i--)
            adjust(i);
    }
    
    int getWinner() { return losers[0]; }
};
```

## 核心定义

顺序查找（Sequential Search）又称线性查找，是最基本的查找算法。其思想极为简单：从表的一端开始，依次将每个记录的关键字与给定值 key 进行比较，若某个记录的关键字等于 key 则查找成功；若直到另一端还未找到，则查找失败。顺序查找既适用于顺序表，也适用于链表；既适用于有序表，也适用于无序表。查找成功的平均查找长度（ASL）为 (n+1)/2（期望比较次数为一半），查找失败的比较次数为 n（或 n+1）。顺序查找的时间复杂度为 O(n)，是 n 数量级，效率较低但实现最简单。使用"哨兵"技巧可以避免每次比较时检查是否越界：将 key 放在 data[0] 处作为哨兵，从后向前查找，省去循环内判断 i>=0 的开销。

## 关键结论

- 时间复杂度 O(n)，平均查找长度 ASL_success = (n+1)/2，ASL_fail = n
- 既适用于顺序表也适用于链表，对数据的排列无要求
- 哨兵技巧可以减少循环内的判断条件，提高效率
- 当 n 较小时顺序查找效率可以接受
- 适合数据量小、无序、且不常做查找的场景

## 易错点

1. 哨兵位置的约定：哨兵放在 data[0] 时需要确保 data[0] 不是有效数据，即有效数据从 data[1] 到 data[n]。不要将哨兵放在 data[n+1]（可能越界）
2. 查找成功/失败的返回条件：成功时返回位置 i（对于哨兵法返回从 n 到 1 的下标），失败时返回 0（哨兵位置）
3. 顺序查找不适合数据量大且频繁查找的场景，此时应考虑索引或哈希

## 例题

**例1：** 在长度为 100 的顺序表中进行顺序查找，查找成功的平均比较次数和查找失败的平均比较次数分别是？

**解答：** 查找成功：ASL_success = (100+1)/2 = 50.5 次；查找失败：需比较所有 100 个元素（不含哨兵），即 100 次。

## 代码示例

```cpp
// 带哨兵的顺序查找（data[0] 作为哨兵，数据从 1 到 n）
int seqSearch(int data[], int n, int key) {
    data[0] = key;  // 设置哨兵
    int i = n;
    while (data[i] != key) i--;
    return i;  // i=0 表示未找到
}

// 普通顺序查找
int seqSearchBasic(int data[], int n, int key) {
    for (int i = 0; i < n; i++)
        if (data[i] == key)
            return i;
    return -1;
}
```

```java
public int seqSearch(int[] data, int key) {
    for (int i = 0; i < data.length; i++)
        if (data[i] == key) return i;
    return -1;
}
```

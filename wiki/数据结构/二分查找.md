## 核心定义

二分查找（Binary Search）又称折半查找，是一种高效的查找算法，但要求查找表必须是有序的（通常是升序排列），且采用顺序存储结构（支持随机访问）。其思想是：在有序表中，取中间位置的元素与关键字 key 比较——若相等则查找成功；若 key 小于中间元素，说明 key 只可能在左半部分，则在左半部分继续二分查找；若 key 大于中间元素，则在右半部分继续。每次比较将搜索范围缩小一半，时间复杂度为 O(log n)。二分查找同样适用于斐波那契查找和插值查找等变种，本质均为"在有序数据上计算下一个查找位置以逼近目标"。

## 关键结论

- 时间复杂度 O(log n)，每次将搜索空间减半
- 只适用于有序的顺序表（需要随机访问），不能用于链表
- 查找成功的 ASL 约为 log2(n+1) - 1；查找过程可用递归或非递归实现
- 判定树（二叉判定树）描述了二分查找的比较过程，树的高度为 ceil(log2(n+1))
- 非递归实现时 while 循环条件为 low <= high（而非 low < high）

## 易错点

1. 循环条件的设置：必须是 `while(low <= high)` 而非 `while(low < high)`，否则当 low==high 且该位置确实等于目标值时可能会错失
2. mid 的计算溢出问题：`mid = (low + high) / 2` 在 low+high 很大时可能溢出，应改为 `mid = low + (high - low) / 2`
3. 二分查找不能用于链表：因为每次需要 O(1) 时间访问中间元素，链表需要 O(n) 时间遍历到中间位置，退化

## 例题

**例1：** 在包含 11 个元素的有序表中做二分查找，等概率下查找成功的平均查找长度（ASL）是多少？

**解答：** 构造判定树，层次为 1,2,3,3,4,4,4,4（共 11 个结点）。ASL = (1*1 + 2*2 + 4*3 + 4*4)/11 = (1+4+12+16)/11 = 33/11 = 3。

## 代码示例

```cpp
// 非递归二分查找
int binarySearch(int data[], int n, int key) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;  // 防止溢出
        if (data[mid] == key)
            return mid;
        else if (key < data[mid])
            high = mid - 1;
        else
            low = mid + 1;
    }
    return -1;
}

// 递归二分查找
int binarySearchRecursive(int data[], int low, int high, int key) {
    if (low > high) return -1;
    int mid = low + (high - low) / 2;
    if (data[mid] == key) return mid;
    if (key < data[mid]) return binarySearchRecursive(data, low, mid - 1, key);
    else return binarySearchRecursive(data, mid + 1, high, key);
}
```

```java
public int binarySearch(int[] data, int key) {
    int low = 0, high = data.length - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;
        if (data[mid] == key) return mid;
        if (key < data[mid]) high = mid - 1;
        else low = mid + 1;
    }
    return -1;
}
```

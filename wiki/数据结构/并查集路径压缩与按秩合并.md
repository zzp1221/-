## 核心定义

并查集（Union-Find/Disjoint Set Union, DSU）是管理不相交集合的数据结构，支持合并（Union）和查询（Find）操作。基本并查集通过树结构表示集合，根节点为集合代表。两个核心优化：(1)路径压缩（Path Compression）：Find时将路径上所有节点直接指向根节点，使后续查询O(1)。实现：`find(x) { if (parent[x]!=x) parent[x]=find(parent[x]); return parent[x]; }`。(2)按秩合并（Union by Rank）：合并时将矮树挂在高树下面，避免树退化为链表。实现：合并时比较rank，rank小的树根指向rank大的树根；rank相同时任选一个并rank++。两种优化单独使用都能达到O(log n)均摊复杂度，同时使用（路径压缩+按秩合并）达到O(α(n))均摊复杂度，其中α是反阿克曼函数，对实际数据几乎为常数（α(2^65536)≤4）。并查集的应用：连通分量判断、Kruskal最小生成树、最近公共祖先（Tarjan离线算法）、动态连通性。

## 关键结论

- 路径压缩和按秩合并同时使用时，单次操作均摊O(α(n))，几乎是O(1)
- 按秩合并可以用"按大小合并"（Union by Size）替代，效果相同
- 路径压缩在递归实现时简洁但有栈溢出风险，迭代实现更安全
- 并查集不支持"删除元素"操作：删除需要额外的数据结构支持（如反向并查集）
- 带权并查集：边权表示节点间的关系（如距离、种类关系），路径压缩时需要更新权值

## 易错点

1. 路径压缩必须在Find时执行，不是在Union时：Union只合并根节点
2. 按秩合并的rank不等于树的实际高度：路径压缩后高度可能比rank小
3. 带权并查集的路径压缩需要同时更新权值：`weight[x] += weight[parent[x]]`

## 例题

**例1：** 对以下操作序列，画出最终的并查集结构（使用路径压缩+按秩合并）：Union(1,2), Union(3,4), Union(2,4), Find(1), Find(3)。

**解答：** 初始：每个元素独立，rank都为0。Union(1,2)：rank相同，2→1，rank[1]=1。Union(3,4)：rank相同，4→3，rank[3]=1。Union(2,4)：find(2)=1(rank=1), find(4)=3(rank=1)，rank相同，3→1，rank[1]=2。Find(1)：直接返回1。Find(3)：parent[3]=1，路径压缩：直接返回1，parent[3]=1（已经指向根）。Find(4)：parent[4]=3，parent[3]=1，路径压缩：parent[4]=1，返回1。最终结构：1为根(rank=2)，2→1，3→1，4→1（扁平化）。

## 核心定义

循环链表（Circular Linked List）是另一种形式的链式存储结构。它的特点是表中最后一个结点的指针域不再为 NULL，而是指向头结点（或第一个结点），从而整个链表形成一个环。从循环链表中的任意一个结点出发都可以找到表中所有其他结点。循环链表通常也带有头结点，空表的判断条件为 `head->next == head`。由于形成了闭环，任何遍历算法都必须设置终止条件（通常用 `while(p != head)` 来判断是否回到了起点），否则会陷入无限循环。循环链表常用于需要循环访问数据的场景，如轮流调度、约瑟夫问题等。

## 关键结论

- 循环单链表的尾结点指针指向头结点，形成闭环
- 从任意结点出发都能遍历整个链表，使操作更加灵活
- 空循环链表的条件是头结点的 next 指向自身：`head->next == head`
- 遍历时必须判断当前指针是否回到了起始位置，而不能以 NULL 为终止条件
- 常采用尾指针表示法（而非头指针），因为通过尾指针可以 O(1) 访问表头和表尾

## 易错点

1. 遍历终止条件错误：循环链表中，p->next 永远不会为 NULL，需使用 `while(p->next != head)` 或 `while(p != head)` 来控制循环。忘记修改终止条件会导致死循环
2. 尾指针和头指针的区分：使用尾指针表示循环链表时，`rear->next->next` 表示首元结点，`rear` 本身是尾结点，`rear->next` 是头结点（或头指针）
3. 合并两个循环链表时忘记更新两个尾结点的链接：先将 rear1->next （即 head1）接到 rear2->next->next（即第二个表的首元结点），再将 rear2->next 指向第一个表的头结点，最后更新尾指针

## 例题

**例1：** 用带尾指针的循环单链表实现约瑟夫（Josephus）问题——n 个人围成一圈，从第 k 个人开始报数，数到 m 的人出列，求出列顺序。

**解答思路：** 建立 n 个结点的循环单链表，用尾指针定位。从第 k 个结点开始，循环移动 m-1 步后删除当前结点的后继结点（即出列者），直到链表为空。

## 代码示例

```cpp
// 判断循环单链表是否为空
bool isEmpty(Node* head) {
    return head->next == head;
}

// 遍历循环链表（打印所有元素）
void traverse(Node* head) {
    if (isEmpty(head)) return;
    Node* p = head->next;  // 从首元结点开始
    while (p != head) {
        cout << p->data << " ";
        p = p->next;
    }
}

// 约瑟夫问题求解
int josephus(int n, int k, int m) {
    // 创建 n 个结点的循环链表（无头结点版本）
    Node* head = new Node(1);
    Node* prev = head;
    for (int i = 2; i <= n; i++) {
        prev->next = new Node(i);
        prev = prev->next;
    }
    prev->next = head;  // 形成环
    
    Node* p = head;
    // 找到第 k 个人
    for (int i = 1; i < k; i++)
        p = p->next;
    
    // 循环删除直到只剩一个结点
    while (p->next != p) {
        // 移动 m-1 步，找到出列者的前驱
        for (int i = 1; i < m - 1; i++)
            p = p->next;
        Node* q = p->next;   // 出列者
        p->next = q->next;
        delete q;
        p = p->next;
    }
    int result = p->data;
    delete p;
    return result;
}
```

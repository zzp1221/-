## 核心定义

双端队列（Deque，全称 Double-Ended Queue）是一种允许在两端进行插入和删除操作的线性表。它融合了栈和队列的特点：元素可以从两端进入，也可以从两端离开。双端队列分为两类：(1) 输入受限的双端队列——插入操作仅限一端，删除可在两端；(2) 输出受限的双端队列——删除操作仅限一端，插入可在两端。在算法中，双端队列常用于滑动窗口最大值问题、单调队列、BFS 中的 0-1 最短路等。STL 中的 `std::deque` 采用分段连续存储结构（中控器+多个缓冲区），支持 O(1) 的随机访问和首尾 O(1) 的插入删除。

## 关键结论

- 双端队列支持 O(1) 时间在头尾进行插入和删除
- 兼具栈和队列的功能：一端操作是栈，两端分别进出是队列
- 单调队列是双端队列的重要应用（维护滑动窗口最值）
- C++ STL 的 deque 内部使用"中控器+缓冲区"的分段结构
- 双端队列可以退化为普通的栈（只在一端操作）或队列（一端进另一端出）

## 易错点

1. 将 deque 误认为就是简单的循环队列：STL 的 deque 实现远比循环队列复杂，是分段连续结构；但算法中手动实现的双端队列确实可以用循环数组完成
2. 单调队列的维护规则：入队时从队尾弹出所有比当前元素小（或大）的元素以保持单调性；出队时从队头弹出过期元素；两端都可能发生删除
3. 输入限制与输出限制的概念混淆："输入受限"是指只有一端可以插入，"输出受限"是指只有一端可以删除。考试中常让判断某个序列是否是受限双端队列的合法排列

## 例题

**例1：** 利用双端队列解决滑动窗口最大值问题——给定数组 [1,3,-1,-3,5,3,6,7] 和窗口大小 k=3，求每个窗口的最大值。

**解答思路：** 维护一个单调递减的双端队列（存储下标）。遍历数组，对每个元素：先将队列中"越界"的（下标 <= i-k）从队头弹出；再将队列中"小于当前值"的从队尾弹出；将当前下标入队尾；窗口形成后（i >= k-1），队头即当前窗口最大值。结果为 [3,3,5,5,6,7]。

## 代码示例

```cpp
#include <deque>
#include <vector>
using namespace std;

// 滑动窗口最大值：单调递减双端队列
vector<int> maxSlidingWindow(vector<int>& nums, int k) {
    deque<int> dq;  // 存储下标而非值
    vector<int> res;
    for (int i = 0; i < nums.size(); i++) {
        // 移除越界元素（队头）
        while (!dq.empty() && dq.front() <= i - k)
            dq.pop_front();
        // 保持单调递减：从队尾移除小于当前值的
        while (!dq.empty() && nums[dq.back()] < nums[i])
            dq.pop_back();
        dq.push_back(i);
        // 窗口形成后记录
        if (i >= k - 1)
            res.push_back(nums[dq.front()]);
    }
    return res;
}
```

```java
public int[] maxSlidingWindow(int[] nums, int k) {
    Deque<Integer> dq = new ArrayDeque<>();
    int[] res = new int[nums.length - k + 1];
    int idx = 0;
    for (int i = 0; i < nums.length; i++) {
        while (!dq.isEmpty() && dq.peekFirst() <= i - k)
            dq.pollFirst();
        while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i])
            dq.pollLast();
        dq.offerLast(i);
        if (i >= k - 1)
            res[idx++] = nums[dq.peekFirst()];
    }
    return res;
}
```

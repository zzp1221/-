## 核心定义

后缀数组（Suffix Array, SA）是将字符串S的所有后缀按字典序排序后的数组。SA[i]表示排序后第i小的后缀的起始位置。后缀数组配合LCP数组（Longest Common Prefix，相邻后缀的最长公共前缀长度）可以高效解决多种字符串问题。构建后缀数组的最优算法：SA-IS算法（线性时间O(n)）、DC3/Skew算法（线性时间）、倍增算法（O(n log²n)，实现简单）。后缀树（Suffix Tree）是将字符串所有后缀压缩存储的Trie树，Ukkonen算法可以在线性时间内构建后缀树。后缀树支持：最长重复子串（O(n)）、最长公共子串（O(n+m)）、模式匹配（O(m)）。后缀数组比后缀树空间效率更高（4n字节 vs 20n字节），实际工程中更常用后缀数组+LCP数组。

## 关键结论

- 后缀数组的空间开销是O(n)，后缀树的空间开销是O(n)但常数因子大（每个节点需要多个指针）
- LCP数组可以通过Kasai算法从后缀数组在线性时间内构建
- 后缀数组+LCP数组可以解决：模式匹配、最长重复子串、不同子串计数、最长回文子串等
- SA-IS算法是目前实践中最快的后缀数组构建算法
- 后缀自动机（SAM）是另一种强大的字符串数据结构，支持在线构建

## 易错点

1. 后缀数组的构建算法选择：竞赛中常用倍增（好写），工程中用SA-IS（最快）
2. LCP数组的含义是SA[i]和SA[i-1]的LCP，不是所有后缀两两的LCP
3. 后缀数组的区间查询需要配合RMQ（Range Minimum Query）才能高效求任意两个后缀的LCP

## 例题

**例1：** 给定字符串S="banana"，构建后缀数组和LCP数组。

**解答：** 所有后缀：banana(0), anana(1), nana(2), ana(3), na(4), a(5)。按字典序排序：a(5), ana(3), anana(1), banana(0), na(4), nana(2)。SA = [5, 3, 1, 0, 4, 2]。LCP数组：LCP[0]=0（a前面无后缀），LCP[1]=1（a与ana的LCP是"a"），LCP[2]=3（ana与anana的LCP是"ana"），LCP[3]=0（anana与banana无公共前缀），LCP[4]=0（banana与na无公共前缀），LCP[5]=2（na与nana的LCP是"na"）。LCP = [0, 1, 3, 0, 0, 2]。最大LCP值3对应"ana"，是S的最长重复子串。

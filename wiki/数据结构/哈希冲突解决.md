## 核心定义

哈希冲突（Hash Collision）是指不同的关键字通过哈希函数映射到同一个哈希地址的现象。由于哈希表的大小有限而关键字空间通常很大，冲突是不可避免的。冲突解决技术分为两大类：(1) 开散列方法——如链地址法（每个桶维护一个链表/红黑树）+ 再哈希法（构造另一个哈希函数，当冲突发生时使用备用哈希函数计算新地址），Java HashMap 的红黑树退化为链表也是类似思路；(2) 闭散列方法——如开放定址法（线性探测、二次探测、伪随机探测）。此外还有公共溢出区法——所有冲突元素集中放入一个溢出表。评判哈希表效率的核心指标是负载因子 alpha = n/m 和平均查找长度 ASL。好的哈希函数应具有均匀性（均匀分布）、确定性（相同 key 相同值）和高效性（计算快）。

## 关键结论

- 冲突是必然的，优秀哈希表通过设计哈希函数和冲突解决方法来降低冲突代价
- 再哈希法：RH_i(key) = (H(key) + i * H'(key)) % m，H' 为第二哈希函数（取值与 m 互质）
- 开放定址中再哈希法（双散列）效果最好：两个哈希函数同时冲突的概率很低
- Java HashMap (JDK 8+)：链地址 + TREEIFY_THRESHOLD=8（链表超长转红黑树）
- 公共溢出区法适合冲突不太频繁且 n 远小于 m 的场景

## 易错点

1. 再哈希法中第二哈希函数不能返回 0：否则 d_i 始终为 0（探测步长为 0），退化为死循环
2. 链地址转红黑树的条件：不仅是链表长度 >= 8，还要求桶容量 >= 64（Java HashMap 实现细节），否则直接扩容而非树化
3. Hash 函数与冲突解决方法的配合：一个好的哈希函数应使 key 均匀分布在表空间内；如果哈希函数本身很差（如总是返回常数），任何冲突解决方法都无法挽救

## 例题

**例1：** 比较链地址法和开放定址法在相同负载因子下的性能特性。

**解答：** (1) 链地址法不受表大小限制，负载因子可大于 1，删除简单；但需要额外指针空间。(2) 开放定址法不需要额外空间，存储密度高；但受负载因子严重制约（通常 < 0.7），删除需要懒惰标记。(3) 缓存性能：开放定址法因数据连续而缓存友好；链地址法链表结点分散，缓存不友好。(4) 对于较小 key，开放定址法效率更高。

## 代码示例

```cpp
// 再哈希法（双散列）
class DoubleHash {
    vector<int> table;
    vector<bool> occupied;
    int m;
    
    int h1(int key) { return key % m; }
    
    int h2(int key) {
        // 第二哈希函数：返回 1 到 m-1 之间的奇数（与 m 互质）
        return 1 + (key % (m - 1));
    }
    
public:
    DoubleHash(int size) : m(size), table(size), occupied(size, false) {}
    
    bool insert(int key) {
        for (int i = 0; i < m; i++) {
            int idx = (h1(key) + i * h2(key)) % m;
            if (!occupied[idx]) {
                table[idx] = key;
                occupied[idx] = true;
                return true;
            }
            if (table[idx] == key) return false;  // 已存在
        }
        return false;  // 表满
    }
    
    int search(int key) {
        for (int i = 0; i < m; i++) {
            int idx = (h1(key) + i * h2(key)) % m;
            if (!occupied[idx]) return -1;  // 未找到
            if (table[idx] == key) return idx;
        }
        return -1;
    }
};
```

```java
// 公共溢出区法示例
public class OverflowHash {
    int[] table;      // 基本表
    List<Integer> overflow;  // 公共溢出区
    int m;
    
    public OverflowHash(int size) {
        m = size;
        table = new int[m];
        Arrays.fill(table, -1);
        overflow = new ArrayList<>();
    }
    
    public void insert(int key) {
        int idx = key % m;
        if (table[idx] == -1) table[idx] = key;
        else overflow.add(key);  // 冲突的元素放入溢出区
    }
    
    public boolean search(int key) {
        int idx = key % m;
        if (table[idx] == key) return true;
        return overflow.contains(key);  // 在溢出区中查找
    }
}
```

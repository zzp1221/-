## 核心定义

链栈（Linked Stack）是采用链式存储结构实现的栈。链栈通常以单链表为基础，栈顶设在链表的头部（头结点之后），栈底在链表尾部。由于只在链表头部进行插入和删除（对应 push 和 pop），不需要使用头结点，直接用一个头指针即可，空栈时头指针为 NULL。与顺序栈相比，链栈不存在空间溢出问题（除非内存耗尽），但每个元素需要额外的指针存储空间。链栈的所有基本操作时间复杂度均为 O(1)。链栈适用于栈的最大容量无法预估或变化范围很大的场景。

## 关键结论

- 链栈将表头作为栈顶，插入和删除均在表头进行，时间复杂度为 O(1)
- 链栈没有栈满问题，理论上可无限增长
- 链栈的存储密度低于顺序栈（需要额外存储指针）
- 链栈的空栈条件为栈顶指针 top == NULL
- 与顺序栈一样遵循 LIFO 原则，但实现方式为链式

## 易错点

1. 将栈顶设在链表尾部：若将尾结点作为栈顶，pop 时无法直接找到前驱，需要遍历整个链表，时间复杂度退化为 O(n)；必须将表头作为栈顶
2. 出栈后忘记释放结点：pop 操作中取出栈顶元素后需要 `delete/free` 释放之前分配的结点，否则造成内存泄漏
3. 入栈时指针链接方向错误：新结点的 next 应指向当前栈顶，再将栈顶指针指向新结点；若顺序颠倒会导致原链丢失

## 例题

**例1：** 链栈的进栈 push 操作中，栈顶与链表头部的对应关系是怎样的？

**解答：** 链栈的栈顶对应于链表的表头。push 等价于单链表的头插法：新结点的 next 指向当前 top，然后 top 更新为新结点。

## 代码示例

```cpp
template<typename T>
struct StackNode {
    T data;
    StackNode* next;
    StackNode(T val, StackNode* n = nullptr) : data(val), next(n) {}
};

template<typename T>
class LinkedStack {
private:
    StackNode<T>* top;  // 栈顶指针（即链表头）
public:
    LinkedStack() : top(nullptr) {}
    
    ~LinkedStack() {
        while (top) {
            StackNode<T>* p = top;
            top = top->next;
            delete p;
        }
    }
    
    void push(T e) {
        StackNode<T>* s = new StackNode<T>(e, top);
        top = s;
    }
    
    bool pop(T& e) {
        if (!top) return false;
        StackNode<T>* p = top;
        e = p->data;
        top = top->next;
        delete p;
        return true;
    }
    
    bool isEmpty() { return top == nullptr; }
};
```

```java
public class LinkedStack<T> {
    class Node {
        T data; Node next;
        Node(T val, Node n) { data = val; next = n; }
    }
    private Node top;
    
    public void push(T e) {
        top = new Node(e, top);
    }
    
    public T pop() {
        if (top == null) return null;
        T val = top.data;
        top = top.next;
        return val;
    }
}
```

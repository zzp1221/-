## 核心定义

分块查找（Block Search）又称索引顺序查找，是一种介于顺序查找和二分查找之间的折中方案。其基本思想是将查找表分成若干个块（子表），满足"块间有序，块内无序"：第 i 块的最大关键字小于第 i+1 块的最小关键字，但块内各元素可以是无序的（或有序）。为每个块建立一个索引项，包含该块的最大关键字和块首地址（指针）。查找时，先在索引表中折半查找（或顺序查找）确定目标 key 属于哪个块，然后在块内顺序查找目标元素。分块查找的平均时间复杂度约为 O(sqrt(n))（最优分块大小取 sqrt(n)），比顺序查找快但比二分查找慢。其最大优势是可以在动态变化（频繁插入删除）的数据上工作，因为插入新元素时只需在块内移动，不影响其他块。

## 关键结论

- 效率介于顺序查找 O(n) 和二分查找 O(log n) 之间，约 O(sqrt(n))
- 需要维护索引表，索引表记录每块的最大关键字和起始地址
- 块间有序保证索引可二分；块内可无序方便插入（直接放块尾）
- 分块数量 b = sqrt(n) 时理论最优，ASL 约 sqrt(n) + 1
- 适合"分块有序"的动态查找表，是线性结构与索引结构的过渡

## 易错点

1. 块间必须有序：这是使用二分查找索引的前提；如果块间无序，索引表也必须顺序查找
2. 最优分块大小：理论计算证明当块数 b = sqrt(n)（即每块约 sqrt(n) 个元素）时 ASL 最小；但实际中可能根据二级存储特征（如磁盘块大小）调整
3. 插入只能在块内进行：新元素插入到对应块的末尾（或内部有序插入），若块满了可能需要分裂块

## 例题

**例1：** 长度为 10000 的有序表，分块查找时每块 100 个元素，索引表用顺序查找。求 ASL。

**解答：** 索引表有 100 块，顺序查找索引 ASL_idx = (100+1)/2 = 50.5；块内查找 ASL_block = (100+1)/2 = 50.5。总 ASL = 50.5 + 50.5 = 101。若索引用二分查找：ASL_idx = log2(101) ≈ 7，总 ASL = 7 + 50.5 = 57.5。

## 代码示例

```cpp
#include <vector>
using namespace std;

struct IndexItem {
    int maxKey;  // 该块的最大关键字
    int start;   // 该块的起始位置
};

int blockSearch(vector<int>& data, vector<IndexItem>& index, int key) {
    // 1. 在索引表中二分查找确定所属块
    int low = 0, high = index.size() - 1;
    int blockIdx = -1;
    while (low <= high) {
        int mid = low + (high - low) / 2;
        if (key <= index[mid].maxKey) {
            blockIdx = mid;
            high = mid - 1;
        } else {
            low = mid + 1;
        }
    }
    if (blockIdx == -1) return -1;
    
    // 2. 在块内顺序查找
    int start = index[blockIdx].start;
    int end = (blockIdx == index.size() - 1) ? (int)data.size() : index[blockIdx + 1].start;
    for (int i = start; i < end; i++) {
        if (data[i] == key) return i;
    }
    return -1;
}
```

```java
public class BlockSearch {
    static class IndexItem { int maxKey, start; }
    
    public static int blockSearch(int[] data, IndexItem[] index, int key) {
        int low = 0, high = index.length - 1, blockIdx = -1;
        while (low <= high) {
            int mid = low + (high - low) / 2;
            if (key <= index[mid].maxKey) { blockIdx = mid; high = mid - 1; }
            else { low = mid + 1; }
        }
        if (blockIdx == -1) return -1;
        int start = index[blockIdx].start;
        int end = blockIdx == index.length - 1 ? data.length : index[blockIdx + 1].start;
        for (int i = start; i < end; i++)
            if (data[i] == key) return i;
        return -1;
    }
}
```

## 核心定义

链队列（Linked Queue）是采用链式存储结构实现的队列。链队列通常用一个带头结点的单链表实现，设置两个指针：front 指向头结点（固定不动），rear 指向队尾结点。入队（enqueue）操作在 rear 端进行——新建结点连接到 rear->next，然后 rear 后移；出队（dequeue）操作在 front 端进行——取出 front->next 的数据，然后删除该结点。当删除的是最后一个结点时，需要将 rear 重新指向头结点。与顺序队列相比，链队列不存在假溢出，可以无限扩展（受限于内存），但每个结点有额外的指针开销。

## 关键结论

- 链队列的入队和出队时间复杂度均为 O(1)
- 不存在队列满的问题（除非内存耗尽），无假溢出
- 空队列的条件：front == rear（都指向头结点）
- 删除最后一个结点后需要把 rear 重新指向头结点
- 通常使用带头结点的方式，使空队列和非空队列的操作统一

## 易错点

1. 出队后忘记更新 rear：当队列只有一个元素时，出队后 rear 仍然指向已被释放的结点。因此出队时需要判断：若删除后 front->next == NULL，则将 rear 指向头结点（即 rear = front）
2. 区分带头结点与不带头结点的实现：不带头结点时空队列 front == NULL，入队第一个元素时需要特殊处理（同时修改 front 和 rear）
3. front 作为队头指针但不出入队：front 通常指向头结点（固定不变），真正的队头元素是 front->next

## 例题

**例1：** 链队列的出队操作在什么情况下需要同时修改 rear 指针？

**解答：** 当原队列中只有一个元素时，出队后队列为空，此时需要将 rear 重新指向头结点（即 rear = front），否则 rear 成为悬空指针。

## 代码示例

```cpp
template<typename T>
struct QNode {
    T data;
    QNode* next;
    QNode(T val) : data(val), next(nullptr) {}
};

template<typename T>
class LinkedQueue {
private:
    QNode<T>* front;  // 头结点
    QNode<T>* rear;   // 队尾指针
public:
    LinkedQueue() {
        front = new QNode<T>(T());
        rear = front;
    }
    
    ~LinkedQueue() {
        while (front) {
            QNode<T>* p = front;
            front = front->next;
            delete p;
        }
    }
    
    bool isEmpty() { return front == rear; }
    
    void enqueue(T e) {
        QNode<T>* s = new QNode<T>(e);
        rear->next = s;
        rear = s;
    }
    
    bool dequeue(T& e) {
        if (isEmpty()) return false;
        QNode<T>* p = front->next;
        e = p->data;
        front->next = p->next;
        if (rear == p) rear = front;  // 最后一个元素
        delete p;
        return true;
    }
};
```

```java
public class LinkedQueue<T> {
    class Node {
        T data; Node next;
        Node(T val) { data = val; }
    }
    private Node front, rear;
    
    public LinkedQueue() {
        front = new Node(null);
        rear = front;
    }
    
    public boolean isEmpty() { return front == rear; }
    
    public void enqueue(T e) {
        Node s = new Node(e);
        rear.next = s;
        rear = s;
    }
    
    public T dequeue() {
        if (isEmpty()) return null;
        Node p = front.next;
        T val = p.data;
        front.next = p.next;
        if (rear == p) rear = front;
        return val;
    }
}
```

## 核心定义

顺序栈（Sequential Stack）是利用顺序存储结构（数组）实现的栈。栈（Stack）是一种操作受限的线性表，只允许在表的一端（栈顶）进行插入和删除操作，遵循后进先出（LIFO, Last In First Out）的原则。顺序栈通常用数组 data[MAXSIZE] 存储元素，用整型变量 top 指示栈顶位置。top 的取值有两种约定：(1) top 指向栈顶元素位置，初始值为 -1 表示空栈；(2) top 指向栈顶元素的下一个位置，初始值为 0 表示空栈。压栈（push）操作将元素放入栈顶并更新 top；出栈（pop）操作取出栈顶元素并更新 top。顺序栈的所有基本操作时间复杂度均为 O(1)。

## 关键结论

- 顺序栈的 push 和 pop 操作均为 O(1)，因为只操作栈顶
- 栈满条件：top == MAXSIZE - 1（top 指向栈顶元素时）
- 栈空条件：top == -1（top 指向栈顶元素时）
- 顺序栈存在空间溢出的问题：上溢（栈满还 push）和下溢（栈空还 pop）
- 多栈共享空间时可以用两头向中间生长的策略提高空间利用率

## 易错点

1. top 初始值混淆：当 top 指向栈顶元素时，初始值应为 -1；当 top 指向下一个空位时，初始值应为 0；两种约定下压栈和出栈的代码完全不一样
2. 入栈先判满、出栈先判空：这是顺序栈最基本的异常检查，很多同学直接操作数组而不做检查会导致越界
3. 出栈操作需要先取值再自减：如 `e = data[top--];` 而非 `e = data[--top];`（当 top 指向栈顶时）

## 例题

**例1：** 设栈 S 的初始状态为空，元素 a, b, c, d, e 依次入栈，以下哪个出栈序列是不可能得到的？A. a,b,c,d,e  B. e,d,c,b,a  C. c,b,a,d,e  D. c,a,b,d,e

**解答：** D 不可能。当 c 出栈时，栈内为 [a,b]，此时只能出 b 而不能出 a（因为 b 在 a 上面）。验证方法：出栈序列中任意元素后面比它先入栈的元素必须逆序排列。

## 代码示例

```cpp
#include <iostream>
using namespace std;
#define MAXSIZE 100

class SeqStack {
private:
    int data[MAXSIZE];
    int top;  // top 指向栈顶元素位置
public:
    SeqStack() { top = -1; }
    
    bool push(int e) {
        if (top >= MAXSIZE - 1) return false;  // 栈满
        data[++top] = e;
        return true;
    }
    
    bool pop(int& e) {
        if (top == -1) return false;  // 栈空
        e = data[top--];
        return true;
    }
    
    bool getTop(int& e) {
        if (top == -1) return false;
        e = data[top];
        return true;
    }
    
    bool isEmpty() { return top == -1; }
};
```

```java
public class SeqStack {
    private int[] data;
    private int top;
    private static final int MAXSIZE = 100;
    
    public SeqStack() {
        data = new int[MAXSIZE];
        top = -1;
    }
    
    public boolean push(int e) {
        if (top >= MAXSIZE - 1) return false;
        data[++top] = e;
        return true;
    }
    
    public Integer pop() {
        if (top == -1) return null;
        return data[top--];
    }
}
```

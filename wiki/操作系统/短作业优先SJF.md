## 核心定义

短作业优先（SJF, Shortest Job First / SPN, Shortest Process Next）是一种选择预计执行时间最短的进程优先获得 CPU 的调度算法。SJF 分为两种：(1) 非抢占式 SJF——一旦进程开始执行，必须运行到完成或因 I/O 阻塞；(2) 抢占式 SJF（SRTF, Shortest Remaining Time First）——也叫最短剩余时间优先，如果新到达进程的剩余执行时间比当前运行的进程剩余时间短，则抢占 CPU。SJF 可以证明在减少平均等待时间和平均周转时间方面是最优的（前提是已知进程的执行时间）。但缺点也是明显的：需要预知执行时间（现实中难以获取）、长作业可能饿死（当连续有短作业到达时）、对交互性不友好。

## 关键结论

- SJF 在平均等待时间上是最优的（数学可证明）
- 非抢占 SJF 和抢占 SJF（SRTF）两种形式
- 关键问题：如何预测执行时间？可用指数平均法（如：τ_{n+1} = α*t_n + (1-α)*τ_n）
- 长作业可能饥饿（starve），因为持续有短作业到达
- SJF 和 SRTF 通常作为理论最优参照，实际工作中很少直接使用

## 易错点

1. "最短"不是指总的作业时间，而是指下一个 CPU 区间（CPU burst）的预期长度。在 I/O 密集型中进程可能频繁短间隔使用 CPU，SJF/SPN 评价的是下一个 CPU burst
2. SJF 最优性证明仅适用于所有进程同时到达的情况：如果进程在不同时间到达，SJF 非抢占不一定最优，SRTF（抢占版）才保证最优
3. 预测算法的可靠性：指数平均法 τ_{n+1} = α*t_n + (1-α)*τ_n，历史窗口的大小由系数 α 决定

## 例题

**例1：** 进程 P1(t=6), P2(t=8), P3(t=7), P4(t=3) 同时到达，用非抢占 SJF 调度，求平均等待时间。

**解答：** 排序按执行时间：P4(3) -> P1(6) -> P3(7) -> P2(8)。等待时间：P4=0, P1=3, P3=9, P2=16。平均 = (0+3+9+16)/4 = 7。

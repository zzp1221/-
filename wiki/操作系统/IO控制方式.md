## 核心定义

I/O 控制方式是指 CPU 如何与 I/O 设备进行数据传输和协调。其发展经历了四个阶段：(1) 程序直接控制 I/O（Programmed I/O）——CPU 不断轮询设备状态寄存器（"忙等/Busy Waiting"），当设备就绪时进行数据传送。CPU 被完全占用，利用率极低。仅用于极简单的嵌入式系统。(2) 中断驱动 I/O（Interrupt-Driven I/O）——CPU 发出 I/O 命令后继续执行其他任务；设备完成 I/O 后通过中断通知 CPU，CPU 响应中断处理后续数据。CPU 与 I/O 可以并行，但每次中断只能传输少量数据（如一个字），频繁中断增加了上下文切换开销。适合低速率设备（键盘、鼠标）。(3) 直接存储器访问（DMA, Direct Memory Access）——DMA 控制器负责在设备和主存之间直接传输数据块，传输完成后向 CPU 发送一次中断。CPU 只需在传输开始时设置 DMA 参数，其余时间可以并行做其他工作。DMA 大幅解放了 CPU，适合高速块设备（磁盘、网卡）。(4) I/O 通道（I/O Channel）——一种特殊的处理器，执行自己的通道程序来管理 I/O 操作。通道能够控制多台设备，可与 CPU 完全并行工作。大型机中常见。

## 关键结论

- 四种 I/O 控制方式：程序轮询 -> 中断驱动 -> DMA -> I/O 通道
- 程序轮询 CPU 全占用；中断驱动 CPU 部分参与；DMA CPU 基本解放
- DMA 是现代 PC 中磁盘、网卡、显卡的主要传输方式
- I/O 通道主要存在于大型机（IBM zSeries 等）
- 关键衡量指标：数据传输率、CPU 利用率、实时性

## 易错点

1. DMA 和中断不是互斥的——DMA 最后一步也是用中断通知 CPU 传输完成；它们的关系是：中断方式每次传输一字就中断一次，DMA 方式传输完一块数据（多字节）才中断一次
2. DMA 与 CPU 访问主存的总线冲突需要仲裁——通常采用"周期窃取"（DMA 窃取总线周期）或物理上分离的内存总线
3. 通道 ≠ DMA：通道有自己的指令集和程序（通道程序），能独立处理复杂的 I/O 任务序列；DMA 只是简单的块传输控制器

## 例题

**例1：** 比较中断驱动 I/O 和 DMA 在传输大块数据时的效率差异。

**解答：** 中断驱动每传输一个字/字节产生一次中断，需要 CPU 保存现场、执行中断服务程序、恢复现场——传输 1MB 数据可能产生 100 万次中断。DMA 方式仅需在传输前后各一次中断，CPU 参与极少，传输 1MB 数据只需 2 次中断，效率高出几个数量级。

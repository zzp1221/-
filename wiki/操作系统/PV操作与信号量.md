## 核心定义

信号量（Semaphore）是由荷兰计算机科学家 Dijkstra 于 1965 年提出的第一个成功的进程同步工具。它是一个由一个整型变量和一个等待队列组成的内核对象。对信号量只能执行两种原子操作——P 操作（Proberen，荷兰语"测试"，等价于 wait）和 V 操作（Verhogen，荷兰语"增加"，等价于 signal）。P 操作定义：将信号量值 s 减 1，若结果 >= 0 则进程继续执行；若结果 < 0 则该进程阻塞并插入等待队列。V 操作定义：将 s 加 1，若结果 <= 0 则从等待队列唤醒一个进程。信号量分为两种：(1) 整型信号量——仅有计数器，不存在等待队列（忙等，已淘汰）；(2) 记录型信号量——计数器 + 阻塞/唤醒机制（让权等待，实用）。

信号量既可以实现互斥（初始值为 1 的二元信号量），也可以实现同步（控制进程执行顺序）。

## 关键结论

- P(S)：若 S > 0 则 S--；若 S <= 0 则阻塞。PV 成对配对使用
- V(S)：S++；若 S <= 0 则唤醒等待队列中的一个进程
- 互斥信号量初值 = 1（二元信号量）；同步信号量初值 = 0（或 N，取决于资源数）
- P、V 操作必须成对出现但不一定在同一函数中
- 信号量是阻塞式同步原语（让权等待），优于基于忙等的方案

## 易错点

1. P、V 操作顺序至关重要：互斥的 P 操作通常在同步的 P 操作之后（先申请资源再申请互斥锁），V 操作顺序相反（先释放互斥锁再释放资源），否则可能导致死锁
2. 信号量初值设置错误：互斥信号量初值为 1 而非 0；同步信号量初值根据等待条件而定（如表示"缓冲区空位数"则初值为 N）
3. P 操作的判断条件是"<0"而非"==0"：s 从 >0 变为 >=0 可继续；s 从 0 变为 -1 阻塞。V 操作判断唤醒条件是"<=0"而非"<0"

## 例题

**例1：** 用信号量实现前驱关系 S1->S2->S3（S2 必须在 S1 之后执行，S3 在 S2 之后）。

**解答：** 设信号量 sem1=0, sem2=0。进程P1: S1; V(sem1); 进程P2: P(sem1); S2; V(sem2); 进程P3: P(sem2); S3。sem 初始为 0 保证了"只有前驱执行了 V，后继才能通过 P"。

## 代码示例

```cpp
#include <semaphore.h>
#include <pthread.h>

// POSIX 信号量实现互斥
sem_t mutex;

void* threadFunc(void* arg) {
    sem_wait(&mutex);   // P 操作
    // 临界区
    sem_post(&mutex);   // V 操作
    return nullptr;
}

int main() {
    sem_init(&mutex, 0, 1);  // 初始化互斥信号量为 1
    // 创建线程...
    sem_destroy(&mutex);
}
```

```java
// Java 中的 Semaphore
Semaphore sem = new Semaphore(1);  // 互斥信号量初始值为1
sem.acquire();     // P 操作
// 临界区
sem.release();     // V 操作
```

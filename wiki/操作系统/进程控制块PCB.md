## 核心定义

进程控制块（Process Control Block, PCB）是操作系统用来描述进程状态、属性和资源占用情况的数据结构。PCB 是进程存在的唯一标志——系统通过 PCB 来感知和管理进程。PCB 通常包含以下四类信息：(1) 进程标识信息——进程 ID（PID）、父进程 ID（PPID）、用户 ID（UID）；(2) 处理机状态信息——通用寄存器、程序计数器（PC）、程序状态字（PSW）、栈指针等，用于上下文切换时保存和恢复现场；(3) 进程调度信息——进程状态（就绪/执行/阻塞）、优先级、等待事件、调度参数等；(4) 进程控制信息——程序和数据的内存地址、资源分配清单、文件描述符表、I/O 设备信息等。在 Linux 中，PCB 用 task_struct 结构体实现，位于头文件 <linux/sched.h> 中，是所有进程信息的总入口。

## 关键结论

- PCB 是进程的唯一标识，系统通过 PCB 感知进程存在
- PCB 包含四类信息：标识、CPU 现场、调度信息、资源控制
- 所有 PCB 组成进程表（链表/数组/树结构）
- Linux 用 task_struct 实现 PCB，内核通过它管理所有进程
- PCB 的创建发生在进程创建时，销毁发生在进程终止时

## 易错点

1. PCB 中保存的不是完整的上下文（general context），而是发生切换时寄存器中需要保存的"现场信息"。上下文切换的开销主要来自 PCB 保存/恢复
2. PCB 存放在内核空间中（不是用户空间），用户程序无法直接访问
3. PID 是唯一的，但 PID 可能被回收重用（非无穷递增）

## 例题

**例1：** 为什么说 PCB 是进程存在的唯一标志？

**解答：** 进程本身是无形的，OS 只能通过 PCB 数据结构来"感知"它的存在。当 PCB 被创建时进程诞生，当 PCB 被销毁时进程消亡。所有有关进程的信息都记录在 PCB 中，调度器遍历 PCB 来选择下一个要运行的进程。

## 核心定义

进程在其生命周期中会在多种状态之间切换。基本的进程三态模型包含：(1) 就绪态（Ready）——进程已获得除 CPU 外的所有必要资源，等待被调度执行；(2) 执行态（Running）——进程正在 CPU 上执行（单核 CPU 同时只能有一个进程处于执行态）；(3) 阻塞态（Blocked/Waiting）——进程因等待某个事件（如 I/O 完成、信号到达）而暂停执行，等事件发生后转为就绪态。状态转换的主要路径：就绪->执行（被调度选中）、执行->就绪（时间片用完或被抢占）、执行->阻塞（等待 I/O 等事件）、阻塞->就绪（I/O 完成等事件发生）。五态模型额外引入了两种状态：(4) 创建态（New）——进程已创建但尚未就绪（正在分配 PCB 和资源）；(5) 终止态（Terminated）——进程已结束，PCB 等待回收。部分系统还有"挂起态"（Suspend）用于内外存交换。

## 关键结论

- 三种基本状态：就绪、执行、阻塞；五种扩展状态增加创建和终止
- 就绪 -> 执行 是进程调度的结果；执行 -> 阻塞 是进程自身主动的行为
- 不能从阻塞态直接变为执行态，必须经过就绪态
- 不能从就绪态直接变为阻塞态
- 挂起（Suspend）将进程从内存交换到外存

## 易错点

1. "就绪 -> 阻塞"和"阻塞 -> 执行"都是非法转换：阻塞必须回到就绪再被调度为执行
2. 执行态转换为就绪态是被动的（时间片到 / 被抢占），执行态转换为阻塞态是主动的（申请 I/O 等资源不足）
3. 挂起态不等于阻塞态：挂起是被动移出内存到外存，阻塞是主动等待事件；两者结合产生"挂起就绪"和"挂起阻塞"

## 例题

**例1：** 一个进程在执行过程中发出了 I/O 请求，描述其状态变化过程。

**解答：** 执行态 -> 阻塞态（发出 I/O 请求后等待设备完成 I/O）-> 就绪态（I/O 完成，中断通知 OS）-> 执行态（被调度程序选中再次运行）。

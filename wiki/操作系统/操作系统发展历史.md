## 核心定义

操作系统的发展经历了多个阶段，每一步都与计算机硬件技术的发展密切相关：(1) 无操作系统阶段（1940s）——人工操作方式，用户独占计算机，效率极低。(2) 单道批处理系统（1950s）——引入监控程序，作业成批提交，减少人工干预，但 CPU 与 I/O 串行。(3) 多道批处理系统（1960s）——内存中同时驻留多道程序，当一道程序等待 I/O 时 CPU 切换到另一道程序，CPU 与 I/O 并行工作，显著提高资源利用率。中断技术和通道技术是实现多道程序的基础。(4) 分时系统（1960s-1970s）——UNIX 的诞生标志着分时系统的成熟，采用时间片轮转将 CPU 时间分片分配给各用户，提供交互式使用体验。(5) 实时系统（1970s 至今）——要求在规定时间内完成处理和响应，用于工业控制、航空航天等。(6) 现代操作系统——多模融合（同时支持批处理、分时、实时），并向着网络化、分布式、嵌入式方向发展。

## 关键结论

- 第五代之前：手工 -> 单道批处理 -> 多道批处理 -> 分时系统 -> 实时系统
- 多道程序设计是操作系统发展的重要里程碑，实现了并发执行
- 中断和通道技术的引入是实现多道程序设计的关键硬件支撑
- 分时系统的关键技术是时间片轮转调度
- 现代操作系统通常融合多种类型的功能特征

## 易错点

1. 多道批处理与分时系统的区别：多道批处理目标是提高资源利用率和吞吐量，无交互能力；分时系统目标是让多个用户实时交互使用
2. 实时系统 ≠ 分时系统：实时系统强调在 deadline 前完成，可靠性要求极高；分时系统强调公平分配 CPU 给各个终端
3. 多道程序设计不等于多任务：多道是让多个程序在内存中，多任务是用户层面的概念（一个用户同时运行多个程序）

## 例题

**例1：** 推动批处理系统发展为分时系统的主要动力是什么？

**解答：** 交互性需求。批处理系统中用户无法干预程序运行（"提交-等待-取结果"模式），分时系统使多个用户可以通过终端实时交互，极大地改善了用户体验和开发效率。

## 核心定义

抖动（Thrashing）是指系统中频繁出现缺页中断，导致 CPU 把大量时间花在页面置换（换入换出）上，而不是执行实际的任务，使系统吞吐量急剧下降的现象。抖动的根本原因是分配给进程的物理页框数小于进程当前实际需要的最小页数。Denning 于 1968年提出工作集模型（Working Set Model）来预防抖动。工作集 W(t, delta) 定义为在时间窗口 [t-delta, t] 内进程所访问过的页面的集合。工作集反映了进程在当前阶段的内存需求。如果所有进程的工作集之和大于物理内存大小，系统就开始抖动。防止抖动的措施包括：(1) 工作集策略——监控每个进程的工作集，当某进程工作集超过分配页框数时为其增加页框；(2) 缺页频率策略（PFF, Page Fault Frequency）——根据缺页频率动态调整页框分配数量。

## 关键结论

- 抖动 = CPU 花费大部分时间在做页面交换而非实际计算
- 根本原因：分配页框数 < 进程工作集大小
- 工作集 W(t, Δ) = 进程在 [t-Δ, t] 间隔内访问的页面集合
- 若 sum(WS_i) > 总物理内存，系统抖动
- PFF（缺页频率）策略：缺页频率过高 -> 增加页框；缺页频率过少 -> 回收部分页框
- 极端情况下 OS 会采用"挂起某些进程"来释放页框给更需要的进程

## 易错点

1. 抖动不是指 CPU 100% 忙——CPU 看着很忙，但实际上是在做页置换这种"无用功"，有效计算几乎没有
2. 工作集与常驻集的区别：工作集是"需要的页"集合（理论上的），常驻集是"实际分配的页"集合（实际具有的）。常驻集应覆盖工作集
3. δ 的选择至关重要：δ 太大会使工作集包含已不用的页面，δ 太小不能充分反映需要

## 例题

**例1：** 某系统物理内存有 100 个页框，当前运行 5 个进程，各自工作集为 15、20、25、18、22。会发生抖动吗？

**解答：** 工作集总和 = 15+20+25+18+22 = 100。等于物理内存，临界状态。若有任何进程工作集增大，就会发生抖动（开始频繁缺页换页）。

## 核心定义

生产者-消费者问题（Producer-Consumer Problem）是最经典的进程同步问题之一，也称为有界缓冲问题（Bounded-Buffer Problem）。问题的描述：一组生产者进程不断生成产品并放入一个大小为 N 的有界缓冲池；一组消费者进程从缓冲池中取出产品进行消费。需要确保：(1) 生产者不能在缓冲池满时放入产品（缓冲区满则阻塞生产者）；(2) 消费者不能在缓冲池空时取出产品（缓冲区空则阻塞消费者）；(3) 多个生产者和消费者对缓冲池的访问是互斥的。解决方案使用三个信号量：互斥信号量 mutex（初值 1），满缓冲区数量信号量 full（初值 0），空缓冲区数量信号量 empty（初值 N）。P 操作顺序：生产者先 P(empty) 再 P(mutex)；消费者先 P(full) 再 P(mutex)。V 操作顺序相反。

## 关键结论

- 三个信号量：mutex=1（互斥）、full=0（满缓冲区数）、empty=N（空缓冲区数）
- 生产者必须先 P(empty)后 P(mutex)，顺序颠倒会导致死锁（先占锁再等空位）
- 消费者必须先 P(full)后 P(mutex)，同理
- V 操作顺序与 P 相反：先 V(mutex) 再 V(full/empty)，但先 V 哪个信号量不影响正确性（只要在释放 mutex 之后）
- Java 中使用 wait/notify 和 synchronized 同样可以解决

## 易错点

1. P 操作的顺序死锁陷阱：若生产者先 P(mutex) 再 P(empty)，当缓冲区满时生产者会持 mutex 等待 empty，而消费者需要 mutex 才能取产品释放空间，形成死锁
2. full 和 empty 的信号量值等价于"可用资源数"，而非单纯标记——这是信号量比互斥锁更强大的地方
3. 多生产者和多消费者下仍需 mutex 保证对缓冲数组的互斥访问

## 例题

**例1：** 现有 1 个生产者、1 个消费者、缓冲区大小为 5。生产者每生产一个产品需要 2s，消费者每消费需要 3s。问系统能否稳定运行？若多个生产者同时写相同位置会怎样？

**解答：** 单生产者单消费者资源流稳定（生产者快于消费者，缓冲区会满然后生产者阻塞）。若多生产者无互斥，可能出现两个生产者同时写同一位置导致数据覆盖丢失，因此必须用 mutex 保护。

## 代码示例

```cpp
#include <semaphore.h>
#include <pthread.h>
#define N 10  // 缓冲区大小

int buffer[N];
int in = 0, out = 0;
sem_t mutex, empty, full;

void* producer(void* arg) {
    while (true) {
        int item = produceItem();
        sem_wait(&empty);   // 等待空位
        sem_wait(&mutex);   // 进入临界区
        buffer[in] = item;
        in = (in + 1) % N;
        sem_post(&mutex);   // 退出临界区
        sem_post(&full);    // 增加满缓冲区计数
    }
}

void* consumer(void* arg) {
    while (true) {
        sem_wait(&full);    // 等待有产品
        sem_wait(&mutex);   // 进入临界区
        int item = buffer[out];
        out = (out + 1) % N;
        sem_post(&mutex);   // 退出临界区
        sem_post(&empty);   // 增加空位计数
        consumeItem(item);
    }
}
```

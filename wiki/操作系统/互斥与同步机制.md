## 核心定义

互斥（Mutual Exclusion）是指一组并发进程中只有一个进程能在给定时间点进入临界区执行，防止竞争条件（Race Condition）。同步（Synchronization）是更广泛的概念，指协调多个并发进程/线程的执行顺序以达到协作目的。实现互斥和同步的机制可以分为三类：(1) 软件方法——如 Dekker 算法、Peterson 算法，纯软件实现不需要硬件支持，但实现复杂且通常有忙等的缺点；(2) 硬件方法——包括关中断（单核有效，多核不可行）、TSL（Test and Set Lock，原子地测试并设置锁）、CAS（Compare and Swap）等，硬件级原子指令是实现高级同步原语的基石；(3) OS 同步工具——包括互斥锁（Mutex）、信号量（Semaphore）、管程（Monitor）、条件变量（Condition Variable）、读写锁等，是操作系统提供给用户程序的高级抽象。现代编程实践中多使用 OS 层面的互斥锁和信号量。

## 关键结论

- 互斥解决"不能同时访问"问题，同步解决"按序协作"问题
- TSL 指令是原子的硬件原语：test_and_set(lock) 返回旧值并设置新值
- 自旋锁（Spinlock）基于 TSL 实现，忙等但适合临界区极短的场景
- 互斥锁/信号量是 OS 级抽象，等锁时主动放弃 CPU（让权等待）
- CAS 指令是实现无锁数据结构的核心硬件原语

## 易错点

1. 自旋锁（Spinlock）与互斥锁（Mutex）的使用场景不同：Spinlock 忙等适合极短临界区（小于上下文切换开销的时间）；普通 Mutex 适合可能等待较长时间的锁
2. TSL 本身不能保证有限等待和前进条件：TSL 只保证原子性，竞争激烈的自旋锁会出现饥饿
3. 关中断只对单 CPU 有效：多核系统中，关一个核的中断不影响其他核的并发访问

## 例题

**例1：** 为什么 TS(Tset and Set) 指令能够实现互斥？其核心是什么？

**解答：** TS 指令的核心是原子性（Atomicity）：它读取锁变量并在同一总线周期内设置锁为 1，整个操作不可被中断。如果 TS 返回 0 说明锁是空闲的且已经被当前进程获取；如果返回 1 说明锁已被占用。由于原子性，不可能有两个进程同时获取到锁。

## 代码示例

```cpp
#include <atomic>
using namespace std;

// 基于 atomic_flag 的自旋锁实现
class SpinLock {
    atomic_flag flag = ATOMIC_FLAG_INIT;
public:
    void lock() {
        while (flag.test_and_set(memory_order_acquire)) {
            // 自旋等待（忙等）
        }
    }
    void unlock() {
        flag.clear(memory_order_release);
    }
};
```

## 核心定义

处理机调度（CPU Scheduling）是多道程序系统的基础，负责选择就绪进程/线程分配 CPU 执行。调度分为三个层次：(1) 长程调度（Long-Term Scheduling / Job Scheduler）——控制多道程序的"道数"，决定哪些作业从外存（后备队列）调入内存变为进程。频率很低（分钟级或更长）。只在批处理系统中存在，分时和实时系统中一般无此调度。(2) 中程调度（Medium-Term Scheduling）——即挂起和激活操作，决定哪些进程被临时换出内存到外存或重新换回内存。用于内存不足时腾出空间。频率中等。(3) 短程调度（Short-Term Scheduling / CPU Scheduler）——决定就绪队列中哪个进程获得 CPU 执行。频率极高（毫秒级），是调度系统中最核心的部分。

调度算法的评价指标包括：CPU 利用率、吞吐量（单位时间完成作业数）、周转时间（提交到完成）、等待时间（就绪队列中等待总时间）、响应时间（提交到首次响应）。

## 关键结论

- 三级调度：长程（调入内存）、中程（挂起/激活）、短程（分配 CPU）
- 短程调度频率最高，是真正意义上的"CPU 调度"
- 调度指标平衡：响应时间 vs 吞吐量（分时系统重响应，批处理重吞吐）
- 抢占式（Preemptive）调度可随时打断正在运行的进程；非抢占式（Non-preemptive）只能等进程主动放弃 CPU
- I/O 密集型 vs CPU 密集型进程：调度需要兼顾两类进程

## 易错点

1. 周转时间 != 执行时间：周转时间 = 完成时刻 - 提交时刻，包含等待时间、执行时间和 I/O 时间
2. 长程调度 ≠ 批处理系统的"作业调度"：实际上长程调度主要负责控制多道度（内存中进程数量），分时系统虽然不需要从后备队列选，但仍有创建进程的限制
3. 响应时间 vs 周转时间：响应时间是从提交到首次获得 CPU 响应，周转时间是从提交到完成。交互式系统关注响应时间，批处理关注周转时间

## 例题

**例1：** 分时操作系统中有长程调度吗？为什么？

**解答：** 分时操作系统中通常没有长程调度。因为分时系统面向交互式用户，用户通过终端直接登录，OS 即时创建进程而不是从后备队列中选择。进程数量由用户连接数自然控制。

## 核心定义

最佳页面置换算法（OPT, Optimal Page Replacement）是由 Belady 于 1966 年提出的理论最优算法。当发生缺页中断且物理内存满时，OPT 选择"未来最长时间内不会被使用"的页面进行淘汰。因为可知未来的引用序列，OPT 的缺页率是所有算法中最低的。然而 OPT 在实践中不可实现（无法预知程序未来的访问模式），它主要用作衡量其他页面置换算法的"理想参照"——其他算法性能与 OPT 越接近则越优秀。OPT 的计算方式是：对于内存中的每个页面，扫描未来的引用串，找到下次被访问的最远距离（或永不访问），淘汰该页面。OPT 的缺页次数是任何页面置换算法所能达到的理论下限。

## 关键结论

- OPT 是理论上最优的页面置换算法，在实践中不可实现
- 决策依据：淘汰未来最长时间不使用的页面
- 用途：作为基准（Benchmark）来评价其他置换算法（FIFO、LRU等）的性能
- OPT 不存在 Belady 异常——增加物理块数不会反而增加缺页率
- OPT 是栈算法（Stack Algorithm），分配帧越多缺页率越低

## 易错点

1. OPT 不可实现不是因为它有缺陷，而是因为"预知未来"需要先知——实际 OS 无法预知程序的访问模式
2. OPT 不存在 Belady 异常（FIFO 有）——因为 OPT 满足"包容性"（包含属性），增加页框数不会导致更多的页面置换
3. 计算 OPT 时，"永不使用"的页面优先级最高被淘汰（未来距离为无穷大）。若多个永不使用的页面，任选一个淘汰

## 例题

**例1：** 系统分配 3 个页框，引用串为 7 0 1 2 0 3 0 4 2 3 0 3 2 1 2 0 1 7 0 1。用 OPT 计算缺页次数。

**解答：** 分配3框。页面走向：7(M), 0(M), 1(M), 2(M 淘汰7, 因7未来最远?), ...算了在考试中需要画表。OPT 缺页次数为 9，缺页率 9/20 = 45%。

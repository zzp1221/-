## 核心定义

I/O系统（Input/Output System）是计算机与外部世界交换信息的子系统，负责在CPU主存和外围设备（键盘、鼠标、显示器、磁盘、网络等）之间传输数据。I/O系统由三大部分组成：I/O设备（外设本身）、I/O接口（设备控制器，连接设备与总线的硬件电路）、I/O软件（设备驱动程序和操作系统I/O子系统）。CPU访问I/O设备有两种编址方式：统一编址（Memory-Mapped I/O, MMIO）——I/O设备寄存器和主存共享统一地址空间，用 Load/Store 指令访问设备（ARM、RISC-V 采用）；独立编址（Port-Mapped I/O, PMIO）——I/O端口拥有独立地址空间，用专用 IN/OUT 指令访问设备（x86 采用）。I/O控制的四种方式按CPU参与度递减排列：程序查询（Polling）——CPU反复检查设备就绪状态；中断驱动（Interrupt-Driven）——设备就绪时发中断通知CPU；DMA方式——DMAC 控制数据块传送；通道方式（I/O Channel）——专门的I/O处理器执行通道程序管理多个设备。

## 关键结论

- I/O总线的性能指标：带宽（吞吐量 MB/s）、延迟（响应时间）、可扩展性（支持设备数）
- MMIO 的优势：简化指令集（无需专门 I/O 指令），统一编程模型
- I/O接口的功能：数据缓冲（匹配速度差异）、格式转换（串并转换）、电气匹配、协议转换、中断管理
- 现代 I/O 层次架构：设备 -> 设备控制器 -> 总线 -> 桥接器 -> 系统总线/PCIe -> 内存控制器

## 易错点

1. MMIO 和 PMIO 的地址空间区别：MMIO 消耗了主存地址空间的一部分区域用于映射设备寄存器，不减少物理RAM容量（映射到了地址空间而非RAM物理存储）。
2. I/O 接口不是设备本身：I/O 接口（串口芯片、USB 控制器等）是主板/芯片上的硬件，外设是插接到接口上的物理设备。
3. I/O 操作的数据可能绕开 Cache（非缓存 I/O）：部分 I/O 区域被标记为不可缓存，以保证每次访问都直达设备而非读取过时的 Cache 副本。

## 例题

**例题1：** 比较程序查询和中断驱动在键盘输入场景的适用性。

**解答：** 键盘输入速率通常为每秒几个到几十个字符（非瓶颈）。若 CPU 全时轮询键盘状态，将浪费 99%+ 的计算能力。中断驱动方式下，键盘仅在用户按键时产生中断，CPU 平时可做其他工作，适合键盘。但程序查询适合高速设备或已知延迟极小的场景（如简单的嵌入式传感器）。

**例题2：** 解释 x86 中 IN/OUT 指令的 I/O 编址机制。

**解答：** x86 支持独立 I/O 地址空间（64KB 端口空间，使用 16 位地址）。IN AL, DX 从 DX 指定的 I/O 端口读取 1 字节到 AL；OUT DX, AL 将 AL 1 字节写入 I/O 端口。CPU 通过 M/IO# 信号线区分访问的是内存还是 I/O 空间，内存控制器或 I/O 桥根据此信号路由到正确的目标。

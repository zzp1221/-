## 核心定义

Cache 映射方式决定主存中的某个数据块可以放置在 Cache 中的哪些位置，影响命中率和硬件复杂度。三种基本映射方式：直接映射（Direct Mapped Cache）——每个主存块只能映射到 Cache 中唯一固定的位置（Block_Index = 主存块号 mod Cache块数），硬件最简单，Tag 比较只需一次，但冲突缺失率最高；全相联映射（Fully Associative Cache）——任何主存块可以放入 Cache 的任何位置，冲突缺失率最低（理论上只有容量缺失），但需要并行 Tag 比较所有 Cache 行（需要大量硬件比较器），成本高昂，仅适合小容量 Cache（如 TLB）；组相联映射（Set-Associative Cache）——将 Cache 分为多个组（Set），每组包含 n 个行（n 路），主存块首先映射到固定的组（组号 = 主存块号 mod 组数），然后可以放入组内的任意一行，是直接映射和全相联的折中方案。现代 CPU 的 L1 Cache 通常为 4 路或 8 路组相联，L2/L3 常为 8-16 路组相联。

## 关键结论

- 直接映射硬件最简单但冲突缺失多，全相联映射冲突缺失最少但硬件最复杂，组相联是工业实践中的甜点
- 路的数量（相联度）增加减少冲突缺失但增加命中时间（Tag 比较器增多、MUX 复杂度增加）
- 地址划分三字段：Tag（标记）、Index（组索引/行索引）、Offset（块内偏移）。Index 指向组（组相联）或行（直接映射）
- 全相联映射中 Index=0（所有位都是 Tag），没有"行号"概念，需对全表并行搜索
- Cache 总容量 = 行数 * (Tag 位数 + 数据块字节数 + 有效位 + Dirty位) / 8 字节

## 易错点

1. 组相联中的"路"（Way）概念：n 路组相联表示每组有 n 个行槽位（可存放 n 个不同的主存块）。"4 路组相联"不是"4 个组"的意思，而是"每组 4 路"。
2. 直接映射和 1 路组相联是等价的：直接映射 = 1 路组相联。
3. Tag 比较器数量 = n * 组数（对于 n 路组相联），不是等于 n（因为每组可独立并行比较）。

## 例题

**例题1：** 某 Cache 容量 8KB，行大小 32 字节，2 路组相联。32 位地址下求 Tag/Index/Offset 位数。

**解答：** Offset = log2(32) = 5 bit。总行数 = 8KB/32B = 256，组数 = 256/2 = 128。Index = log2(128) = 7 bit。Tag = 32-7-5 = 20 bit。

**例题2：** 分析三种映射方式的冲突缺失差异。假设 Cache 4 行，循环访问地址 0, 64, 128, 0, 64, 128, 0, 64, 128（地址映射到同一组/行）。

**解答：** 直接映射（1路4组）：0/64/128 都映射到组0，每次访问都驱逐前一个，命中率 0%。2路组相联（2路2组）：0 和 64 共享组0（两路可共存），128 与它们竞争但最多保留2个，命中部分改善。全相联（4路1组）：0/64/128 共3个，4行够存，第一次循环后（强制缺失后）全部命中。

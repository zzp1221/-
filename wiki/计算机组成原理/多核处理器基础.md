## 核心定义

多核处理器（Multi-Core Processor / Chip Multiprocessor, CMP）是在单一芯片上集成了多个独立的处理器核心（Core）的新型 CPU 架构。由于单核主频受功耗墙（Power Wall）和 ILP 墙的限制无法持续提升，工业界从 2000 年代中期起转向在每个芯片上放置多个较简单的核心，通过线程级并行（Thread-Level Parallelism, TLP）延续性能增长。每个核心拥有自己独立的 PC、寄存器堆、L1 Cache（I 和 D），通常还有私有的 L2 Cache；多个核心共享芯片上的 L3 Cache（Last-Level Cache, LLC）、内存控制器等资源。多核处理器的分类：同构多核（所有核心架构相同，如 Intel Core i7）；异构多核（不同性能/功耗特性的核心混合，如 ARM big.LITTLE、Intel Core 12代 P 核+E 核）。多核带来并行处理的巨大潜力，但也引入了并发编程、Cache 一致性、同步开销、Amdahl 定律限制等新挑战。

## 关键结论

- 多核的性能提升在于 TLP 而非 ILP：多个线程并行运行在不同核上，各自独立执行
- 共享 L3 Cache 减少了多副本内存数据的总存储开销并促进数据共享
- Amdahl 定律约束多核加速比：S = 1 / ((1-f) + f/N)，其中 f 为可并行比、N 为核心数。串行部分成为瓶颈
- 超线程（SMT, Simultaneous Multithreading）：在单核上同时运行多个线程以隐藏延迟、提高利用率
- 多核编程挑战：数据竞争、死锁、负载不均衡、伪共享、一致性开销

## 易错点

1. 多核不等于多 CPU：多核是单芯片多核心；多 CPU 是多颗独立的 CPU 芯片通过主板互联。现代服务器两者兼有。
2. 核心数翻倍不等于性能翻倍：Amdahl 定律和信号/互连开销、内存瓶颈共同限制了实际扩展性。
3. 多核上错误的单线程程序仍然只在单核上运行：没有使用多线程或多进程的程序无法自动利用多核。

## 例题

**例题1：** 程序 60% 可并行化，在 4 核处理器上的理论加速比。

**解答：** Amdahl 定律：S = 1 / ((1-0.6) + 0.6/4) = 1 / (0.4 + 0.15) = 1/0.55 = 1.82。即即使 4 核，加速仅为 1.82 倍。若有 100 核极限加速约为 2.5。

**例题2：** 对称多处理（SMP）与非对称多处理（AMP）的区别。

**解答：** SMP（Symmetric Multi-Processing）中每个核心平等运行同操作系统、同职责分配，由调度器动态分配任务。AMP（Asymmetric）中核心职责不同（如核 0 跑 OS，核 1-3 跑专用 DSP 代码），大型主机常见。现代通用 CPU 均为 SMP。

## 核心定义

在计算机中，有符号整数的表示方法主要有四种编码方式：原码（Sign-Magnitude），最高位为符号位（0正1负），其余位表示数值的绝对值，如 +5(8位)=00000101，-5=10000101；反码（Ones' Complement），正数与原码相同，负数则将原码除符号位外的每位按位取反，如 -5 的反码=11111010；补码（Two's Complement），现代计算机统一采用的标准有符号表示法，正数的补码与其原码相同，负数的补码等于其反码加1，或理解为模 2^n 下的等价表示，如 -5 的补码=11111011；移码（Biased Notation），在补码基础上将符号位取反（即加上一个偏移量 2^{n-1}），常用于浮点数阶码的表示，便于指数的大小比较。补码是现代计算机的标准选择，因为它简化了ALU设计。

## 关键结论

- 补码的五大优势：唯一零表示（无+0/-0）；符号位可直接参与运算；减法可用加法实现（A-B = A+(-B)的补码）；硬件简单；数值范围不对称 (-2^{n-1} 到 2^{n-1}-1)
- 补码的快速求法（手算）：从右向左找到第一个1，该位之后的所有高位取反
- 反码存在的问题：有两个零（+0=00000000, -0=11111111），循环进位（End-Around Carry）
- 8位补码范围：[-128, 127]；16位补码范围：[-32768, 32767]；32位补码范围：[-2147483648, 2147483647]
- 移码（偏置 127 或 1023）在 IEEE754 浮点数中用于阶码，确保非负值且全0表示绝对值最小的数

## 易错点

1. 补码整数溢出时符号位翻转：127+1 在 8 位补码中变为 -128。这是由于模运算的溢出回绕特性。
2. 反码加法需要处理"循环进位"：两数反码相加若产生进位则需将该进位加到最低位（End-Around Carry），增加了硬件复杂度。
3. 移码和补码的关系：移码 = 补码的符号位取反（将符号位从 0->1, 1->0），相当于值 = 补码值 + 2^{n-1}。

## 例题

**例题1：** 用 8 位补码计算 45 + (-23)。

**解答：** 45 补码=00101101；-23 原码=10010111，反码=11101000，补码=11101001。相加：00101101 + 11101001 = (1)00010110。舍去进位得 00010110=22，正确。注意符号位的进位自然舍去，无需特殊处理。

**例题2：** 求 8 位补码中 -128 的表示。

**解答：** 128 = 10000000，-128 的补码 = ~10000000 + 1 = 01111111 + 1 = 10000000。注意这是唯一符号位为 1 且数值位全为 0 的特殊情况（没有对应的正数）。在 8 位补码中，10000000 表示 -128 而非 -0。

## 代码示例

```python
def to_twos_complement(value, bits):
    """将整数转为指定位宽的补码表示（字符串）"""
    if value < 0:
        return bin((1 << bits) + value)[2:].zfill(bits)
    else:
        return bin(value)[2:].zfill(bits)

def from_twos_complement(binary_str):
    """将补码二进制字符串转为十进制"""
    bits = len(binary_str)
    value = int(binary_str, 2)
    if binary_str[0] == '1':  # 负数
        return value - (1 << bits)
    return value

def add_twos_complement(a, b, bits):
    """补码加法（截取指定位宽）"""
    result = (a + b) & ((1 << bits) - 1)
    # 检测溢出：同号相加结果异号
    overflow = ((a ^ result) & (b ^ result) & (1 << (bits - 1))) != 0
    return result, overflow

# 演示
print("45 补码(8位):", to_twos_complement(45, 8))  # 00101101
print("-23 补码(8位):", to_twos_complement(-23, 8))  # 11101001
print("45 + (-23) 补码:", to_twos_complement(45 + (-23), 8))  # 00010110 = 22

# 补码加法和溢出检测
a, b = 100, 50  # 127 是 8位补码最大值，100+50 溢出
result, ov = add_twos_complement(a, b, 8)
print(f"{a}+{b}={result}, 溢出={ov}")  # 150->-106, 溢出=True
```

## 核心定义

中断（Interrupt）是 CPU 暂停当前程序的正常执行流程，转而处理紧急或异步事件（外设请求、硬件故障、程序异常）的一种硬件机制。中断来源于两个方面：外部中断（硬件中断 / Interrupt）——由 CPU 外部设备通过 INTR 和 NMI（不可屏蔽中断）引脚发出，用于 I/O 设备通知数据就绪、定时器到期等；内部中断（异常 / Exception / Trap）——由 CPU 内部在执行指令过程中产生，包括故障（Fault，如缺页、除零）、陷阱（Trap，如系统调用 INT指令）、终止（Abort，严重硬件错误）。传统上 x86 使用中断向量表（IVT），每个中断源对应一个 0-255 的中断向量号（Interrupt Vector Number），CPU 根据向量号在中断描述符表（IDT）中查找对应的处理程序入口地址。现代 CPU 如 ARM 和 RISC-V 使用更精简的异常向量表 + 异常原因的软件查询/硬件编码（如 ARM 的 Vector Table + 异常类型寄存器）。

## 关键结论

- 中断响应过程：保存断点（PC 和 PSW 压栈）、查询中断源（中断应答/INTA周期）、获取中断向量、转入中断服务程序 ISR、执行 ISR、恢复现场、返回断点
- 中断优先级：多个中断同时发生时，CPU 响应优先级高的中断（由中断控制器如 8259A/APIC 管理）
- 中断嵌套：高级中断可以打断正在处理的低级中断（需要中断屏蔽寄存器控制）
- 中断与轮询（Polling）对比：中断在设备低活动率时 CPU 利用率高；轮询在设备高活动率时响应快且开销小
- ARM 的异常模型：6 种异常类型，固定入口地址（向量表 0x00000000 或 0xFFFF0000）

## 易错点

1. 中断和异常的区别：中断是异步的外部事件（I/O 请求），异常是同步的指令执行副产品（除零、缺页）。但统称为"中断"在某些教材中常见（广义中断）。
2. 中断返回地址：中断返回到被中断指令的下一条（中断发生在指令边界），而故障（如缺页）发生在指令执行中，返回地址是该指令本身以重试。
3. 中断服务程序（ISR）必须是可重入的或原子保护的：若高优先级中断触发了相同的 ISR，可能导致数据竞争。

## 例题

**例题1：** 描述 x86 CPU 响应一次外部中断的完整过程。

**解答：** (1) 外设向中断控制器（如 APIC）发中断请求；(2) 中断控制器向 CPU 发 INTR 信号；(3) CPU 执行完当前指令后检查 IF 标志（若 IF=0 则挂起）；(4) CPU 向中断控制器发送 INTA 周期，获取中断向量号 N；(5) 将 EFLAGS, CS, EIP 依次压栈；(6) 从 IDT[N] 加载新 CS 和 EIP；(7) 清除 IF（禁止中断嵌套，除非 ISR 主动开中断）；(8) 执行 ISR；(9) 执行 IRET 指令恢复 EFLAGS, CS, EIP。

**例题2：** 比较中断、陷阱和故障三种内部异常的异同。

**解答：** 陷阱（Trap）：同步、在指令执行后触发（如 INT 3 断点）、返回地址是下一条指令。故障（Fault）：同步、在指令执行中触发（如缺页）、返回地址是当前指令（重试）。终止（Abort）：同步/异步、严重错误（如双故障 Double Fault）、不可恢复，程序无法继续。

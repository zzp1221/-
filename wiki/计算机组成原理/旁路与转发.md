## 核心定义

旁路/转发（Forwarding / Bypassing）是解决流水线中数据冒险（RAW依赖）的关键硬件技术。当一条指令的结果在尚未写回寄存器时就已被后续指令需要时，转发机制通过额外的数据通路（旁路路径，Bypass Paths），将结果数据在产生的同时"转发"到后续指令的输入端，从而避免流水线停顿。在 MIPS 五段流水线中，结果可以在 EX 阶段的输出（EX/MEM 流水线寄存器）、MEM 阶段的输出（MEM/WB 流水线寄存器）被截获，通过多路选择器（MUX）直接送至下一条或下两条指令的 EX 阶段 ALU 输入端。转发检测逻辑（Forwarding Unit）根据流水线寄存器中的目的寄存器号与当前指令的源寄存器号比较，判断是否需要激活转发路径并选择数据源。转发使得大多数数据冒险可以在零停顿下解决，只有 Load-Use 冒险（LW 后紧跟使用该 Load 结果的指令）需要 1 个周期的停顿（因为 Load 数据要等到 MEM 阶段才从存储器读出）。

## 关键结论

- 转发路径：EX/MEM -> EX 输入（1 条指令间隔）、MEM/WB -> EX 输入（2 条指令间隔）
- 转发检测条件：前一条指令的 RegWrite=1 且目标寄存器 Rd != 0 且 Rd == 当前指令的 Rs 或 Rt
- 当多个转发源同时匹配时需设定优先级（通常取较近的 EX/MEM 结果优先于 MEM/WB 结果）
- Load-Use 冒险无法单靠转发解决：Load 数据在 MEM 阶段才可用，而后一条指令在 EX 阶段就需要，必须插入 1 个气泡
- 现代乱序执行 CPU 使用寄存器重命名（Register Renaming）消除 WAR 和 WAW 依赖

## 易错点

1. $0 寄存器（MIPS）或 x0（RISC-V）恒为 0，不能作为转发目标：需在检测逻辑中排出 Rd=0 的情况。
2. 转发优先级错误：若 EX/MEM 和 MEM/WB 都有匹配的目标寄存器，应优先选择 EX/MEM（更新的结果）。
3. Load-Use 冒险不仅需要 1 个停顿周期，还需要在停顿后激活转发路径以获取 Load 的结果。

## 例题

**例题1：** 下列指令序列是否需要转发？需要停顿？
```
LW R1, 0(R2)
ADD R3, R1, R4
SUB R5, R1, R6
```
**解答：** ADD 需要 R1，但 R1 来自 LW，LW 在 MEM 阶段才将数据从内存读到 MEM/WB 寄存器，而 ADD 在 EX 阶段就需要。必须停顿 1 周期（ADD 的 EX 推迟 1 周期），然后从 MEM/WB 转发 R1 到 ADD 的 EX。SUB 在 ADD 后一条，其 ID 与 ADD 的 EX 同时，可从 MEM/WB 获得 R1（此时 LW 已在 WB），或从 ADD 的 EX/MEM 转发（如果 ADD 写 R1 的话），但 ADD 的目标是 R3 不影响 R1。

**例题2：** 设计转发单元的检测逻辑。

**解答：** ForwardA 控制信号：if (EX/MEM.RegWrite and EX/MEM.Rd != 0 and EX/MEM.Rd == ID/EX.Rs) then ForwardA = 10（选自 EX/MEM 结果）。else if (MEM/WB.RegWrite and MEM/WB.Rd != 0 and MEM/WB.Rd == ID/EX.Rs) then ForwardA = 01（选自 MEM/WB 结果）。else ForwardA = 00（选自寄存器堆）。

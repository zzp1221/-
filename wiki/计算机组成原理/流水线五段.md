## 核心定义

经典的五段流水线（5-Stage Pipeline）是讲解指令流水线原理的标准模型，源于 MIPS 处理器的设计。它将一条指令的执行划分为条件均衡的五个阶段：IF（Instruction Fetch，取指）——使用 PC 从指令存储器（或 I-Cache）中取出当前指令，同时计算 PC+4 作为顺序下一条指令地址；ID（Instruction Decode，译码/寄存器读）——解码指令操作码和操作数字段，从寄存器堆中读取源操作数 Rs 和 Rt，对立即数进行符号扩展，同时检测控制/数据冒险；EX（Execute，执行/地址计算）——ALU 执行运算（R-type 的算术逻辑运算）、计算访存有效地址（I-type 的 Load/Store）、计算分支目标地址并判断分支条件；MEM（Memory Access，存储器访问）——对 Load 指令从数据存储器读取数据，对 Store 指令将数据写入数据存储器，其他指令（R-type）本阶段空转；WB（Write Back，写回）——将结果写回寄存器堆（Load 写存储器读出数据，R-type 写 ALU 结果），完成指令的生命周期。流水线寄存器将各阶段隔离，每段的结果在时钟沿打入下一段寄存器。

## 关键结论

- 五段流水的经典分配来自 MIPS：IF 访问 I-Cache，ID 读 RegFile，EX 运算，MEM 访问 D-Cache，WB 写 RegFile
- 各阶段工作量的平衡至关重要：最慢的阶段决定时钟周期长度（通常 EX 或 MEM 阶段最长）
- PC 的更新在 IF 阶段完成，但分支的目标地址要到 EX 阶段才确定（导致控制冒险）
- 寄存器堆的写发生在 WB 阶段（时钟下降沿或下个时钟上升沿），而读发生在 ID 阶段（组合逻辑），有效地支持了同周期读写
- 流水线寄存器保存每阶段产生的所有控制信号和数据，逐级传递至后续阶段

## 易错点

1. 各阶段重叠执行带来的状态混乱：第 N 条指令在 EX，第 N+1 条在 ID，第 N+2 条在 IF——需要流水线寄存器隔离各指令的状态。
2. WB 阶段写寄存器的目标地址需要沿途传递：MIPS 中目标寄存器号从 ID 开始经 EX/MEM 传到 MEM/WB 才用于写回。
3. MEM 阶段可能有 Cache 缺失：导致该阶段无法在一个周期完成，需要暂停整个流水线。

## 例题

**例题1：** 假设 5 段流水各段延迟为：IF=2ns, ID=1.5ns, EX=2.5ns, MEM=2ns, WB=1ns。求最佳时钟周期和完成 1000 条指令的时间。

**解答：** 时钟周期由最慢阶段决定 = 2.5ns。1000 条指令：4 段填充 + 1000 指令执行 = 1004 周期。时间 = 1004 * 2.5ns = 2510ns = 2.51μs。

**例题2：** 跟踪 ADD R1, R2, R3 在五段流水线中的数据流动。

**解答：** IF: PC->IM, IR=ADD指令, PC=PC+4。ID: Rs=R2, Rt=R3 读 RegFile（值送入 ID/EX 寄存器），控制信号生成。EX: ALU 对 R2+R3 计算结果，结果和 Rd 号送入 EX/MEM 寄存器。MEM: 不做操作，结果经 MEM/WB 透传。WB: 结果写回 R1。

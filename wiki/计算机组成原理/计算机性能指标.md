## 核心定义

计算机性能指标是衡量计算机处理能力和效率的量化标准，主要包括两大类：时间类指标和速率类指标。时间类指标：响应时间（Response Time）/ 执行时间（Execution Time）表示完成一个任务所需的总时间，是用户最直观感受到的性能参数——CPU 时间 = 程序总时钟周期数 * 时钟周期长度 = 指令数 * CPI / 主频；吞吐率（Throughput）表示单位时间内完成的任务数量，在服务器、数据中心场景中更为重要。速率类指标：主频（Clock Rate）以 GHz 为单位，表示 CPU 每秒震荡次数，时钟周期 = 1/主频；CPI（Cycles Per Instruction）表示执行每条指令所需的平均时钟周期数；MIPS（Million Instructions Per Second）衡量每秒执行百万条指令数；MFLOPS（Million Floating-point Operations Per Second）侧重浮点性能。加速比（Speedup）定义为改进后性能与改进前性能之比。

## 关键结论

- CPU 执行时间公式（CPU Performance Equation）：T = IC * CPI * T_cycle = IC * CPI / f，优化任何一个因子都会减少执行时间
- CPI 的组成：CPI = 基本CPI + 存储器停顿周期 + 流水线停顿周期
- Amdahl 定律：系统加速比受限于不可加速部分。S = 1 / ((1 - f) + f / s)，f 为可加速比例，s 为加速倍数
- MIPS 作为性能指标的缺陷：不同指令集的 MIPS 不可直接比较；同一机器执行不同程序 MIPS 波动大；具有误导性
- 基准测试套件（SPEC CPU, Geekbench, Linpack）提供标准化评测环境

## 易错点

1. 主频不是性能的绝对指标：由于 CPI 和 IPC 的差异，主频 3GHz 的 CPU A 可能比主频 4GHz 的 CPU B 更快。性能 = 频率 / CPI，不能只看频率。
2. MIPS 的反比关系：在某些系统中，更高的 MIPS 反而可能对应更低性能，这称为 MIPS 悖论（MIPS paradox），因为不同指令集每指令完成的工作量不同。
3. 吞吐率和响应时间是不同维度：提高吞吐率（流水线）有时会增加个体响应时间（延迟）；降低延迟（更快响应）也可能降低吞吐率。

## 例题

**例题1：** 某程序含 10^9 条指令，CPI=1.2，主频 2.5GHz。求执行时间。

**解答：** T = IC * CPI / f = 10^9 * 1.2 / (2.5 * 10^9) = 1.2 * 10^9 / 2.5 * 10^9 = 0.48 秒。

**例题2：** 某优化将某程序的 80% 部分加速 4 倍，求整体加速比。

**解答：** Amdahl 定律：S = 1 / ((1-0.8) + 0.8/4) = 1 / (0.2 + 0.2) = 1/0.4 = 2.5。即整体仅加速了 2.5 倍，而非直观的 4 倍。未被优化的 20% 部分成为瓶颈。

## 代码示例

```python
def cpu_execution_time(instructions, cpi, freq_ghz):
    """
    instructions: 指令数
    cpi: 每指令时钟周期数
    freq_ghz: 主频 (GHz)
    """
    t_cycle = 1 / (freq_ghz * 1e9)
    clock_cycles = instructions * cpi
    time_sec = clock_cycles * t_cycle
    return time_sec

def amdahl_law(f_optimized, speedup_factor):
    """Amdahl 定律计算整体加速比"""
    return 1 / ((1 - f_optimized) + f_optimized / speedup_factor)

def mips_to_exec_time(mips, instructions):
    """MIPS 转执行时间（秒）"""
    return instructions / (mips * 1e6)

# 示例计算
print(f"执行时间: {cpu_execution_time(1e9, 1.2, 2.5):.3f} 秒")  # 0.480 秒
print(f"Amdahl加速比: {amdahl_law(0.8, 4):.2f}")  # 2.50
```

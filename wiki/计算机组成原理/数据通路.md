## 核心定义

数据通路（Datapath）是 CPU 内部执行数据处理和传输的硬件集合体，包括组合逻辑单元（ALU、加法器、符号扩展器、多路选择器）、时序逻辑单元（寄存器堆、程序计数器 PC、流水线寄存器、状态寄存器）以及连接这些单元的互连总线。简单来说，数据通路是数据在 CPU 内部流动的物理通道和操作单元。在控制器的协调下，数据通路根据当前指令的需求选择适当的操作组件和传输路径，实现取指、译码、运算、访存和写回等功能。不同的指令（R-type 算术逻辑、I-type 访存、分支/跳转等）在数据通路中经过不同的组件路径。数据通路的设计目标是在满足时钟周期约束的前提下将关键路径（Critical Path）最小化，使各阶段延迟均衡。

## 关键结论

- 数据通路的核心组件：PC（程序计数器）、指令存储器 IM、寄存器堆 RF、ALU、数据存储器 DM、符号扩展器、多路选择器 MUX、加法器
- 单周期数据通路：所有指令共享同一数据通路，各组件通过 MUX 和去MUX 连接；最长指令路径决定时钟周期
- 多周期数据通路：可复用昂贵资源（如 ALU 既用于计算也用于地址计算、加法器既 PC+4 也用于分支），减少硬件冗余
- 数据通路的关键路径分析：从 PC 出发 -> IM -> RF -> ALU -> DM -> RF，计算该路径中各组件延迟总和
- Load 指令使用了数据通路的所有五个阶段（全部单元），是路径最长的指令

## 易错点

1. 寄存器堆"先写后读"的时序细节：写入发生在时钟上升沿，读出是组合逻辑（异步），若同周期需写一个寄存器并读另一个，读出的组合逻辑看到的是写入前的旧值（前半个周期写入还未发生）。
2. PC 的更新逻辑：PC = PC + 4（顺序）或 PC = Branch_Target（分支时），使用单独的加法器和 MUX。
3. 数据通路图中箭头方向表示数据流向，可能有汇聚（多个源通过 MUX 进入同一目的地）。

## 例题

**例题1：** 分析 LW R1, 100(R2) 在单周期 MIPS 数据通路中的数据流向。

**解答：** IF: PC -> IM -> IR。ID: IR[25:21]=R2 作为读地址 -> RF 读出 R2 的值；IR[15:0]=100 经符号扩展为 32 位。EX: ALUSrc=1（选符号扩展后的立即数），R2 值与 100 送入 ALU 做加法得访存地址。MEM: MemRead=1，ALU 结果作为地址从 DM 读数据。WB: MemtoReg=1，DM 读出数据通过 MUX 写入 R1（IR[20:16]）。

**例题2：** 单周期 CPU 中，为何 lw 指令的延迟决定了整个 CPU 的时钟周期？

**解答：** lw 指令的数据路径最长：IF(IM访问) + ID(RF读) + EX(ALU+移位) + MEM(DM访问) + WB(RF写)。任何缩短时钟周期的尝试都会导致 lw 指令无法在一个周期完成。多周期和流水线 CPU 将长路径切分以解决此瓶颈。

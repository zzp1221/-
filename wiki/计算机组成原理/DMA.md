## 核心定义

DMA（Direct Memory Access，直接存储器访问）是一种高速数据传输机制，允许 I/O 设备（如磁盘控制器、网卡、显卡）直接与主存交换数据，而无需 CPU 介入每一个数据字的传输。DMA 的核心组件是 DMA 控制器（DMAC），它作为一个特殊的总线主设备（Bus Master），代替 CPU 掌控总线控制权，按设定的源地址、目标地址和传输字节数自动完成数据块的传送。DMA 的工作流程：(1) CPU 初始化 DMA 控制器——设置源地址、目标地址、传输字节数和传输方向；(2) I/O 设备向 DMAC 发出 DMA 请求（DREQ）；(3) DMAC 向 CPU 发出总线请求（HOLD/BR）；(4) CPU 完成当前总线周期后释放总线，回送总线应答信号（HLDA/BG）；(5) DMAC 占用总线并进行地址输出、数据传送；(6) 传输完毕，DMAC 释放总线并向 CPU 发中断通知。DMA 将 CPU 从"搬运工"角色中解放出来，使 CPU 能并行处理其他任务。

## 关键结论

- DMA 的三种传送方式：块传送（Burst Mode，一次将整个数据块传完，期间 CPU 无法使用总线）、周期窃取（Cycle Stealing，每次只传 1 字/字节后释放总线，CPU 可间歇工作）、透明传送（Transparent Mode，利用 CPU 不使用总线的空闲周期传送）
- DMA 显著降低 CPU 在 I/O 操作中的开销：将 CPU 从逐字搬运降为仅初始化和中止处理
- 分散-聚集（Scatter-Gather）DMA：支持从非连续的多个内存区域分别传输数据，使用描述符链表
- DMA 与 Cache 一致性问题：DMA 写入的数据可能导致 Cache 中副本过时，需要硬件/软件一致性维护
- 现代系统中的 DMA 已深度集成（如 Intel IOMMU/VT-d 提供 DMA 重映射和保护）

## 易错点

1. DMA 不是"不需要 CPU"：它是"减少 CPU 的干预"，初始化配置和结束处理仍需要 CPU。CPU 整体开销从每个字的介入变成每个数据块的介入。
2. DMA 传送单位不一定是字节：可以是字（word）或数据块，根据 DMAC 的配置和总线宽度决定。
3. DMA 请求和中断请求是两个不同的信号通路：DMA 请求是设备请求 DMAC 的传送服务，中断是设备请求 CPU 的 ISR 服务。

## 例题

**例题1：** 用 DMA 传送 40KB 数据块，DMA 初始化 2000 时钟周期，中断处理 1500 时钟周期。总线频率 100MHz，数据宽度 4 字节。求 CPU 为此次传输付出的时钟周期和 DMA 占用的总线时间。

**解答：** CPU 付出：初始化 2000 + 中断处理 1500 = 3500 周期。DMA 传输：40KB / 4B = 10K = 10000 次总线传输，每次至少 1 个总线周期。若每次传输 1 个总线周期，DMA 占用 10000 周期。若 CPU 每字都介入（程序查询），需额外 10000+ 周期。

**例题2：** 比较程序查询、中断方式和 DMA 方式在大量数据传送中的效率。

**解答：** 程序查询：CPU 循环检查设备状态，每传 1 字都需 CPU 执行若干指令（读状态、读/写数据、更新指针），CPU 开销 = O(数据量)。中断：每传 1 字触发一次中断，CPU 保存/恢复现场开销巨大，大量数据时极低效。DMA：CPU 仅初始化和结束介入，开销 = O(1)（每个数据块），是大量数据传送的首选。

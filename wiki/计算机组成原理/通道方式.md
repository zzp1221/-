## 核心定义

通道（I/O Channel）是一种专门用于管理 I/O 设备数据交换的协处理器，介于 CPU 和 I/O 设备控制器之间。通道拥有自己的指令集（通道指令/Channel Command Word, CCW）和简单的执行能力，可以独立于 CPU 执行存储在内存中的通道程序（Channel Program），管理多个设备的高速数据传送。通道是实现 I/O 控制的最高级方式，将 CPU 从繁琐的 I/O 管理任务中彻底解放出来，CPU 只需向通道发出一条"启动 I/O"（SIO, Start I/O）指令，通道即可独立完成指定数据块的传送，完成后向 CPU 产生一个中断。通道方式主要用于大型机（Mainframe，如 IBM System/360）和高端服务器。按工作方式，通道分为三种类型：字节多路通道（Byte Multiplexer Channel）——以字节为单位轮流为多个低速设备（打印机、终端）服务，分时共享通道带宽；选择通道（Selector Channel）——成组地为一个高速设备（磁盘、磁带）服务，直到传送完成才释放通道；数组多路通道（Block Multiplexer Channel）——结合两者优点，以数据块为单位轮流为多个高速设备服务。

## 关键结论

- 通道是具有 DMA 能力的专用处理器：可执行存储在内存中的通道程序（CCW 链），实现复杂的数据传送序列
- 通道程序由通道指令（CCW）组成，包括读、写、控制、转移通道指令、标记设备状态等
- 通道与 CPU 共享内存，采用周期窃取（Cycle Stealing）方式在总线空闲周期获取内存访问权
- 通道方式的硬件入口：启动 I/O 指令 -> 通道从固定内存地址（如 CAW, Channel Address Word）获取 CCW 链地址 -> 执行通道程序
- 现代系统中，"通道"概念已演进为智能 I/O 处理器（IOP）和独立的 DMA 引擎（如 Intel I/OAT）

## 易错点

1. 通道和 DMA 控制器的本质区别：DMA 控制器仅能完成预设的简单地址-长度的数据块传送；通道可以执行存储在内存中的通道程序，实现更复杂的传送逻辑和条件分支。
2. 通道不是"无 CPU"的替代：通道辅助 CPU 完成 I/O 设备管理，但永远在 CPU 的调度（SIO 指令）之下工作。
3. 字节多路、选择和数组多路并非互斥的物理实体：同一通道可以以不同的逻辑模式工作，为不同的设备服务。

## 例题

**例题1：** 比较四种 I/O 控制方式（程序查询、中断、DMA、通道）的 CPU 负载。

**解答：** 程序查询：CPU 100% 忙于轮询（每字/每字节）。中断：每 1 次传送触发 1 次中断，CPU 执行 ISR 完成 1 字传送，大量数据时开销大。DMA：CPU 仅初始化和结束处理，每数据块开销 O(1)。通道：CPU 仅发 SIO 指令，通道独立执行通道程序，CPU 负载最低，可以完全释放 CPU 去执行其他任务。

**例题2：** 解释选择通道为什么不适用于同时为多个设备服务。

**解答：** 选择通道一旦选定一个设备，便将全部通道带宽独占给该设备，直到该设备的数据传送完成（如完整的磁盘块），其间其他设备被完全阻塞。这保证了高速设备的最大吞吐率，但牺牲了并发服务能力。例如，若磁盘传送 8KB 数据需 1ms，其他设备必须等待这 1ms。数组多路通道通过分块（如 256B 一块）服务多个设备来解决此问题。

## 核心定义

指令系统（Instruction Set Architecture, ISA）是计算机硬件与系统软件之间的接口规范，定义了 CPU 可执行的全部机器指令的集合及其行为。ISA 规定了以下几个方面：指令格式——每条指令的编码布局（操作码 OP + 地址码/操作数），包括字段划分和位宽；数据类型——指令可以操作的数据类型（整数、浮点数、向量等）和大小（8/16/32/64 位）；寄存器组织——通用寄存器和特殊寄存器的数量、名称和功能约定（如 x86 的 EAX/EBX 和 ARM 的 R0-R15）；寻址方式——指令获取操作数的方式（立即、寄存器、直接、间接等）；异常处理——中断、陷阱、系统调用的机制。ISA 向上支撑操作系统和编译器，向下约束处理器微架构的实现，是软硬件之间最稳定的契约。著名的 ISA 包括 x86（Intel/AMD，CISC）、ARM（RISC）、RISC-V（开源RISC）、MIPS 等。

## 关键结论

- ISA 是区分不同类型 CPU 的根本标志：同为 x86 ISA 的 CPU 可运行相同的二进码
- 操作码（Opcode）的编码方式分为定长和变长两种：RISC 通常定长（32 位），CISC 通常变长（x86 1-15 字节）
- 三地址指令格式：OP Rd, Rs1, Rs2（如 RISC-V），两地址格式：OP Rd, Rs（Rd 兼作源和目标，如 x86）
- 通用寄存器型（Load-Store）架构是目前的主流：所有运算操作数必须来自寄存器，存储器访问通过 Load/Store 独立指令
- ISA 设计权衡：指令数量、寻址方式丰富度、编码效率、硬件解码复杂度之间的折中

## 易错点

1. ISA 不等于微架构：ISA 定义了"什么能做"（指令语义），微架构定义了"怎么做"（流水线/超标量/乱序执行等实现）。同样的 ISA 可以有不同的微架构实现。
2. 向后兼容性的代价：x86 维持了 40 多年的向后兼容，但新增了大量扩展（SSE, AVX, x86-64），导致解码逻辑极其复杂。
3. 指令长度和编码空间的关系：定长指令浪费编码空间（简单指令也占 32 位），变长指令压缩代码体积但解码复杂。

## 例题

**例题1：** 比较 RISC-V RV32I 和 x86 在 ADD 指令上的格式差异。

**解答：** RISC-V: add x1, x2, x3 -> 32 位定长编码：funct7 | rs2 | rs1 | funct3 | rd | opcode。x86: add eax, ebx -> 变长 2 字节：01 D8（ModRM 和 opcode 融合），操作数之一隐含为累加器或由 ModRM 指定。

**例题2：** Load-Store 架构的限制和优势。

**解答：** 限制：无法直接对内存操作数做运算（必须先 Load 到寄存器，运算后再 Store 回去），指令数增加。优势：指令格式规整（便于流水线）、Load/Store 延迟可被隐藏、便于编译器和优化。几乎所有现代 RISC 都采用 Load-Store 架构。

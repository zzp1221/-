## 核心定义

哈佛结构（Harvard Architecture）是一种将程序指令存储器和数据存储器在物理上分开的计算机体系结构。与冯诺依曼结构不同，哈佛结构使用两条独立的地址总线和数据总线，允许 CPU 在同一时钟周期内同时访问指令和数据，避免了冯诺依曼结构中的"取指-取数"争用瓶颈。该结构最早在哈佛大学的 Harvard Mark I（1944年）实现，因此得名。纯粹的哈佛结构中，指令存储器和数据存储器的地址空间完全独立（可以重叠编址），指令只能从指令存储器读取，数据操作使用数据存储器。现代处理器多采用"改进型哈佛结构"（Modified Harvard Architecture），即一级缓存分为独立的指令 Cache（I-Cache）和数据 Cache（D-Cache），但对外（主存/外存）呈现统一的冯诺依曼地址空间，兼顾两者的优势。

## 关键结论

- 哈佛结构的核心优势是指令和数据并行访问：可以同时取指和取操作数，显著提升性能
- 独立地址空间使得指令存储器可以使用只读 ROM，数据存储器使用可读写 RAM，适合嵌入式
- 改进型哈佛结构（Modified Harvard）是当前主流：L1 分为 I-Cache 和 D-Cache，L2/L3 和主存共享
- 哈佛结构在 DSP（数字信号处理器）和微控制器（如 Atmel AVR、PIC）中非常普遍
- 对比冯诺依曼：冯诺依曼更简单灵活、共享存储可动态平衡指令和数据；哈佛结构更高性能但硬件复杂

## 易错点

1. 将 L1 I-Cache / D-Cache 分离等同于哈佛结构：现代 CPU 虽然有分离的 L1 Cache，但对外呈现统一地址空间，实质上是改进型哈佛结构，而非纯哈佛结构。
2. 混淆纯哈佛结构的地址范围独立性：纯哈佛结构中地址 0x1000 可以同时对应一个指令和一个数据，二者由访问总线区分而非地址区分。
3. 现代 PC CPU 是冯诺依曼还是哈佛？答案：两者兼有。指令集层面是冯诺依曼，微架构 L1 层面是改进型哈佛。

## 例题

**例题1：** 对比冯诺依曼和哈佛结构在一个简单加法操作中的执行效率。

**解答：** 冯诺依曼结构：1 周期取指（占用总线），1 周期取数（占用总线），1 周期执行，共 3 周期。哈佛结构：可以 1 周期同时取指和取数（独立总线），1 周期执行，共 2 周期。当有大量 Load/Store 操作时，哈佛的效率优势明显。

**例题2：** 改进型哈佛结构如何实现"自修改代码"？

**解答：** 纯哈佛结构不允许修改指令（指令 ROM），但改进型哈佛通过对指令存储区开放写入通道（部分 ARM 处理器支持写入 I-Cache 的指令）或通过统一 L2 Cache 桥接 I-Cache 和 D-Cache 来实现。在需要刷新指令缓存时，使用专门的同步指令（如 ARM 的 ISB）保证一致性。

## 代码示例

```asm
; 哈佛结构示意：指令和数据分别存储

; === 指令存储器 (ROM) ===
; 地址 0x0000: 取数指令（从数据存储器地址 0x1000）
; 地址 0x0002: 加法指令
; 地址 0x0004: 存数指令（到数据存储器地址 0x1002）

; === 数据存储器 (RAM) ===
; 地址 0x1000: 5
; 地址 0x1002: 0 (结果)

; ARM Cortex-M3（改进型哈佛）示例
; I-Bus: 取指，D-Bus: 取数/存数，S-Bus: 系统外设

    LDR R0, =0x20000000   ; 数据存储器地址（D-Bus 访问）
    LDR R1, [R0]          ; 从数据存储器取数
    ADD R1, R1, #10       ; 运算
    STR R1, [R0]          ; 存回数据存储器
    ; 上述指令自身通过 I-Bus 从 Flash 取指
```

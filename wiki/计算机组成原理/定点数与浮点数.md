## 核心定义

在计算机中表示实数有两种基本方式：定点数（Fixed-Point）和浮点数（Floating-Point）。定点数将小数点固定在某一确定位置：定点整数的小数点固定在最低位之后，表示纯整数；定点小数的小数点固定在最高位之前（符号位之后），表示纯小数。定点数的优势在于硬件实现简单、运算速度快，但数值表示范围和精度互相制约——对给定字长 n，定点整数范围约为 [-2^{n-1}, 2^{n-1}-1]，定点小数精度固定为 2^{-(n-1)}。浮点数则采用科学计数法的形式 N = (-1)^S * M * R^E，其中 S 为符号位，M 为尾数（Mantissa/Significand），R 为基数（计算机中 R=2），E 为阶码（Exponent）。浮点数以牺牲部分尾数精度为代价，换取极大的数值表示范围（可表示极大或极小的数），是现代计算机实数计算的标准方案（遵循 IEEE754 标准）。

## 关键结论

- 定点数特点：精度均匀分布，实现简单，适合 DSP、嵌入式等对精度和速度有严格要求的场景
- 浮点数特点：精度非均匀分布（接近零精度高，远离零精度低），数值范围极大
- 定点数运算：加减法需小数点对齐（通过移位）；乘法结果需截断或舍入
- 浮点数能表示的数值范围由阶码位数决定，精度由尾数位数决定
- 浮点数的规格化（Normalization）确保尾数最高位为 1（隐含位优化），使表示唯一并最大化精度

## 易错点

1. 定点小数和定点整数的字面混淆：在小数点左移/右移的运算中（如乘法后的截位），极易将小数点和符号位混淆。
2. 浮点数不能精确表示所有实数：0.1 在二进制中是循环小数，无法精确存储，导致浮点舍入误差。
3. 定点数和浮点数的范围与精度是一对权衡：增加尾数位提高精度但缩小范围；增加阶码位扩大范围但降低精度。

## 例题

**例题1：** 8 位定点小数（1位符号位，7位数值位）的表示范围和精度。

**解答：** 符号位 1 位，数值位 7 位。最大正数 = 0.1111111 = 1 - 2^{-7} ≈ 0.9921875；最小负数 = 1.1111111 = -(1-2^{-7})；分辨率（精度）= 2^{-7} = 1/128 ≈ 0.0078125。

**例题2：** 比较定点和浮点表示数值 123456 和 0.0000123456。

**解答：** 定点32位：123456 可精确表示（在 ±2.1*10^9 范围内），0.0000123456 需按最小步长 2^{-31}≈4.7e-10 截断。浮点32位（IEEE754）：两者均可表示，但 123456 精度约 10^{-7}（约 7 位有效数字），0.0000123456 精度同样约 7 位有效数字。

## 代码示例

```python
# 定点小数的简单模拟
class FixedPoint:
    def __init__(self, bits_int=16, bits_frac=16):
        self.scale = 1 << bits_frac  # 比例因子 2^16
        self.bits_frac = bits_frac
    
    def to_fixed(self, value):
        """将浮点数转为定点整数"""
        return int(value * self.scale)
    
    def to_float(self, fixed_value):
        """将定点整数转回浮点数"""
        return fixed_value / self.scale
    
    def add(self, a, b):
        return a + b  # 加法直接相加（比例因子相同）
    
    def mul(self, a, b):
        # 乘法后需右移恢复比例因子
        return (a * b) >> self.bits_frac

# 演示
fp = FixedPoint(16, 16)
a = fp.to_fixed(3.25)
b = fp.to_fixed(2.5)
print(f"a={a}, b={b}")
print(f"a+b={fp.to_float(fp.add(a, b))}")  # 5.75
print(f"a*b≈{fp.to_float(fp.mul(a, b)):.4f}")  # ~8.125
```

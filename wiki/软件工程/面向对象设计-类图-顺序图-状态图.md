## 核心定义

面向对象设计（Object-Oriented Design, OOD）是以对象和类为基本构建单元的设计方法论，遵循封装、继承、多态三大特征。UML（统一建模语言）提供了标准化的模型图来可视化 OOD。

**类图**（Class Diagram）是 OOD 中最核心的静态结构图，描述系统中的类、接口、协作关系及其相互关系。基本元素：(a) 类（矩形框，分三栏：类名、属性、操作/方法）；(b) 可见性：`+` public、`-` private、`#` protected、`~` package；(c) 关系：**关联**（Association，实线，两端可标注多重性如 1, 0..*, 1..*）、**聚合**（Aggregation，空心菱形，整体-部分弱关联）、**组合**（Composition，实心菱形，整体-部分强关联，部分生命周期由整体管理）、**泛化**（Generalization，空心三角箭头，继承）、**依赖**（Dependency，虚线箭头，一个类使用另一个类）、**实现**（Realization，虚线空心三角，实现接口）。

**顺序图**（Sequence Diagram）是行为交互图，强调消息在对象之间传递的**时间顺序**。纵轴为时间线（Lifeline，竖直虚线），横轴为对象。激活条（Activation Box）表示对象正在执行操作。消息以水平箭头表示。顺序图特别适合描述用例的"正常流程"——谁向谁发送了什么消息、以什么顺序。

**状态图**（State Diagram / State Machine Diagram）描述单个对象在其生命周期内的状态变迁。要素：状态（圆角矩形）、初始状态（实心圆）、终止状态（牛眼）、转移（Transition，带箭头的线，标注 `事件[守卫条件]/动作`）。状态图适合描述"有明确状态对象"的行为——如订单（待支付→已支付→已发货→已完成→已取消）、TCP 连接、用户会话。

## 关键结论

- 类图是"骨架"，顺序图是"叙事"，状态图是"传记"——三图从不同视角互补
- 类图要回答"有哪些类、类之间什么关系"；顺序图要回答"这个用例中各个对象如何协作"
- 聚合 vs 组合的关键：组合中部分对象的生命周期**依附**于整体（整体销毁则部分也销毁）
- 状态图的核心：在给定状态下收到事件→根据守卫条件决定→执行动作→转移到新状态
- OOD 的设计输出通常伴随 CRC 卡片（Class-Responsibility-Collaborator）——类、职责、协作者
- 领域驱动设计（DDD）中，实体、值对象、聚合根等概念与 UML 类图紧密配合

## 易错点

1. 聚合与组合的混淆：组合 = 整体对部分有**独占性所有权**（如汽车与引擎）；聚合 = 整体包含部分但部分可独立存在（如班级与学生）
2. 顺序图中同步消息与异步消息的箭头：同步 = 实心箭头（调用者等待返回），异步 = 开放箭头（发送后不等待）
3. 状态图中未覆盖所有事件：一个完备的状态图应为每个状态定义对所有可能输入事件的响应——遗漏事件意味着未定义行为

## 例题

**例题1**：为"图书馆借书"场景画顺序图。参与者：借书者（Borrower）、图书馆员（Librarian）、系统（LibrarySystem）、数据库（Database）。

**解答**：
```
Borrower → Librarian: 提出借书请求(bookId)
Librarian → LibrarySystem: checkOut(borrowerId, bookId)
LibrarySystem → Database: query(bookId)
Database --→ LibrarySystem: bookInfo
LibrarySystem → LibrarySystem: 验证借阅资格
LibrarySystem → Database: updateInventory(bookId)
Database --→ LibrarySystem: OK
LibrarySystem --→ Librarian: 借阅成功
Librarian --→ Borrower: 交付图书
```

**例题2**：为订单对象画状态图。状态包括：新建(New)、已支付(Paid)、已发货(Shipped)、已完成(Completed)、已取消(Cancelled)。

**解答**：
New → Paid [支付成功]
New → Cancelled [用户取消 / 超时]
Paid → Shipped [商家发货]
Shipped → Completed [用户确认收货]
Paid → Cancelled [商家取消]
Shipped → Cancelled [不常见但可能，用户拒收等]

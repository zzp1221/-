## 核心定义

黑盒测试与白盒测试是从不同视角出发的两大类测试设计方法。"黑盒"和"白盒"的比喻来源于被测程序对测试者的可见程度。

**黑盒测试**（Black-Box Testing）忽略程序内部结构和实现细节，仅基于**输入/输出**和功能需求设计测试用例——把程序当作一个"黑盒子"，只关心输入什么输出什么。核心方法包括：
- **等价类划分**（Equivalence Partitioning）：将输入域划分为若干等价类（有效等价类和无效等价类），从每个等价类选一个代表值测试。原理：程序对同一等价类中的不同输入通常表现相同。例：年龄输入 0-120 为有效等价类（如取 50），<0 和 >120 为无效等价类（各取一个代表）。
- **边界值分析**（Boundary Value Analysis, BVA）：大量错误发生在输入域的边界附近。对每个等价类，不仅测试代表值，更要测试边界值——最小值、最大值、最小值-1、最大值+1。例：有效年龄 0-120，边界值 = -1, 0, 120, 121。
- **决策表测试**（Decision Table）：当输入组合和对应的输出逻辑复杂时，用决策表系统化列出所有规则。
- **因果图**（Cause-Effect Graph）：用图形化方式表示输入（因）与输出（果）的逻辑关系，生成测试用例。

**白盒测试**（White-Box Testing / Structural Testing）关注程序内部逻辑结构和代码路径——测试者能看到（或假设能看到）源代码，设计测试用例以覆盖特定的代码路径。核心覆盖标准（覆盖面从弱到强）：
- **语句覆盖**（Statement Coverage）：每条可执行语句至少执行一次。最弱的覆盖标准——可能遗漏条件分支的特定取值。
- **判定覆盖**（Branch / Decision Coverage）：每个判定的真分支和假分支至少各执行一次。比语句覆盖强，但可能遗漏条件内部的子条件。
- **条件覆盖**（Condition Coverage）：每个布尔子条件（如 `a>3` 在 `a>3 && b<10` 中）的每个可能的真/假值至少各出现一次。
- **判定-条件覆盖**：同时满足判定覆盖和条件覆盖。
- **多重条件覆盖**（Multiple Condition Coverage）：所有布尔子条件的所有真/假组合至少各出现一次（2ⁿ 种组合）。最严格的覆盖标准。
- **路径覆盖**（Path Coverage）：程序中所有可能执行路径都被覆盖——通常因循环导致路径无限多而不切实际，实际中使用"基本路径覆盖"（McCabe 圈复杂度）。

## 关键结论

- 等价类划分 + 边界值分析的组合是最高效的黑盒测试设计方法——少量测试用例覆盖大量输入域
- 黑盒测试关注"功能正确"，白盒测试关注"逻辑正确"——两者互补
- 语句覆盖是最低要求——任何测试都应至少 100% 语句覆盖
- 条件/判定覆盖（条件组合覆盖）是多数工业标准（如 DO-178C 航空软件）的要求
- 基本路径测试 = McCabe 圈复杂度 + 独立路径枚举，是实践中最常用的白盒方法
- 100% 路径覆盖在所有实际程序中都不可达到（因为循环），不必追求

## 易错点

1. 黑盒测试中等价类划分不完整：容易遗漏无效等价类（异常输入、空值、超长字符串等）
2. 白盒测试中认为语句覆盖 = 判定覆盖：`if (condition) { statement; }` 中若 condition 恒真，语句覆盖 100% 但判定覆盖仅 50%（未覆盖 condition 为假的情况）
3. 混淆条件覆盖和分支覆盖：条件覆盖涉及子表达式，分支覆盖涉及整个条件判断的真/假

## 例题

**例题1**：某函数 `grade(score)` 根据分数返回等级：90-100=A, 80-89=B, 70-79=C, 60-69=D, 0-59=F。设计黑盒测试用例（含等价类和边界值）。

**解答**：
等价类及代表值：有效：A(95), B(85), C(75), D(65), F(50)。无效：<0(-10), >100(110)。

边界值：-1, 0, 59, 60, 69, 70, 79, 80, 89, 90, 100, 101。

**例题2**：给定 `if (a>0 && b>0)`, 设计满足语句覆盖、判定覆盖、条件覆盖和多重条件覆盖的测试用例。

**解答**：
- 语句覆盖：a=1, b=1（条件真，进入if体）即可（如果if体外还有其他语句）
- 判定覆盖：需要 True 和 False 各一次。用例 (a=1, b=1) → True；(a=-1, b=0) → False
- 条件覆盖：a>0 需要 T 和 F 各一次；b>0 需要 T 和 F 各一次。用例 (a=1,b=1) → aT,bT；(a=-1,b=-1) → aF,bF
- 多重条件覆盖（4种组合）：(T,T): (1,1); (T,F): (1,-1); (F,T): (-1,1); (F,F): (-1,-1)

## 核心定义

创建型设计模式（Creational Design Patterns）是 GoF 23 种设计模式中的第一类，专注于**对象创建机制**的抽象。它们将对象的创建和使用解耦——客户端不直接 `new` 对象，而是通过模式封装创建过程，提高系统的灵活性和可配置性。

**(1) 单例模式**（Singleton）：确保一个类有且仅有一个实例，并提供一个全局访问点。实现要点：私有构造函数、静态私有实例变量、静态公有获取方法。多线程环境需双重检查锁（Double-Checked Locking）或静态初始化保证线程安全。适用场景：数据库连接池、配置管理器、日志记录器。注意：单例隐含全局状态，可能阻碍单元测试（用依赖注入替代可缓解）。

**(2) 工厂方法模式**（Factory Method）：定义一个创建对象的接口，但让子类决定实例化哪个类——将对象的实例化延迟到子类。结构：Creator（抽象工厂，含 factoryMethod()）、ConcreteCreator（重写工厂方法返回具体产品）、Product（产品接口）、ConcreteProduct。适用场景：框架让用户扩展内部组件（如文档编辑器让子类决定创建什么类型的文档）。

**(3) 抽象工厂模式**（Abstract Factory）：提供一个接口用于创建一系列相关或相互依赖的对象，而不指定其具体类。与工厂方法的区别：工厂方法创建一个对象，抽象工厂创建一族对象（如不同操作系统的 GUI 组件：按钮、文本框、菜单）。适用场景：跨平台 UI 工具包（Windows/Mac/Linux 各自一套组件）。

**(4) 建造者模式**（Builder）：将复杂对象的构建过程与其表示分离——同样的构建过程可以创建不同的表示。包含：Director（导向器，控制构建步骤）、Builder（抽象建造者）、ConcreteBuilder（具体建造者）、Product（产品）。适用场景：构建包含多个部件、有复杂装配步骤的对象（如富文本转换器、SQL 查询构造器）。

**(5) 原型模式**（Prototype）：通过复制（克隆）现有对象（原型）来创建新对象，而非通过 `new`。实现：定义 clone() 方法（深拷贝 vs 浅拷贝需根据需求）。适用场景：对象创建成本高（如复杂初始化）、需要动态注册原型并在运行时克隆。

## 关键结论

- 创建型模式的共同目标：封装对象创建的"what, who, how, when"
- 单例模式是唯一关注"限制实例数"的模式，其他模式关注创建过程的抽象和灵活性
- 工厂方法和抽象工厂分别对应"单一产品等级"和"产品族"
- 建造者模式与抽象工厂的区别：建造者逐步构建复杂对象（关注步骤），抽象工厂一次性返回产品族
- 原型在 JavaScript 等原型继承语言中有天然的语言支持
- 创建型模式在实践中广泛用于依赖注入（DI）框架和 ORM 框架中

## 易错点

1. 单例的双重检查锁在旧版 Java（<1.5）中由于内存模型问题会失效——需 volatile 变量或使用枚举实现
2. 工厂方法与静态工厂方法的混淆：GoF 的工厂方法是多态方法（依赖继承），静态工厂方法（如 `Integer.valueOf()`）更接近简单工厂
3. 建造者模式中 Director 不是必需的——当构建步骤不复杂时，可以省略 Director 直接用 Builder 链式调用

## 例题

**例题1**：设计一个跨平台 UI 工具包，支持 Windows 和 Mac，包含 Button 和 Checkbox 组件。使用何种创建型模式？

**解答**：抽象工厂模式。定义 GUIFactory 接口含 createButton() 和 createCheckbox()。WinFactory 返回 WinButton 和 WinCheckbox；MacFactory 返回 MacButton 和 MacCheckbox。新增 Linux 平台时只需增加 LinuxFactory，客户端代码无需修改（开闭原则）。

**例题2**：评价以下单例实现的问题。
```java
public class Singleton {
    private static Singleton instance;
    private Singleton() {}
    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
```

**解答**：多线程问题——两个线程可能同时通过 `if(instance==null)` 检查，都创建实例。解决：(a) `synchronized` 方法（简单但性能差）；(b) 双重检查锁 + `volatile`；（c）静态内部类持有实例（推荐）；(d) 枚举单例（Java 最佳实践）。

## 视频简介

Rust 最核心也最难的概念——所有权（Ownership）、借用（Borrowing）、生命周期（Lifetime），用图解和对比的方式讲清楚。通过对比 C/C++ 的手动内存管理和 Java/Go 的 GC，让你理解 Rust "零成本抽象的 RAII + 编译期借用检查"这套独特方案的设计动机和实际效果。

适合有 C/C++ 或 Java 基础，想理解 Rust 为什么敢宣称"无 GC 也能内存安全"的学生。

## 覆盖知识点
- 所有权三规则：每个值有且仅有一个所有者、值离开作用域自动 drop、所有权可以转移（Move）
- 栈分配 vs 堆分配：Copy trait vs Move 语义
- 引用与借用：不可变引用（&T）与可变引用（&mut T）
- 借用规则：同时只能有一个可变引用或多个不可变引用
- 生命周期标注：函数签名中的生命周期参数 'a
- 智能指针：Box<T>、Rc<T>、Arc<T>、RefCell<T>
- 对比：C/C++ 手动管理 vs Java/Go GC vs Rust 编译期检查

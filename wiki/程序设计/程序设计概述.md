## 核心定义

程序设计（Programming）是使用形式化语言将问题的解决方案转化为计算机可执行指令的过程。程序设计的核心是**抽象**——将复杂问题分解为可管理的模块，每个模块通过精确定义的接口交互。程序设计不仅是"写代码"——它涵盖问题分析、算法设计、数据结构选择、编码实现、测试验证、优化和维护的完整过程。

**编程语言分类**（Programming Language Classification）：

**按执行方式分类**：
- **编译型语言**（Compiled）：源代码通过编译器（Compiler）转换为目标机器码（Machine Code）或中间码（Bytecode），随后在目标平台上执行。特点：执行前有完整的编译过程，运行时速度快但修改后需重新编译。代表语言：C、C++、Go、Rust。
- **解释型语言**（Interpreted）：源代码由解释器（Interpreter）逐行翻译并执行，无独立的可执行文件。特点：开发调试灵活、跨平台性好但执行速度较慢。代表语言：Python、JavaScript (传统)、Ruby、Perl。
- **混合型**（Hybrid/JIT）：编译为字节码（Bytecode），运行时由 JIT（Just-In-Time）编译器动态编译为机器码。代表语言：Java (JVM + JIT)、C# (.NET CLR)、JavaScript (现代 V8 引擎)。

**按范式分类**：
- **命令式**（Imperative）：描述"如何做"——逐步操作状态。C、Pascal。
- **面向对象**（OOP）：数据和操作封装为对象，通过继承和多态组织。Java、C++、Python（多范式）。
- **函数式**（Functional）：将计算视为无副作用的函数求值。Haskell、Elixir、F#。现代语言大多支持函数式特性（Python 的 Lambda, Java 8+ 的 Stream API, Rust 的 Iterator）。
- **声明式**（Declarative）：描述"做什么"而非"怎么做"。SQL、HTML、Prolog。
- **逻辑式**（Logic）：声明事实和规则，系统自动推理。Prolog、Datalog。

**按类型系统分类**：
- **静态类型**（Static Typing）：类型在编译时已知和检查。按类型推断程度分为显式类型（C/Java）和推断类型（Haskell/Rust/TypeScript）。
- **动态类型**（Dynamic Typing）：类型在运行时确定。Python、Ruby、JavaScript。
- **强类型 vs 弱类型**：强类型（Strong Typing）不会隐式转换不兼容类型（Python `"1" + 1` 报错）；弱类型（Weak Typing）自动执行类型强制转换（JavaScript `"1" + 1` → `"11"`）。

**程序设计的核心概念**：
- **变量与赋值**（Variable and Assignment）：存储和操作数据的基本单位
- **控制流**（Control Flow）：顺序（Sequence）、选择（Selection，if/switch）、循环（Iteration，for/while）
- **子程序/函数**（Subroutine/Function）：封装逻辑复用的抽象单位
- **数据结构**（Data Structures）：组织和存储数据的方式（数组、链表、树、哈希表、图）
- **算法**（Algorithms）：问题求解的步骤化方法（排序、搜索、图遍历、动态规划等）

**编程语言发展的关键里程碑**：Fortran (1957, 第一个高级语言) → LISP (1958, 函数式编程) → C (1972, UNIX 开发和系统编程) → Smalltalk (1980, OOP 先驱) → C++ (1985, C + OOP) → Java (1995, 跨平台 + GC) → Python (1991, 可读性和简单性) → C# (2000, .NET) → Go (2009, 并发和系统编程) → Rust (2015, 内存安全零成本抽象) → TypeScript (2012, JavaScript 的超集带类型)。

## 关键结论

- 编程语言的选择是权衡：性能 vs 生产力、安全性 vs 灵活性、抽象 vs 控制
- 多范式语言（Python, C++, TypeScript）比单一范式语言更灵活——但也更容易写出不一致风格的代码
- 编译型和解释型不是绝对的二元对立——现代语言几乎都有 JIT 或提前编译选项
- 类型系统不是负担——它是对错误的提前捕捉，将运行时错误转化为编译时错误
- 不存在"最好的编程语言"——最适合特定场景的语言才是最佳选择

## 易错点

1. "编译型语言快于解释型语言"是笼统的——JIT 编译和 Just-In-Time 优化（热点分析/动态去虚拟化）可使解释执行的语言运行时性能接近甚至在某些情况下超过静态编译
2. 语言范式和语言特性混为一谈——C 是命令式语言但函数指针可模拟函数式风格；C++ 是 OOP 但模板元编程是函数式；Python 是多范式
3. 强/弱类型和静态/动态类型的概念混淆——类型何时检查（compile-time vs runtime）与是否隐式转换是两个独立的维度

## 例题

**例题1**：比较 C、Java、Python 三种语言在内存管理策略上的差异。

**解答**：
- C：手动内存管理——开发者通过 `malloc/free` 分配和释放堆内存。无垃圾回收（GC），忘记 `free` 导致内存泄漏，过早 `free` 导致悬空指针。赋予最大控制力但也最容易出错。
- Java：垃圾回收（GC）的自动内存管理——通过 `new` 分配，JVM 的 GC（G1、ZGC 等）自动回收不可达对象。开发者不需要手动释放但 GC 可能在不可预测的时机触发 STW（Stop-The-World）暂停。
- Python：引用计数 + 循环引用 GC——每个对象维护引用计数，计数为 0 时立即回收（确定性）。对于循环引用的对象，gc 模块中的分代回收器定期运行。引用计数回收比 Tracing GC 延迟低但无法处理循环引用。

**例题2**：给出 3 个场景并为每个场景推荐最合适的编程语言，说明理由。

**解答**：
(a) 高性能 Web 服务器、高并发 10 万连接——推荐 Go。理由：goroutine 轻量协程（2KB 栈起步）+ 协程调度器天然支持高并发；编译为单二进制部署简单；GC 延迟低（<100μs）。
(b) 数据科学原型和机器学习——推荐 Python。理由：最丰富的科学计算和 ML 生态（NumPy/Pandas/Scikit-learn/PyTorch/TensorFlow）；交互式开发 Jupyter Notebook；非技术用户的友好语法。
(c) 嵌入式系统的固件开发（RAM < 64KB）——推荐 C。理由：零运行时开销和最小内存足迹；直接硬件访问和控制；大多数微控制器的标准编译器支持。

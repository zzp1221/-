## 核心定义

排序算法（Sorting Algorithm）是将元素按特定顺序（升序或降序）重新排列的算法。排序是计算机科学中最基础、研究最深入的算法问题——排序算法的选择直接影响应用的性能和可靠性。

### 比较排序算法

**O(n²) 基础排序**：

1. **冒泡排序**（Bubble Sort）：相邻元素两两比较交换，每轮确定一个最大元素浮到最后。稳定性：稳定（相等元素不交换）。最坏/平均 O(n²)，最好 O(n)（已有序且带提前终止优化）。实践中仅用于教学——数据量小时效率不及插入排序。

2. **选择排序**（Selection Sort）：每轮选择最小元素与当前位置交换。不稳定（交换顺序可能改变相等元素的相对位置）。无论输入，比较次数恒定 n(n-1)/2。仅适用于极小数据集或交换代价极高的情况。

3. **插入排序**（Insertion Sort）：将未排元素依次插入已排序列的适当位置（类似整理扑克牌）。稳定。最坏 O(n²)（逆序），最好 O(n)（正序）。数据量小（< 50 元素）时实践中快于 O(n log n) 算法（常数因子极小 + CPU 分支预测友好）。Timsort（Python/Java 的默认排序）的底层即优化的插入排序合并。

**O(n log n) 高效排序**：

4. **快速排序**（Quick Sort）：选择 pivot，将数组分为 ≤ pivot 和 > pivot 两部分，递归排序两部分。不稳定。平均 O(n log n) 常数小——实践中常为最快的通用排序。最坏 O(n²)（每次 pivot 为极值 → 退化为插入排序）——可通过随机化 pivot 或三数取中优化为期望 O(n log n)。分治递归导致递归栈 O(log n)（平均）。

5. **归并排序**（Merge Sort）：二分数组→递归排序→合并有序两半（Merge）。稳定。平均/最坏 O(n log n)。需要额外 O(n) 空间（非原地）。适合链表排序（O(1) 额外空间，合并操作链表高效）和外部排序（磁盘）。

6. **堆排序**（Heap Sort）：建最大堆 → 依次取出堆顶（最大值）与末尾元素交换（缩小堆再堆化）。不稳定。平均/最坏 O(n log n)。O(1) 额外空间（原地）。相比快排常数略大（堆化跳转导致缓存不友好）。

### 非比较排序（线性排序）

突破了 Ω(n log n) 比较排序理论上限——但这些算法要求数据有特殊性质：

7. **计数排序**（Counting Sort）：适用于整数范围有限的输入 [0, K)。创建计数数组，依次累计得到每个元素的位置。稳定，O(n + K)。当 K >> n 时空间浪费严重。

8. **基数排序**（Radix Sort）：按数字位（个位→十位→百位）或多关键字逐层排序。稳定（每层需稳定排序如计数排序）。O(d × (n + k))，d = 位/关键字数。适用于等长字符串、数字排序。

9. **桶排序**（Bucket Sort）：将数据均匀分配到若干个桶中，每个桶用其他排序（如插入排序）排序，最后拼接各桶。平均 O(n)（数据均匀分布时），最坏 O(n²)。适用于数据均匀分布且范围已知（如浮点数 [0, 1)）。

### 排序算法关键性质对比

| 算法 | 平均 | 最坏 | 空间 | 稳定 | 原地 |
|------|------|------|------|------|------|
| 冒泡 | O(n²) | O(n²) | O(1) | Y | Y |
| 选择 | O(n²) | O(n²) | O(1) | N | Y |
| 插入 | O(n²) | O(n²) | O(1) | Y | Y |
| 快速 | O(n log n) | O(n²) | O(log n) | N | Y |
| 归并 | O(n log n) | O(n log n) | O(n) | Y | N |
| 堆 | O(n log n) | O(n log n) | O(1) | N | Y |
| 计数 | O(n+k) | O(n+k) | O(k) | Y | N |

**排序的稳定性的含义**：若两个元素相等，经排序后其相对顺序不变。稳定性对于多关键字排序很重要——先按次要关键字排序，再按主要关键字排序（稳定排序）得到符合预期的结果。

## 关键结论

- 比较排序的理论下界是 Ω(n log n)（决策树模型）——没有任何基于比较的排序可突破这个下界
- 选择排序算法要考虑：待排数据规模、范围、分布、稳定性需求、内存限制、并行化可能性
- 实践中 Python/Java 使用 Timsort（混合插入 + 归并）——利用实际数据通常存在部分有序的段（天然归并段）达到自适应性能
- 对于小数据（< 50），O(n²) 插入排序是实际最佳（常数小、分支预测友好）——优秀库中的排序会切换到插入排序处理小块

## 易错点

1. 计数排序的计数数组大小用错范围——最大值是 K 则数组大小应为 K+1（元素值 ∈ [0, K]）
2. 快速排序的在 pivot 选取中等值处理不当——三路比两路更能应对大量重复值
3. 基数排序的内部排序必须稳定——否则高位排序会"破坏"低位排序的成果
4. 稳定性与相等元素的定义——浮点数排序中 NaN 破坏了传递性（`NaN != NaN`），可能导致异常

## 例题

**例题1**：实现三路快速排序（处理重复值过多的问题）。

**解答**：标准快速排序在数组中大量重复值的场景退化为 O(n²)。三路快排划分为三个区域：< pivot, == pivot, > pivot。中间相等区无需递归。
```python
def quicksort_3way(arr, lo, hi):
    if lo >= hi: return
    lt, gt = lo, hi
    pivot = arr[lo]
    i = lo
    while i <= gt:
        if arr[i] < pivot:
            arr[lt], arr[i] = arr[i], arr[lt]
            lt += 1; i += 1
        elif arr[i] > pivot:
            arr[gt], arr[i] = arr[i], arr[gt]
            gt -= 1  # i 不前进——新换来的元素还未检查
        else:
            i += 1
    quicksort_3way(arr, lo, lt - 1)
    quicksort_3way(arr, gt + 1, hi)
```

**例题2**：给出以下场景的最佳排序选择并说明理由：(a) 1000 万 64-bit 整数已接近有序；(b) 1000 个学生的成绩按总分排序（成绩 0-750）；(c) 1 万条姓名链表需要按字母排序。

**解答**：
(a) Timsort 或优化的插入排序——接近有序数据中 Timsort 检测到已排序的段并 O(n) 合并。通用快速排序在这类数据中触发最坏情况（已排序的情况下 pivot 选择不当会导致 O(n²)）。
(b) 计数排序——数据范围小（0-750），O(n+751) ≈ O(n)（对 1000 名学生），常数因子极小。1000 元素用快速排序开销也很小但计数排序仍是理论上最优。
(c) 归并排序——链表天生的数据结构优势：归并排序在链表上 O(1) 额外空间且合并操作仅需调整指针，无随机访问问题（快速排序在链表上性能差因为 pivot 选取耗时）。

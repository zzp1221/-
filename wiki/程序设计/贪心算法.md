## 核心定义

贪心算法（Greedy Algorithm）是一种在每一步选择中都采取当前状态下**局部最优**（最佳）的选择，希望累积的结果最终是**全局最优**的算法策略。贪心算法不回溯——一旦做出选择就不再重新考虑。这种"一往无前"的特性使得贪心算法通常非常高效（通常是 O(n log n) 或 O(n)），但前提是问题满足贪心算法成功的两个性质。

**贪心算法成功的两个条件**：
1. **贪心选择性质**（Greedy-Choice Property）：全局最优解可以通过一系列局部最优（贪心）选择达到——即第一步的贪心选择不排除得到全局最优解的可能。这意味着贪心算法不需要在求解过程中回溯或查看前面的选择。
2. **最优子结构**（Optimal Substructure）：做出贪心选择后剩余的子问题的最优解 + 第一步的贪心选择构成原问题的全局最优解。

**贪心 vs 动态规划的区别**：
- DP 考虑所有可能的选择（在所有选择中取最大/最小值）
- 贪心仅考虑"当前看起来最好"的选择（不需要枚举所有子问题）
- 贪心更高效（通常无内层循环），但适用问题更少
- 两者的共同点：都需要最优子结构

**证明贪心算法正确性的常用方法**：
1. **归纳法**（Induction）：假设前 k 步贪心选择正确（存在于某最优解），证明第 k+1 步贪心选择仍然正确
2. **交换论证**（Exchange Argument）：假设存在一个不包含第一步贪心选择的最优解，将其第一个不同选择与贪心选择交换，证明交换后的解不差于原最优解
3. **切分-粘贴法**（Cut-and-Paste）：假设贪心解与某个最优解在某步不同，调换后证明贪心解不差

**经典贪心算法**：

1. **活动选择问题**（Activity Selection）：n 个活动各自有起止时间，选取互不冲突的活动最大集合。贪心策略：每次选择**最早结束**的活动。复杂度 O(n log n)（排序起止时间）。正确性：包容性最优——最早结束剩余时间最多，留给后续活动的空间最大。

2. **赫夫曼编码**（Huffman Coding）：数据压缩的最优前缀编码——字符出现频率不同，为频繁字符分配短编码。贪心策略：每次从优先队列中取出频率最低的两个节点合并为一个新节点（频率相加），重复直到只剩一个根节点——结果是前缀码最优二叉树。O(n log n)。应用：JPEG 压缩、MKV 容器、gzip 的 DEFLATE。

3. **最小生成树（MST）**：
   - **Kruskal 算法**：每次选取**权重最小**的边（若加入后不产生环）。用并查集（Union-Find）维护连通性。O(E log E)。
   - **Prim 算法**：从任意顶点出发，每次将当前集合的**最小权重出边**连接到新顶点。O((V+E) log V)。

4. **Dijkstra 最短路径**（非负权重图）：单源最短路。贪心策略：每次从优先队列中选出当前距离最小的顶点并松弛其邻接顶点的距离。O((V+E) log V)。

5. **硬币找零**（部分面值体系）：用最少硬币凑出给定金额。贪心策略：总是使用剩余金额下可用的最大面额硬币。但贪心算法仅在特定面值体系下保证最优（如人民币 1,5,10,20,50,100 元和美元 1,5,10,25 美分）。反例：面值 1,3,4 找零 6，贪心结果为 4+1+1=3 枚，最优为 3+3=2 枚——贪心失败，需用 DP。

## 关键结论

- 贪心算法的最大难点不是实现而是**证明**——需要证明贪心选择性质和最优子结构均成立
- "看起来贪心可行"往往靠直觉但直觉可能错误——必须严格证明
- 贪心算法常配合优先队列（堆）使用——每次快速获取当前最优选择
- 贪心算法通常是最优算法——若问题满足贪心性质则没有更快的算法能得到最优解

## 易错点

1. 贪心不一定得到正确解——不满足贪心选择性质的问题用贪心只会得到可行解而非最优解
2. 硬币找零问题在任意面额体系中的贪心不一定全局最优——因此 DP 是对该问题的通用解，而贪心仅对"规范面值体系"成立
3. Kruskal 算法不检查负权边——MST 在负权图中同样正确（与 Dijkstra 不同）
4. 活动选择的贪心与"最早开始时间"或"最短持续时间"不同——错误策略会得到少于最优的活动数

## 例题

**例题1**：证明活动选择问题的贪心策略（选择最早结束）的正确性（归纳法）。

**解答**：设 S 为 n 个活动按结束时间排序的集合，a₁ 为第一个结束的活动。需证明存在一个最优解包含 a₁。
设 OPT 是任意一个最优解。若 OPT 包含 a₁ 则定理成立。若 OPT 不包含 a₁，设 OPT 的第一个活动为 a_j。由于 a₁ 是 S 中最早结束的活动，finish(a₁) ≤ finish(a_j)。在 OPT 中用 a₁ 替换 a_j——得到的活动集合 A' = (OPT \ {a_j}) ∪ {a₁}。由于 a₁ 比 a_j 早结束，a₁ 不会与 OPT 中 a_j 之后的活动产生冲突。且 |A'| = |OPT|，A' 也是最优解且包含 a₁。综上，存在包含 a₁ 的最优解——贪心选择性质成立。最优子结构：做出 a₁ 的选择后，原问题退化为子问题 S' = {a_i | start(a_i) ≥ finish(a₁)}。S' 的最优解加上 a₁ 构成 S 的最优解。

**例题2**：以下哪种情况可以用贪心得到最优解：(a) 面值 {1,5,11} 找零 15；(b) 面值 {1,2,5,10} 找零 18；(c) 背包问题（物品可分割）？

**解答**：
(a) 贪心：11+1+1+1+1=5 枚；最优：5+5+5=3 枚；贪心失败。
(b) 贪心：10+5+2+1=4 枚；最优：10+5+2+1=4 枚；贪心得最优。{1,2,5,10} 面值体系属于规范面值系统（每个面额可被前一个面额整除或类似条件？不通用）——需验证。
(c) 分割背包问题（物品可分割成任意比例）：贪心策略——按"价值密度"（value/weight）降序取物品，能全取全取否则取满背包容量的比例部分。这是保证最优解的少数贪心场景——证明：交换论证。

## 核心定义

动态规划（DP, Dynamic Programming）由 Richard Bellman 于 1950 年代发展，是求解具有**最优子结构**和**重叠子问题**的优化问题的算法范式。DP 将原问题分解为子问题，将子问题的解存储在表中以防止重复计算——"将指数时间的朴素递归降到多项式时间"。

"动态规划"的名称来源于 Bellman 当时的工作——"规划"指决策制定（Planning），"动态"指问题随时间变化。"Programming"在此是"表格化计算"（Tabular Method）的古义，与编程语言无关。

**DP 的两个必要条件**：
1. **最优子结构**（Optimal Substructure）：原问题的最优解包含其子问题的最优解——全局最优可通过子问题的最优组合获得。
2. **重叠子问题**（Overlapping Subproblems）：在递归求解过程中相同的子问题被反复计算——如果没有重叠子问题，普通递归、分治法即可（没必要 DP）。

**DP 的两种实现方式**：

**自顶向下**（Top-Down / Memoization / 记忆化搜索）：递归地分解问题，将子问题的解存入记忆表（如 HashMap / 数组）中，若子问题已解则直接查表返回——将重复调用转化为 O(1) 查表。保留递归结构但消除重复。实例如斐波那契：`fib(n) = fib(n-1) + fib(n-2)` 配合 memo 表。

**自底向上**（Bottom-Up / Tabulation）：先求解最小的子问题（基况），然后利用小问题的解逐步构建更大规模问题的解，最终填完整张 DP 表。通常用迭代循环实现——无递归，避免了递归调用和栈溢出。

**DP 解题框架**（5 步法）：
1. 定义**状态**（State）：找出什么参数唯一确定子问题。`dp[i]`、`dp[i][j]`、`dp[i][k]...`
2. 找出**状态转移方程**（State Transition / Recurrence Relation）：问题的最优解如何由子问题的最优解决定。这是 DP 最核心、最难的一步。
3. 初始化**基况**（Initial Conditions / Base Cases）
4. 确定**计算顺序**（Order of Computation）：填充 DP 表的顺序——确保每当计算 `dp[state]` 时其依赖的子状态已先被计算。
5. 返回**最终状态**的值

**经典 DP 问题**：

1. **01 背包问题**（0/1 Knapsack）：n 件物品，每件有重量 w_i 和价值 v_i，背包容量 W，每件物品最多取一件 —— 最大化总价值。
   状态：`dp[i][j]` = 前 i 件物品在容量 j 下的最大价值。
   转移：`dp[i][j] = max(dp[i-1][j], dp[i-1][j-w_i] + v_i)` if j ≥ w_i else `dp[i-1][j]`。
   复杂度 O(nW)，可优化为一维 dp 数组（逆序遍历 j）。

2. **最长公共子序列**（LCS）：序列 X 和 Y 的最长公共子序列长度。
   状态：`dp[i][j]` = X[0..i-1] 和 Y[0..j-1] 的 LCS 长度。
   转移：若 X[i-1] == Y[j-1] 则 `dp[i][j] = dp[i-1][j-1] + 1`；否则 `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`。

3. **最长递增子序列**（LIS）：序列中严格递增的子序列的最大长度。
   转移：`dp[i] = max(dp[j] + 1)` for all j < i 且 nums[j] < nums[i]。O(n²)。可优化至 O(n log n) 用耐心排序法。

4. **编辑距离**（Levenshtein Distance / Edit Distance）：将字符串 A 转化为 B 的最少单字符编辑数（插入/删除/替换）。
   状态：`dp[i][j]` = A[0..i-1] 和 B[0..j-1] 的最小编辑距离。
   转移：若 A[i-1] == B[j-1] 则 `dp[i][j] = dp[i-1][j-1]`；否则 `dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])`。

5. **硬币找零**（Coin Change）：无限量供应的硬币面额 coins[]——组成金额 amount 的最少硬币数。转移：`dp[a] = min(dp[a - coin] + 1)` for each coin。

**空间优化技巧**：
- 滚动数组（Rolling Array）：二维 DP 中若 `dp[i][j]` 只依赖于 `dp[i-1]` 行，可以用两个一维数组或多个变量
- 原地修改：在原始输入矩阵上直接存储 DP 状态（如最小路径和问题）
- 状态压缩：用位掩码表示集合——如旅行商问题的 dp[mask][city]（子集 DP）

## 关键结论

- 重叠子问题是 DP 区别于分治的关键——分治的子问题是不重叠的（如归并排序）；DP 的子问题重叠
- Memoization 和 Tabulation 互为拓扑排序逆序——自顶向下回溯值，自底向上正向构建
- DP 的核心难点是状态定义和转移方程——代码通常短但推导不简单
- 不是所有有"最大""最优"词的问题都是 DP——贪心、最大流、匹配等有各自的适用条件

## 易错点

1. 状态数量过多导致内存溢出——二维 `dp[10^5][10^5]` 不可行，需要维度压缩或改用其他算法
2. 忘了基况初始化——dp 数组的默认值（0/负无穷）不正确导致错误结果
3. 遍历方向和依赖顺序不一致——计算 `dp[i]` 时它所需的子状态未计算
4. 01 背包降一维后的 j 正序遍历出错（正序是非 01 的完全背包）——物品被重复选用

## 例题

**例题1**：给定一个 m×n 棋盘，从左上角走到右下角，每次只能右移或下移一格。求最小路径和（格子有正值权重）。

**解答**：`dp[i][j]` = 走到 (i,j) 的最小路径和。转移：`dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])`。基况：`dp[0][0] = grid[0][0]`。计算顺序：按行或列顺序均可。空间可优化为 O(n)（仅需上一行结果 + 当前行）。

**例题2**：字符串 "abcde" 和 "ace" 的 LCS 长度和 DP 表的填充过程。

**解答**：
```
    ∅  a  c  e
∅   0  0  0  0
a   0  1  1  1
b   0  1  1  1
c   0  1  2  2
d   0  1  2  2
e   0  1  2  3
```
LCS 长度 = 3（"ace"）。递推：
- (a,a) 匹配 → 1; (a,c) 不匹配→max(上,左)=max(1,1)=1
- (c,c) 匹配 → dp[a][c]+1=1+1=2; (e,e) 匹配 → dp[c][c]+1=2+1=3

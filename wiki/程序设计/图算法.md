## 核心定义

图算法（Graph Algorithms）是处理图数据结构（顶点和边）的算法集合。图可以表示网络、关系、流程、依赖等丰富信息——图算法在社交网络、导航系统、编译器优化、电路设计、推荐系统中有广泛应用。

### 图的表示

- **邻接矩阵**（Adjacency Matrix）：V×V 矩阵，`M[i][j] = 1（或权重）` 若存在边 i→j。空间 O(V²)，边查询 O(1)。适合稠密图。
- **邻接链表**（Adjacency List）：每个顶点维护一个邻接顶点（和权重）的列表。空间 O(V+E)，边查询 O(degree(v))。适合稀疏图。
- **边列表**（Edge List）：简单存储所有边的集合（from, to, weight）。适合仅需遍历所有边的算法（Kruskal, Bellman-Ford）。

### 图的遍历

1. **深度优先搜索**（DFS）：从起点出发，沿一条路径走到底遇到死路则回溯尝试其他路径。使用栈（递归或显式栈）。O(V+E)。应用：连通分量计数、环检测、拓扑排序、强连通分量（SCC）检测（Kosaraju / Tarjan）、桥和关节点检测。

2. **广度优先搜索**（BFS）：从起点出发按"层"遍历——先访问所有距离 1 的顶点，再访问距离 2 的顶点 etc。使用队列。O(V+E)。应用：无权图最短路径、网络爬虫、社交网络"距离"（六度分隔）、二分图检测（染色法）。

### 最短路径

3. **Dijkstra 算法**：单源最短路（非负权重图）。贪心策略：每次从优先队列中获取当前距离最小的未访问顶点，松弛其邻居。O((V+E) log V)。应用：导航系统、网络路由（OSPF）、游戏 AI 寻路（A* 是 Dijkstra 的启发式扩展）。

4. **Bellman-Ford 算法**：单源最短路（可处理负权重图）。进行 V-1 轮所有边的松弛（每轮保证传播一个边的最短路信息）。O(VE)。可检测负权环——第 V 轮仍有路径被缩短则存在负权环。应用：货币套汇（汇率乘积 → 对数 → 负环检测）。

5. **Floyd-Warshall 算法**：全对最短路（All-Pairs Shortest Path）。DP 思维：`dp[k][i][j]` = 仅通过顶点 {1..k} 作为中间点时 i→j 的最短距离。O(V³) 但常数极小（三重循环）。支持负权边（无负权环）。应用：交通网络分析、传递闭包、社交网络关系推断。

### 最小生成树

6. **Kruskal 算法**：按权重升序遍历所有边，若新边不形成环则加入 MST（使用并查集判环）。O(E log E)。适合稀疏图。

7. **Prim 算法**：从任意顶点出发，每次将最小权重出边连接新顶点加入 MST。O((V+E) log V)。适合稠密图。

### 拓扑排序

8. **拓扑排序**（Topological Sort）：有向无环图（DAG）的顶点线性排序——对所有有向边 u→v，u 排在 v 前面。算法：(a) Kahn 算法——BFS + 入度计数（不断移除入度为 0 的顶点）；(b) DFS 后序遍历（完成后时间戳降序）。O(V+E)。应用：构建系统中的依赖编译顺序、任务调度、Git 的 commit 排序、Makefile 依赖解析。

### 连通性

9. **强连通分量**（SCC, Strongly Connected Components）：有向图的最大子图——任意两顶点互相可达。算法：Kosaraju（两遍 DFS）、Tarjan（一遍 DFS + 回溯检测 low-link）。O(V+E)。应用：社交网络社区检测、垃圾网页检测、模型检查、死代码检测。

10. **并查集**（Union-Find / Disjoint Set Union, DSU）：维护不相交集合的合并（Union）和查找所属集合（Find）。路径压缩 + 按秩合并 → 均摊 O(α(n))（近似 O(1)）。应用：Kruskal MST 判环、动态连通性问题、图像像素连接组件。

## 关键结论

- BFS 和 Dijkstra 都是"优先队列 + 贪心"，但 BFS 优先距离（无权图）、Dijkstra 优先累计距离
- 图算法通常为 O(V+E) 复杂度——遍历所有顶点和边在现代硬件上可快速完成，但 E 可以非常大（稠密图 E ≈ V²）
- 负权边打破了许多算法的贪心正确性——Dijkstra 无法处理负权边，需要 Bellman-Ford
- 图算法的困难源于组合爆炸——旅行商问题（TSP）、图着色等在图论中属于 NP-complete，无已知多项式时间精确解

## 易错点

1. BFS 的距离计算错误——入队后立即标记为已访问（而非出队时），否则同一顶点多次入队导致错误距离
2. 拓扑排序中忘记处理环——Kahn 算法结束时若还有剩余顶点则存在环，忽略此检查视为成功
3. Dijkstra 与堆的使用——更新键值而非重复插入（或插入后出堆时检查是否已访问）
4. 并查集的路径压缩忘了在 Find 中实施——不压缩导致性能退化为 O(n)

## 例题

**例题1**：给定有向边列表，用 Kahn 算法判断是否存在环并输出拓扑排序。

**解答**：
```python
from collections import deque
def topological_sort(n, edges):
    adj = [[] for _ in range(n)]
    indegree = [0] * n
    for u, v in edges:
        adj[u].append(v)
        indegree[v] += 1
    q = deque([i for i in range(n) if indegree[i] == 0])
    result = []
    while q:
        u = q.popleft()
        result.append(u)
        for v in adj[u]:
            indegree[v] -= 1
            if indegree[v] == 0:
                q.append(v)
    return result if len(result) == n else []  # [] 表示存在环
```

**例题2**：Dijkstra 算法在含有负权边的图中为何失败？给出反例。

**解答**：Dijkstra 依赖"一旦顶点出优先队列（标记为永久距离），此距离就不可能再被缩短"这个性质——这仅在非负权边下成立（因为无论后续添加什么边，新距离 ≥ 当前永久距离）。反例：A→B(权重=2), A→C(权重=5), C→B(权重=-4)。Dijkstra 处理序列：A(0)→B(2,永久)→C(5,永久)；但实际 A→C→B = 1 比 2 短——违背了永久标记。Bellman-Ford 可正确处理此例。

## 核心定义

算法复杂度分析（Algorithmic Complexity Analysis）是衡量算法效率的理论框架——评估算法随着**输入规模 n** 的增长，执行时间和空间消耗的增长速率。复杂度分析不依赖于具体硬件、编程语言、编译器——它是算法本质效率的度量。

**大 O 记号**（Big-O Notation, 上界）：O(f(n)) 表示存在正数 c 和 n₀，对所有 n ≥ n₀，算法运行时间 T(n) ≤ c × f(n)。大 O 描述**最坏情况上界**——"增长率不高于 f(n)"。

**常见复杂度等级**（耗能递增顺序）：
| 记号 | 名称 | 典型场景 | n=10⁶ 时约计操作 |
|------|------|---------|------------------|
| O(1) | 常数 | 哈希查找 | 1 |
| O(log n) | 对数 | 二分搜索 | ~20 |
| O(log² n) | 对数平方 | 某些高级树操作 | ~400 |
| O(√n) | 根号 | Grover 量子搜索 | 1000 |
| O(n) | 线性 | 遍历数组 | 10⁶ |
| O(n log n) | 线性对数 | 归并排序/堆排序 | ~20×10⁶ |
| O(n²) | 平方 | 冒泡/插入排序 | 10¹² |
| O(n³) | 立方 | Floyd-Warshall 全对最短路 | 10¹⁸ |
| O(2ⁿ) | 指数 | 子集生成 | 极大 |
| O(n!) | 阶乘 | 排列生成 | 极大 |

**其他记号**：
- Ω (Big-Omega, 下界)：最好情况保证——"增长率不低于 g(n)"。Ω(n) 意味着至少需要查看每个输入一次。
- Θ (Theta, 精确界)：同时 O 和 Ω——算法的增长率精确等于某个函数。
- o (小-o, 上界非渐进紧)：增长率严格慢于 g(n)。如 2n = o(n²) 但 2n² ≠ o(n²)。
- ω (小-omega, 下界非渐进紧)：增长率严格快于 g(n)。

**复杂度分析方法**：

1. **加法规则**：顺序执行的操作相加。O(f(n)) + O(g(n)) = O(max(f(n), g(n)))。两个循环依次运行 O(n) + O(n²) = O(n²)。

2. **乘法规则**：嵌套操作相乘。外层 O(n) 循环 * 内层 O(log n) = O(n log n)。

3. **递归算法分析**：
   - **主定理**（Master Theorem）：对于 T(n) = aT(n/b) + f(n)，比较 f(n) 与 n^(log_b a) 的增长：
     - 若 f(n) = O(n^(log_b a - ε))，则 T(n) = Θ(n^(log_b a))
     - 若 f(n) = Θ(n^(log_b a))，则 T(n) = Θ(n^(log_b a) * log n)
     - 若 f(n) = Ω(n^(log_b a + ε)) 且满足正则条件，则 T(n) = Θ(f(n))
   - **递归树法**（Recursion Tree）：将递归展开为树形结构，累加每层代价
   - **代入法**（Substitution Method）：猜测解并用数学归纳法证明

4. **均摊分析**（Amortized Analysis）：不是单次操作的最坏情况而是**一系列操作的平均成本**。动态数组（Vector）的倍增插入——单次扩容为 O(n)，但经 N 次插入共执行约 N+(N+N/2+N/4+...)≈3N 个操作 → 均摊 O(1)。均摊分析方法：(a) 聚合分析（Aggregate）：总成本 / 操作次数；(b) 记账法（Accounting Method）：每个操作存入"信用"预付未来昂贵操作；(c) 势能法（Potential Method）：定义一个势能函数衡量"储蓄工作量"。

**空间复杂度**：除输入外算法使用的额外存储空间。原地算法（In-place）O(1) 额外空间——如堆排序 O(1)、快速排序 O(log n) 递归栈、归并排序 O(n)。

**实际 vs 理论考量**：
- 常数因子被大 O 隐藏——O(100n) 和 O(n) 在 n 小时差距显著（如 100 vs 500n 的实际数据）
- 缓存效率——O(n²) 的矩阵行序优先遍历远快于列序优先遍历（空间局部性）
- 分析的最坏情况 vs 平均情况——快速排序最坏 O(n²) 但平均 O(n log n) 且实际中通常快于堆排序（常数因子更小 + 缓存友好）

## 关键结论

- 大 O 是分析算法可扩展性的工具——当 n 翻倍时运行时间如何增长是最核心的问题
- O(n) 的缓存友好算法可能在实际性能上优于 O(log n) 的缓存不友好算法——复杂性理论有盲区
- 主定理并非万能——仅覆盖 T(n) = aT(n/b) + f(n) 形式，不能处理 T(n) = T(n-1) + n（用代入法）
- **指数时间复杂度的算法在 n>30 时实际上不可行**——这定义了"易处理问题"和"难处理问题"的边界（P vs NP）

## 易错点

1. 混淆 O 和 Θ——"算法是 O(n²)"不排除它也可以是 O(n)（上界宽松），而 Θ(n²) 是精确的
2. 将 O(2n) 写成 O(2ⁿ)——2n 是 O(n)，2ⁿ 是 O(2ⁿ)，前者是线性后者是指数，天壤之别
3. 忽略递归算法中递归调用栈的空间——归并排序空间 O(n)（辅助数组）+ O(log n) 递归栈，但递归栈的 O(log n) 被 O(n) 掩盖
4. 均摊分析不等于平均情况——均摊是确定性分析，平均情况是概率分析

## 例题

**例题1**：分析以下代码的时间复杂度：
```python
for i from 1 to n:
    j = 1
    while j < n:
        j *= 2
```

**解答**：外层循环 n 次。内层 while：j 每次 ×2，从 1 增长到 n，执行次数 = ⌈log₂ n⌉ ≈ O(log n)。总复杂度 = O(n) × O(log n) = O(n log n)。

**例题2**：用主定理分析归并排序 T(n) = 2T(n/2) + n，并指出 a, b, f(n) 的值。

**解答**：a=2（两个子问题），b=2（每个子问题规模为 n/2），f(n)=n（合并步骤）。n^(log_b a) = n^(log₂ 2) = n¹ = n。比较 f(n)=n 与 n^(log_b a)=n——它们是同一量级 Θ(n)。适用于主定理情况 2：T(n) = Θ(n * log n)。

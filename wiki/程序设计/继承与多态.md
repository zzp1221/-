## 核心定义

继承（Inheritance）和多态（Polymorphism）是面向对象编程的两个核心支柱。继承使子类复用父类的代码和接口定义；多态使相同接口操作不同类型的对象产生不同的行为。

### 继承

子类从父类（基类/超类）自动获得其属性和方法，并可添加新的或重写已有方法。继承在语言间的差异显著：

**单继承 vs 多继承**：
- 单继承（Java, C#, Kotlin）：一个类只能继承一个父类——避免菱形问题（Diamond Problem）的歧义。可用接口补充实现多"继承"的行为。
- 多继承（C++ 允许直接多继承、Python 支持）：一个类可继承多个父类——处理歧义需 MRO（Method Resolution Order, Python 的 C3 线性化算法、C++ 的虚拟继承以解决菱形问题）。

**继承的访问控制**（C++/Java 细化）：
- `public` 继承（is-a 关系）
- `private`/`protected` 继承（C++：实现继承/限制）——基类的公共接口成为子类的私有/保护成员
- `final` 类（Java, C#, C++11 `final`）：不能被继承——防止子类破坏精心的设计（如 `String` 是 final）

**继承的隐式行为**：
- 构造函数调用链：子类构造前先调用父类构造函数（初始化基类部分），析构时逆序
- 向上转型（Upcasting）：子类对象可赋给父类引用——类型安全
- 向下转型（Downcasting）：父类引用转换为子类引用——需要显式检查（`instanceof` + 强制转换，运行时可能抛 ClassCastException）

### 多态

多态允许不同类的对象以统一的接口被操作，产生各自特定的行为。

**子类型多态**（Subtype Polymorphism / Dynamic Dispatch）：通过父类引用调用方法时，实际运行的是运行时对象实际类型对应的方法——而非引用类型的方法。这是 OOP 多态的核心。

**实现机制**：虚表（Vtable / Virtual Method Table）——每个多态类有一个虚函数表，存储该类中所有虚函数（可被重写的方法）的地址。对象中隐藏一个指向虚表的指针（vptr）。调用虚函数时：根据 vptr→虚表→偏移量查找函数地址→调用。虚表查找是一次额外间接寻址（运行时成本较低）。

**多态各语言的关键字和机制**：
- Java/C#：默认方法是虚的（可被重写）。`@Override` 标注。`final`/`sealed` 禁用重写。
- C++：需 `virtual` 关键字使方法参与动态分派，否则为非虚（编译时绑定）。析构函数若可继承则应 virtual。`override` (C++11) 检查是否真的重写了基类虚函数。
- Python：所有方法默认是虚的（`self` 绑定到实际对象的类型），不需要关键字。鸭子类型（不继承也可多态——只要有同名方法）。
- Go：无继承——通过接口实现多态（结构类型——任何类型实现了接口方法即满足）。方法集决定类型实现哪些接口。
- Rust：无继承——通过 Trait 对象实现动态分派（`Box<dyn Trait>` / `&dyn Trait`）。静态分派用泛型 + Trait 约束。

**重载 vs 重写**：
- **重载**（Overload）：同一作用域内多个同名函数，不同的参数列表——编译时决定调用哪个（编译时多态）
- **重写**（Override）：子类重新定义父类的虚函数，运行时根据实际对象类型调用（运行时多态）

**开闭原则（OCP）和多态的关系**：多态是实现"对扩展开放、对修改关闭"的核心语言特性——新的行为通过添加子类实现，已有代码无需修改即可使用新行为了（只要它们依赖父类接口）。

## 关键结论

- 继承是"白盒复用"——子类可见父类的内部实现，破坏了封装。组合是"黑盒复用"——通过持有另一对象的引用并调用它的方法。优先组合而非继承
- 虚函数的核心价值：在改变行为时不需修改调用方的代码（Dependency Inversion 的基础）
- 多态在编译时和运行时架起了桥梁——编译时看到的是接口（父类型），运行时执行的是实现（子类型）
- Java 的 `@Override` 和 C++ 的 `override` 关键字不是多余的——它消除重写失败（拼错签名）的风险

## 易错点

1. Java 子类中"重载"父类方法而非重写——`@Override` 标注的编译错误可避免签名不同的错误
2. 多态中父类析构函数非虚（C++ 不标注 virtual）——通过父类指针删除子类对象导致子类部分不析构（资源泄漏）
3. 继承破坏了父类封装——子类依赖父类的实现细节，父类的改动可影响子类
4. 不要混淆 C++ 的"隐藏"（非虚函数子类定义同名方法遮盖父类的方法——根据引用类型调用，非虚）和"重写"（虚函数动态分派）

## 例题

**例题1**：C++ 中 const 成员函数能否被非 const 成员函数重载？解释原理。

**解答**：可以——`void foo() const` 和 `void foo()` 是不同的签名（因为隐含 `this` 指针的类型不同——const 成员函数的 `this` 是 `const X*`，非 const 的 `this` 是 `X*`）。编译器基于对象是否为 const 决定调用哪个版本：
```cpp
class Demo {
public:
    void f() const { std::cout << "const\n"; }
    void f() { std::cout << "non-const\n"; }
};
Demo d; d.f();        // "non-const"
const Demo& cd = d; cd.f();  // "const"
```

**例题2**：以下是 Java 代码，`result` 的值是多少？

```java
class A { int val() { return 1; } }
class B extends A { int val() { return 2; } }
class C extends B { int val() { return 3; } }
A obj = new C();
int result = obj.val();
```

**解答**：结果为 3。Java 所有方法默认为虚方法。`obj` 声明类型为 A，实际对象为 C，调用 `val()` 由 JVM 根据虚表解析到 C 的 `val()` 方法（按虚表查找——实际执行 C::val）。

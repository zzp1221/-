## 核心定义

递归（Recursion）是函数直接或间接调用自身的编程技术。递归将原问题分解为规模更小的同类型子问题，直到到达可直接解决的基况（Base Case），然后组合子问题的解得到原问题的解。递归的正确性建立在数学归纳法之上——基况对应归纳基础，递归步骤对应归纳假设。

**递归的两个基本要素**：
1. **基况**（Base Case）：最简单的情况，无需递归可直接求解，终止递归的锚点。
2. **递归步骤**（Recursive Step）：将问题规模减小并向基况逼近，假设子问题的解已得到并据此构建原问题的解。

**递归的经典例子**：
- 阶乘：`fact(n) = n * fact(n-1)`, base: `fact(0) = 1`
- 斐波那契：`fib(n) = fib(n-1) + fib(n-2)`, base: `fib(0) = 0, fib(1) = 1`
- 汉诺塔：`hanoi(n, from, to, aux) = hanoi(n-1, from, aux, to) + move(n, to) + hanoi(n-1, aux, to, from)`, base: `n=1`
- 二叉树遍历：前序/中序/后序都是天然的递归算法

**递归 vs 迭代**：任何递归算法都可以通过显式栈转化为迭代算法（编译器自动做了这个转化——通过调用栈）。递归代码更清晰但每次调用有函数调用开销和栈帧分配。栈溢出风险：递归深度超过可用栈空间（通常 1-8 MB）导致崩溃。Python 默认递归深度限制为 1000，可通过 `sys.setrecursionlimit` 调整。

**尾递归**（Tail Recursion）：递归调用是函数的最后一个操作——`return func_rec(args)`。尾递归可以通过重用当前栈帧而无需新建栈帧（尾递归优化 / TCO）——编译器将尾递归转为循环消除栈溢出且节省空间。Scheme/Haskell 保证 TCO，Java 不保证，C/C++ 取决于优化级别，Rust 在某些条件下执行 TCO。

**分治算法**（Divide and Conquer）：递归的算法设计范式——将问题(1)分解（Divide）为若干个互不相交的相同性质的子问题，(2)解决（Conquer）子问题（递归或直接），(3)合并（Combine）子问题的解。分治算法的复杂度由主定理描述。

**经典分治算法**：
1. **归并排序**（Merge Sort）：数组分为两半 → 递归排序两半 → 合并有序两半。T(n) = 2T(n/2) + O(n) = O(n log n)。稳定、适合外部排序（磁盘）。
2. **快速排序**（Quick Sort）：选择 pivot 将数组分为两个部分（小于 pivot / 大于 pivot）→ 递归排序两部分。T(n) = T(k) + T(n-k-1) + O(n)。平均 O(n log n)，最坏 O(n²)（pivot 始终最小值）。
3. **二分搜索**（Binary Search）：有序数组中间元素与目标比较 → 递归在左半或右半搜索。T(n) = T(n/2) + O(1) = O(log n)。
4. **Karatsuba 乘法**：大整数乘法，用三个 n/2 位乘法替代四个。T(n) = 3T(n/2) + O(n) = O(n^log₂3) ≈ O(n^1.585)。
5. **Strassen 矩阵乘法**：将两个 n×n 矩阵相乘用 7 个子矩阵乘法替代常规的 8 个。T(n) = 7T(n/2) + O(n²) = O(n^log₂7) ≈ O(n^2.807)。

**递归树分析**（Recursion Tree）：将递归展开为树形结构——每个节点代表一个递归调用的工作（不包括递归调用的子问题工作），树的深度是递归最大深度——所有节点工作总和是算法复杂度。

## 关键结论

- 递归不是免费的——每递归调用一层消耗栈空间，深度过大可能导致栈溢出
- 尾递归是递归和迭代之间的桥梁——TCO 支持的编译器使尾递归有与循环相同的空间复杂度
- 分治法对"可并行化"问题特别有利——子问题独立互不依赖，可分布式执行（如并行归并排序、MapReduce）
- 记忆化（Memoization）可消除递归中的重叠子问题——存储已计算结果避免重复递归调用

## 易错点

1. 忘记基况导致无限递归——函数永远调用自己直到栈溢出
2. 递归步骤不是严格向基况逼近——`fact(n) = n * fact(n-1)`是正确的（n 递减），`fact(n) = n * fact(n+1)`永远不到基况
3. 尾递归优化的前提是严格意义上的最后一个操作是递归调用——`return n * fact(n-1)`不是尾递归（乘法仍需递归返回后的结果）
4. 重叠子问题的重复计算——朴素递归 `fib(n)` 的时间复杂度是 O(2ⁿ) 指数（重复计算大量子问题），添加 memoization 降到 O(n)

## 例题

**例题1**：分析以下递归函数的时间复杂度：
```python
def T(n):
    if n <= 1: return 1
    return 2 * T(n // 2) + n
```

**解答**：递归关系式 T(n) = 2T(n/2) + n。主定理：a=2, b=2, f(n)=n。n^(log_b a) = n¹ = n。f(n)=n 与 n^(log_b a)=n 同一级别 → 主定理第 2 种情况：T(n) = Θ(n log n)。

**例题2**：将递归函数改写为尾递归或迭代形式：
```python
def sum_list(lst):
    if not lst: return 0
    return lst[0] + sum_list(lst[1:])  # 非尾递归
```

**解答**：尾递归写法（使用累加器）：
```python
def sum_list_tail(lst, acc=0):
    if not lst: return acc
    return sum_list_tail(lst[1:], acc + lst[0])
```
迭代：`total = 0; for x in lst: total += x; return total`

## 核心定义

高阶函数（Higher-Order Function, HOF）是接受一个或多个函数作为参数、或返回一个函数作为结果的函数。闭包（Closure）是一个函数及其引用的外部环境变量（自由变量）的组合——"带着环境走的函数"。

### 高阶函数

高阶函数是函数式编程的基础——将函数视为与其他值（整数、字符串）同等的"一等公民"（First-Class Citizen）：
- 可以赋值给变量
- 可以作为参数传递给其他函数
- 可以作为函数的返回值
- 可以存储在数据结构中

**经典的 HOF 三元组**：
1. **map**（变换）：对集合中每个元素应用函数得到新集合。`[1,2,3].map(x => x * x) → [1,4,9]`
2. **filter**（筛选）：保留谓词为真的元素。`[1,2,3,4].filter(x => x % 2 == 0) → [2,4]`
3. **reduce/fold**（归约）：将集合通过二元函数归约为单值。`[1,2,3,4].reduce((acc, x) => acc + x, 0) → 10`

**函数组合**（Function Composition）：将多个简单函数组合成复杂函数。`compose(f, g)(x) = f(g(x))` —— 从右向左执行。管道 `pipe(f, g)(x) = g(f(x))` —— 从左向右执行。组合的重要前提是——类型对接：g 的输出类型与 f 的输入类型匹配。

```javascript
const double = x => x * 2;
const addOne = x => x + 1;
const pipe = (...fns) => x => fns.reduce((v, f) => f(v), x);
const doubleThenAddOne = pipe(double, addOne);
doubleThenAddOne(5);  // 11
```

**柯里化**（Currying）和**偏应用**（Partial Application）：
- 柯里化：将 `f(a, b, c)` 转换为 `f(a)(b)(c)`——每次接受一个参数并返回一个新函数等待下一个参数
- 偏应用：预先固定一部分参数，剩余参数等待调用——`add(1)(*remaining_args)` => 偏应用返回 `addFive`

### 闭包

闭包是函数创建时的作用域快照——函数可以"记住"并访问其被创建时所在的词法作用域中的变量，即使这个函数在其原始作用域之外被调用。

```javascript
function makeCounter() {
    let count = 0;          // 被闭包捕获的外部变量
    return function() {     // 返回的闭包持有 count 的引用
        count += 1;
        return count;
    };
}
const counter = makeCounter();
counter(); // 1
counter(); // 2  ← count 存活在闭包环境中，即使 makeCounter 已返回
```

**闭包的实现原理**：编译器/解释器在分析到内部函数引用了外部变量时，将外部变量分配到堆上（而非栈帧），并在内部函数的隐式环境（Environment Record）中存储这些变量的引用——当外部函数返回后这些变量仍被内部闭包引用，存留于堆上。

**闭包的应用**：
1. 数据隐私和封装——JS 模块模式（在 IIFE 中保护私有状态，返回公开 API）
2. 回调——`setTimeout(() => console.log(data), 1000)` 闭包让 callback 记住 data
3. 函数工厂——生成定制函数（`const double = multiplyBy(2)`）
4. 记忆化（Memoization）——缓存函数返回值避免重复计算（闭包持有缓存 Map）

**闭包的陷阱**：
- 循环中的闭包捕获循环变量——经典坑：setTimeout 循环中所有回调打印相同的最终值（var 的原因）。解决方案：`let`（块级作用域）或 IIFE 固化当前值
- 内存泄漏——闭包无意中持有了大对象的引用阻碍 GC 回收

### 各语言支持

- **JavaScript**：全面支持——函数一等公民、箭头函数（lambda）、闭包是 JS 最核心的语言特性
- **Python**：一等函数 + lambda（单行匿名函数）+ `functools` 模块（reduce, partial, wraps）
- **Java**：Java 8+ lambda + `java.util.function` 包（`Function<T,R>`, `Predicate<T>`, `Consumer<T>`, `Supplier<T>`, `BiFunction<A,B,R>`），Stream API
- **Rust**：闭包（`|x| x + 1`）可捕获环境变量（通过借用、可变借用、移动所有权三种模式的 Fn/FnMut/FnOnce trait）
- **C++11+**：lambda（`[capture](params) -> ret { body }`），`std::function` 类型擦除封装
- **Go**：函数一等公民 + 闭包（匿名函数可捕获外部变量），无 lambda 语法（需 `func` 关键字）

## 关键结论

- 高阶函数 = "将行为参数化"——接受不同的函数作为行为参数，避免重复控制结构
- 闭包 = "带着环境走的代码"——不仅保存了逻辑也保存了数据，是 Object 的轻量备选（"Closures are a poor man's objects; objects are a poor man's closures"）
- 柯里化和偏应用使函数更可复用和可组合——预先注入通用行为，留下变化部分
- 三元组 map/filter/reduce 是声明式数据处理的通用语言——多范式语言也广泛使用它们

## 易错点

1. JavaScript `var` + 闭包循环捕获的错误——`for (var i=0; i<3; i++) { setTimeout(() => console.log(i), 100); }` 输出 3,3,3 而不是 1,2,3
2. Python lambda 是单行限制——不适用于多行逻辑（多行定义常规函数并赋给变量）
3. Rust 闭包的 Fn/FnMut/FnOnce 捕获模式区别——应选择最宽松的模式（Fn 优先于 FnMut 优先于 FnOnce）
4. 过早闭包化——过度使用闭包使代码难以调试（环境隐藏在编译的闭包结构中）

## 例题

**例题1**：用 JavaScript 和 Python 各实现一个 `once` 高阶函数——保证包装的函数仅执行一次，后续调用返回第一次的结果。

**解答**：
JavaScript:
```javascript
function once(fn) {
    let called = false, result;
    return function(...args) {
        if (!called) {
            result = fn.apply(this, args);
            called = true;
        }
        return result;
    };
}
```
Python:
```python
def once(fn):
    called, result = False, None
    @functools.wraps(fn)
    def wrapper(*args, **kwargs):
        nonlocal called, result
        if not called:
            result = fn(*args, **kwargs)
            called = True
        return result
    return wrapper
```

**例题2**：解释 Rust 闭包的三种 trait Fn/FnMut/FnOnce 及其区别和适用场景。

**解答**：
- `FnOnce`：闭包最多被调用一次——获取捕获变量的所有权（move）。适用：`thread::spawn` 需要闭包被发送到另一线程获取变量的所有权。
- `FnMut`：闭包可被调用多次，并且可以修改捕获的变量——可变借用捕获。适用：`Iterator::map` 中的可变累积器、sort_by 的比较器。
- `Fn`：闭包可被调用多次且不修改捕获变量——不可变借用或 Copy 捕获。适用：纯函数变换、懒惰求值。
选择顺序（最少要求）：使用 `Fn`（最通用、对调用方最灵活）；如需可变借用使用 `FnMut`；需所有权转移则 `FnOnce`。

## 核心定义

搜索（Search）是在问题空间中系统地探索候选解以找到满足约束的解（或证明无解）的过程。回溯（Backtracking）是一种系统化搜索解空间的通用算法模板——它逐步构建候选解，当发现当前部分候选解无法完成有效解时（违反约束），立即放弃该部分解并回溯到上一个决策点尝试另一个选择。回溯可看作是"带剪枝的深度优先搜索"。

**回溯的核心思想**：
- **增量构建**（Incremental Construction）：逐变量、逐部分构建解
- **及早放弃**（Early Pruning / Forward Checking）：一旦检测到当前部分解不可行即可回溯（避免了遍历整个子树）
- **回溯**（Backtracking）：撤销最近的决策，恢复状态，尝试下一选项

不回溯就是暴力穷举（Brute-Force Enumeration）——回溯通过剪枝将指数级搜索空间缩小到可管理的规模。

**回溯算法框架**：
```
def backtrack(state, current):
    if is_solution(current):
        record_solution(current)
        return
    for choice in available_choices(state, current):
        if is_valid(choice, current):        # 剪枝条件
            make_choice(state, choice)
            backtrack(state, current + choice)
            undo_choice(state, choice)       # 回溯
```

**经典回溯问题**：

1. **N 皇后问题**（N-Queens）：在 N×N 棋盘上放置 N 个皇后互不攻击（不同行、列、对角线）。搜索空间 N^N，回溯剪枝可极大缩小。典型解法：逐行放置（深度 N），每行尝试 N 列但立即检查冲突。

2. **子集生成**（Subset / Power Set）：给定集合，生成所有子集。状态空间树：每个元素有"选"和"不选"两种选择——生成 2^n 个叶子。回溯可不显式建树即产生所有子集。

3. **排列生成**（Permutations）：所有元素的排列。状态空间树全搜索 n! 个叶子——典型回溯用法使用 `used` 标记已选元素。

4. **数独求解**（Sudoku Solver）：在 9×9 网格上填数字 1-9 满足行列和 3×3 宫约束。回溯从第一个空格开始，尝试 1-9（先检查约束），填入后递归到下一空格。若遇死路则回溯换数字。

5. **迷宫 / 图的哈密顿路径**：在最坏情况下的指数时间回溯仍然是目前的最佳通用算法（NP-complete）。

**剪枝**（Pruning）是回溯效率的关键——消除不可能产生解的分支：
- **可行性剪枝**（Feasibility Pruning）：当前部分解已违反约束 → 剪枝（如 N 皇后列冲突）
- **最优性剪枝**（Optimality Pruning）：当前部分解即使扩展完成也已不可能优于已找到的最佳解 → 剪枝（分支定界法，Branch and Bound）
- **对称性剪枝**（Symmetry Pruning）：利用问题的对称性避免搜索本质上等价的解（如 N 皇后的第一行仅需搜索前半列）
- **启发式搜索顺序**（Heuristic Ordering）：先尝试"最有希望"的选择——更早找到好解可为最优性剪枝提供更强上界

**分支定界法**（Branch and Bound）是回溯 + 最优性剪枝——用于优化问题（不只是找可行解，还要找最优解）。维护一个全局最优值（上界或下界），用当前节点可能达到的"最佳可能"（界函数）进行比较——若当前节点即使扩展完成也不可能超越当前全局最优，则剪枝丢弃整棵子树。分支定界常用于整数规划（ILP）、TSP（旅行商问题）。

**BFS 相关搜索**：
- **广度优先搜索**（BFS）是逐层扩大搜索——适合解靠近起点的空间或需要找最短步骤的问题（无权图的最短路径）
- **双路 BFS**（Bidirectional BFS）：同时从起点和目标状态搜索，在中间相遇，将搜索空间从 O(b^d) 降至 O(2×b^(d/2))
- **迭代加深 DFS**（Iterative Deepening DFS, IDDFS）：结合 DFS 的空间优势和 BFS 的最短路径保证——逐级增加 DFS 深度限制直到找到解

## 关键结论

- 回溯的时间复杂度仍然是指数级（最坏）——剪枝仅在实践意义上使得问题可解，不改变理论最坏复杂度
- 搜索顺序极大影响回溯性能——将约束最紧的变量放前优先赋值（First-Fail Principle / 最少剩余值启发）
- 回溯是元算法模板——可用位掩码、布尔数组、递归枚举等不同机制实现
- 绝大多数 NP-complete 问题的最优解只能通过回溯/分支定界（或现代 SAT/SMT Solver、ILP Solver）找到 —— 无已知的多项式时间算法

## 易错点

1. 回溯后忘记撤销状态（undo）——导致后续选择看到被"已选"错误标记的数据（代码见但未出）
2. Python 中使用列表的可变默认参数做回溯状态——`def backtrack(state=[])` 不可用（需要显式复制或在调用处新建）
3. 剪枝条件不完整——没有覆盖所有可行约束，导致搜索无效分支或（更糟）剪掉了可行解
4. 不限制递归深度——某些问题可能无限递归（如迷宫中不标记已访问节点），需要显式深度限制或 visited 集合

## 例题

**例题1**：用回溯算法解 N=4 的皇后问题并画出解空间树（前两层）。

**解答**：
```
行1: 尝试列0 —— 可用
  行2: 尝试列0 —— 冲突(同列) 剪枝
  行2: 尝试列1 —— 冲突(对角) 剪枝
  行2: 尝试列2 —— 可用
    行3: 尝试列0 —— 可用
      行4: 尝试列0 —— 冲突 → 回溯到行3
      行4: ... 所有尝试都冲突 → 回溯到行2
    行3: ...
  行2: 尝试列3 —— 可用
    ...
行1: 尝试列1 —— 可用
  → 最终得到一个解组合
```
4 皇后问题共有 2 个基本解（旋转/镜像等价于 4×4 标准棋盘）。

**例题2**：编写回溯函数生成集合 {1,2,3} 的所有子集。不使用标准库。

**解答**：
```python
def subsets(nums):
    result = []
    def backtrack(start, current):
        result.append(list(current))  # 记录当前子集
        for i in range(start, len(nums)):
            current.append(nums[i])     # 选择
            backtrack(i + 1, current)   # 递归
            current.pop()               # 回溯/撤销
    backtrack(0, [])
    return result
# 输出：[[],[1],[1,2],[1,2,3],[1,3],[2],[2,3],[3]]
```

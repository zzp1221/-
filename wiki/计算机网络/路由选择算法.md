## 核心定义

路由选择算法（Routing Algorithm）是网络层确定数据包从源主机到目的主机所经历的路径的计算方法，其核心任务是找到一条"好"的路径（通常以最小代价、最短时延或最大带宽为目标）。路由选择算法分为两大类：全局式路由算法和分布式路由算法。全局式/链路状态算法（Link State, LS）要求每个路由器掌握完整的网络拓扑（全网链路状态数据库LSDB），独立运行Dijkstra最短路径优先算法计算最优路由——代表例子是OSPF协议。分布式/距离矢量算法（Distance Vector, DV）中每个路由器仅维护到每个目的网络的"方向"（下一跳）和"距离"（代价/跳数），通过邻居间交换路由表迭代收敛到全局最优——代表例子是RIP协议和BGP（路径矢量是DV的扩展）。两种算法各有优劣势：LS收敛快、无环路、但消息开销大（LSA洪泛需要全网传输）；DV开销小、实现简单、但收敛慢、容易产生环路。现代网络中整个互联网的路由采用层次式方法——AS内部使用IGP（OSPF/IS-IS/RIP），AS之间使用BGP，逐层路由聚合和策略控制。

## 关键结论

- Dijkstra最短路径优先算法（LS算法核心）：维护两个集合S（已找到最短路径的节点集合）和U（未找到的节点集合）。每次从U中选择距离最小的节点加入S，并用该节点更新其邻居的距离。时间复杂度O(n²)，使用二叉堆优化到O(nlogn)。每个节点都以自己为根运行该算法获得以自己为中心的最短路径树，据此填写路由表
- Bellman-Ford方程（DV算法核心）：D_x(y) = min_v { c(x,v) + D_v(y) }，即节点x到节点y的最小距离=min{x到所有邻居v 的代价 + 邻居v到y的最小距离}。当链路代价变化时，节点重新运用该方程迭代更新自己的距离表。这个迭代过程就是DV算法的收敛过程
- 无穷计数问题（Count-to-Infinity Problem）：DV算法中坏消息传播慢——当某条链路突然断开（代价变大），路由信息的更新可能无限循环：A认为通过B可达目的地（跳数=3），B认为通过A可达（跳数=3），形成路由环路。每轮更新跳数加1...无限进行直到达到跳数上限（如RIP的16跳不可达）。毒性反转可以缓解但无法根治此问题
- LS vs DV的核心对比：消息复杂度LS的LSA洪泛在所有单条消息上是O(|N|×|E|)（洪泛到全网），DV仅与邻居交换（O(|N|)消息数）。收敛速度LS能迅速（一旦全网LSA洪泛完成就收敛），DV可能经历暂时环路和慢收敛。鲁棒性LS中一个错误LSA的传播有限，DV中错误的距离表会扩散给整个网络
- 层次路由：互联网的规模使得平面路由不可行（每个路由器维护数十万条路由）。层次路由将网络组织为自治系统（AS）：同一AS内运行IGP（域内路由），AS之间通过BGP交换域间路由。BGP还做路由聚合，将一组内部详细路由汇总为一个AS级可达前缀，显著减少路由表规模

## 易错点

1. **链路状态算法在每个节点独立运行Dijkstra**：每个路由器使用相同的LSDB（整网拓扑对称），但Dijkstra以自己为根节点，所以每台路由器的最短路径树不同（根节点是自己的位置造成的）。路由表中下一跳自然就不同。

2. **DV算法中"好消息传得快、坏消息传得慢"**：如果链路故障（坏消息），DV需要多轮交换才能消除。如果没有毒性反转或触发更新等机制，无穷计数问题会让路由循环很久。这是DV的根本性缺陷，而不是某个具体协议的错误。

3. **费用/度量值的定义不统一**：距离可以是以跳数计算（RIP）、以链路带宽/时延综合作为代价（OSPF Cost）、以速率/丢包率等计算的复合度量值（EIGRP），甚至BGP的路径最长度/策略权重。不同的路由协议对应不同的度量体系，最小值在不同的定义下可能给不同的最佳路径。

4. **最小代价路径和最优路由不能完全等同**：算法找到的是最小代价路径。实践中可能因为策略原因（如避免特定国家的路由器、优选商业关系较好的对等体）而人为设定非最小代价路径——这就是策略路由的实际情况。

## 例题

**例题1**：一个6节点的网络如下图所示。请分别用Dijkstra算法（以节点A为源）和距离矢量算法（节点C参与）计算节点A到节点F的最短路径和代价。
网络：A-B:2, A-C:5, B-C:1, B-D:3, C-D:1, C-E:4, D-E:2, D-F:1, E-F:2

**解答**：Dijkstra算法（从A开始）——初始S={A}, D(B)=2, D(C)=5。选择B加入S={A,B}，更新D(C)=min(5, 2+1)=3, D(D)=2+3=5。选择C加入S={A,B,C}，更新D(D)=min(5, 3+1)=4, D(E)=3+4=7。选择D加入S={A,B,C,D}，更新D(E)=min(7, 4+2)=6, D(F)=4+1=5。选择F加入S={A,B,C,D,F}。A到F的最短代价为5，路径为A→B→C→D→F（或A→C→D→F，但A→B→C→D更短）。继续Dijkstra验证最终解：A→B(2)→C(1)→D(1)→F(1)=5，A→C(5)→D(1)→F(1)=7，所以最短路径是A-B-C-D-F代价5。

**例题2**：解释链路状态路由算法中"可靠洪泛"机制的重要性和实现原理。

**解答思路**：可靠洪泛保证所有的LSA都能被同一区域中的所有节点完整收到，使得每台路由器维护完全一致的LSDB——这是链路状态算法的前提。原理：（1）每台路由器生成LSA，为其赋予单调递增的序列号（32位循环）和老化时间；（2）LSA通过洪泛在每个邻接路由器上传播，每个收到LSA的路由器对该LSA从除入端口外的所有其他端口转发；（3）收到重复LSA时通过序列号判断新旧（更新/忽略/回滚）；（4）LSA需要LSAck确认收妥（LSU→LSA×LSAck）确保可靠性。（5）LSA老化超时（3600秒）后从LSDB中清除，避免过期信息残留。

## 代码示例

```python
import heapq

def dijkstra(graph, start):
    """Dijkstra最短路径算法"""
    dist = {node: float('inf') for node in graph}
    dist[start] = 0
    prev = {node: None for node in graph}
    pq = [(0, start)]
    
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]:
            continue
        for v, weight in graph[u].items():
            new_dist = d + weight
            if new_dist < dist[v]:
                dist[v] = new_dist
                prev[v] = u
                heapq.heappush(pq, (new_dist, v))
    
    return dist, prev

def bellman_ford(graph, start):
    """Bellman-Ford距离矢量算法"""
    dist = {node: float('inf') for node in graph}
    dist[start] = 0
    
    for _ in range(len(graph) - 1):
        for u in graph:
            for v, w in graph[u].items():
                if dist[u] + w < dist[v]:
                    dist[v] = dist[u] + w
    
    return dist

# 例题的网络拓扑
graph = {
    'A': {'B': 2, 'C': 5},
    'B': {'A': 2, 'C': 1, 'D': 3},
    'C': {'A': 5, 'B': 1, 'D': 1, 'E': 4},
    'D': {'B': 3, 'C': 1, 'E': 2, 'F': 1},
    'E': {'C': 4, 'D': 2, 'F': 2},
    'F': {'D': 1, 'E': 2}
}

dist_d, path_d = dijkstra(graph, 'A')
dist_bf = bellman_ford(graph, 'A')
print(f"Dijkstra: {dist_d}")
print(f"Bellman-Ford: {dist_bf}")
```

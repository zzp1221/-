## 核心定义

成帧（Framing）是数据链路层将网络层传来的数据包（分组）封装为数据帧的过程，其核心任务是在比特流中标记每帧的起始和结束边界，使接收方能够正确地从连续的比特流中提取出一个个完整的帧。成帧面临的关键挑战是"帧定界"（Frame Delimiting）——即在帧的数据字段中可能出现与帧定界标志相同的比特模式，导致接收方误判帧边界。解决这一问题的四种主要成帧方法为：（1）字符计数法——在帧头中用计数字段标明帧长度；（2）字符填充法——使用特殊字符作为帧首（SOH）和帧尾（EOT），数据中的同类型字符用转义字符（ESC）转义；（3）比特填充法（零比特填充）——使用01111110作为帧定界标志，发送方在数据部分每遇到5个连续的1就插入一个0，接收方每收到5个连续的1就删除后面的0；（4）物理层编码违例法——利用物理层编码中不使用的冗余信号作为帧边界。

## 关键结论

- 比特填充法（零比特填充）是HDLC和PPP协议采用的标准成帧方法，以01111110（0x7E）为帧定界标志，通过"逢5个1插0"保证数据部分不会出现标志序列，实现透明传输
- 字符计数法的严重缺陷：计数字段一旦出错，后续所有帧的边界都失去同步，且无法自动恢复，实践中很少单独使用
- 字符填充法与ASCII字符集绑定，不适应于非ASCII传输场景（如传输二进制图片/音频），且依赖于8位字符对齐，灵活性不足
- 编码违例法利用了物理层编码中的冗余信号模式，不增加额外开销，但只适用于特定的物理层编码方案（如令牌环中的差分曼彻斯特编码的J/K违例码）
- 无论采用哪种成帧方法，数据链路层都需要配合差错检测（如CRC校验）来发现传输过程中的帧损坏

## 易错点

1. **透明传输的真正含义**：透明传输是指无论上层传来的数据包含什么样的比特组合，数据链路层都能正确传输。不是指数据"看得见/看不见"，而是指任何数据组合都不会干扰帧的定界和传输协议的正常工作。如PPP协议通过比特填充实现了透明传输。

2. **误认为字符填充法和比特填充法相同**：字符填充法工作在字节层面（面向字符），使用转义字符ESC；比特填充法工作在比特层面（面向比特），使用0比特插入技术。两者实现的透明传输机制不同，适用场景也不同：前者用于PPP的异步传输模式，后者用于HDLC和PPP的同步传输模式。

3. **帧定界标志可能连续出现**：在实际通信中，两个帧之间可能连续出现多个0x7E标志字节（如PPP的空闲填充），接收方需要能正确处理这种情况——连续出现的标志字节不表示新的帧开始。

4. **忽略了帧尾也有CRC校验的关系**：成帧和差错检测是两个相关联的任务。只有在正确定界每一帧的起始和结束后，接收方才能正确计算和验证CRC。帧定界出错会直接导致CRC验错的连锁问题。

## 例题

**例题1**：采用比特填充法的HDLC协议，发送方收到网络层以下比特序列：0110111110111111010，请写出经过比特填充后实际发送的比特序列（假设帧定界标志为01111110）。

**解答**：比特填充规则：每遇到5个连续的1后就插入一个0。扫描输入序列：0110111110 111111010。序列中"011111"处5个连续的1后插入一个0（成为0111110），继续扫描"111110"处5个连续的1后插入一个0（成为1111100）。所以填充后的序列为：01101111100 111110100（实际发送时还需要前后各加01111110帧标志）。

**例题2**：比较字符填充法和比特填充法的优缺点，并说明为什么PPP协议同时支持两种填充方式。

**解答思路**：字符填充法优点包括实现简单、和ASCII文本传输自然兼容；缺点是依赖8位字符格式、效率较低（每个转义字符都要两字节表示）、无法传输任意二进制数据。比特填充法优点包括与数据格式无关（真正的透明传输）、效率高（仅特定序列才插入额外比特）；缺点是实现相对复杂（需要硬件或专门的比特处理逻辑）。PPP协议同时支持两种方式的原因是：PPP设计用于多种链路层载体，在异步串行链路上（如拨号Modem）使用字符填充（因为异步传输本身就是字符为单位的），在同步串行链路上（如SDH/SONET）使用比特填充。这种灵活性使得PPP可以在不同的物理层技术上统一配置。

## 代码示例

```python
def bit_stuffing(data_bits):
    """HDLC比特填充（零比特填充）"""
    flag = '01111110'
    stuffed = ''
    ones_count = 0
    for bit in data_bits:
        stuffed += bit
        if bit == '1':
            ones_count += 1
            if ones_count == 5:
                stuffed += '0'  # 插入0
                ones_count = 0
        else:
            ones_count = 0
    return flag + stuffed + flag

def bit_destuffing(stuffed_bits):
    """HDLC比特解填充（去零）"""
    destuffed = ''
    ones_count = 0
    i = 0
    while i < len(stuffed_bits):
        bit = stuffed_bits[i]
        destuffed += bit
        if bit == '1':
            ones_count += 1
            if ones_count == 5:
                # 跳过下一个插入的0
                i += 1
                ones_count = 0
        else:
            ones_count = 0
        i += 1
    return destuffed

# 测试
data = '0110111110111111010'
stuffed = bit_stuffing(data)
print(f"原始数据: {data}")
print(f"填充后:   {stuffed}")
print(f"解填充后: {bit_destuffing(stuffed[8:-8])}")
```

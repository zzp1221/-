## 核心定义

子网划分（Subnetting）和CIDR（Classless Inter-Domain Routing，无类域间路由）是IPv4地址管理的两种核心技术，旨在提高IP地址的利用率和路由效率。子网划分是在传统分类地址的基础上，向主机号部分"借位"来创建子网，通过子网掩码区分网络位、子网位和主机位，使得一个大的网络可以被分割为多个较小的逻辑子网。CIDR则完全抛弃了地址分类的概念，采用"网络前缀"（任意长度的网络号，以"IP地址/前缀长度"格式表示）进行地址分配和路由选择。VLSM（可变长子网掩码）支持对同一分类网络的不同子网使用不同的掩码长度，进一步细化地址分配。CIDR的超网技术（路由聚合/路由汇总）将多个连续的地址块合并为一个更大的地址块，用一个路由条目替代多个条目，有效减缓了路由表膨胀问题。

## 关键结论

- 子网掩码是一个32位的值，网络位和子网位对应的比特全为1，主机位对应的比特全为0。如/26的子网掩码=255.255.255.192（前26位为1）。网络地址=IP地址 AND 子网掩码
- 借n位创建子网时，子网数=2^n，每个子网的主机数=2^(剩余主机位)-2（减2是去掉子网地址和子网广播地址）。对于/24借3位得到/27：子网数=2^3=8，每个子网主机数=2^5-2=30
- CIDR的地址聚合/超网技术：将连续的地址块合并为一个更大的路由条目，条件是这些连续地址块的网络前缀相同。如4个/24网络（192.168.0.0/24, 192.168.1.0/24, 192.168.2.0/24, 192.168.3.0/24）可以聚合为192.168.0.0/22
- 最长前缀匹配（Longest Prefix Match）是CIDR的路由查找原则：当路由表中有多个条目匹配目的地址时，选择前缀最长（最具体）的那个条目进行转发
- VLSM允许对同一主网络的不同子网使用不同的前缀长度：如172.16.0.0/16可以下挂/24的点对点链路子网（2个地址足矣：172.16.0.0/30）和/22的大型子网（约1022台主机）

## 易错点

1. **CIDR不是对"所有地址都重新划分"**：CIDR是分配策略和路由表的组织方式，它不改变现有IP地址自身的网络号和主机号布局。同样的一个地址，在分类编制下和处理不同，在CIDR下处理也不同——但地址本身没变。
2. **路由聚合可能导致路由黑洞**：当聚合路由覆盖了部分未分配的地址块时，发往这些地址的数据包会被错误路由（因为路由器不知道它们是未分配的）。解决方法是使用更具体的路由（更长的前缀）覆盖聚合路由，或者确保聚合边界与实际地址分配完全对齐。
3. **全0和全1子网的可用性**：在早期的RFC 950中，全0子网和全1子网是禁用的（因为全0子网地址与网络地址相同，全1子网的广播地址与网络广播地址相同）。但现代操作系统和路由器普遍支持使用这两个子网（ip subnet-zero命令在Cisco IOS 12.0以上默认启用）。考试中应根据题目上下文判断。
4. **VLSM与CIDR的概念关系**：VLSM是在分类地址内部做不同长度子网划分（同一主类网络下子网长度不同）；CIDR是在分类地址之间做任意长度的前缀分配（取消了类的概念）。可以说VLSM是CIDR的子集和前身技术。

## 例题

**例题1**：某单位获得192.168.0.0/22的地址块，需要将其划分为4个大小相等的子网，分别给4个部门，每个部门约250台主机。请给出子网划分方案和每个子网的地址范围。

**解答**：/22共有2^10-2=1022个可用地址，分给4个部门每部门约250台需要8位主机位（2^8-2=254）。从/22借2位到/24：子网1：192.168.0.0/24（192.168.0.1-192.168.0.254）；子网2：192.168.1.0/24（192.168.1.1-192.168.1.254）；子网3：192.168.2.0/24（192.168.2.1-192.168.2.254）；子网4：192.168.3.0/24（192.168.3.1-192.168.3.254）。注意/24正好对应C类子网掩码255.255.255.0。

**例题2**：简述路由聚合的条件，并判断以下4个路由条目能否聚合：203.10.56.0/24, 203.10.57.0/24, 203.10.58.0/24, 203.10.59.0/24。如果能，给出聚合后的路由条目。

**解答思路**：路由聚合的条件是连续的地址块具有相同的前缀部分。判断方法：找出所有地址块IP范围的最大公共前缀。56=00111000, 57=00111001, 58=00111010, 59=00111011。它们的前22位完全一致（203.10是前16位，第三字节56-59的高6位均为001110），所以可以聚合为203.10.56.0/22。但/22覆盖的地址范围是203.10.56.0-203.10.59.255，恰好覆盖了4个/24。如果聚合前有额外的地址块（如203.10.60.0/24未分配），则需要注意不可过度聚合。

## 代码示例

```python
import ipaddress

# CIDR地址块操作
network = ipaddress.IPv4Network('192.168.1.0/24')

print(f"网络地址: {network.network_address}")
print(f"广播地址: {network.broadcast_address}")
print(f"地址总数: {network.num_addresses}")
print(f"可用主机数: {network.num_addresses - 2}")

# 判断地址归属
ip = ipaddress.IPv4Address('192.168.1.100')
print(f"{ip} 在 {network} 中: {ip in network}")

# 列出子网（VLSM分号）
# 将/22划分为4个大小相等的/24子网
super_net = ipaddress.IPv4Network('192.168.0.0/22')
subnets = list(super_net.subnets(new_prefix=24))
for i, subnet in enumerate(subnets):
    print(f"子网{i}: {subnet}, 主机范围: {subnet[1]}-{subnet[-2]}")

# 路由聚合示例
nets = [
    ipaddress.IPv4Network('203.10.56.0/24'),
    ipaddress.IPv4Network('203.10.57.0/24'),
    ipaddress.IPv4Network('203.10.58.0/24'),
    ipaddress.IPv4Network('203.10.59.0/24')
]
# 使用collapse_addresses进行路由聚合
aggregated = list(ipaddress.collapse_addresses(nets))
print(f"聚合后路由: {aggregated[0] if aggregated else '无法聚合'}")
```

## 核心定义

计算机网络性能指标是衡量和评估网络服务质量（QoS）的量化标准，是网络规划、设计、运营和优化的重要依据。主要性能指标包括：速率（数据率/比特率，单位bps）、带宽（两个含义——频域带宽指信号频率范围，单位为Hz；时域带宽指通信线路的最大传输能力，单位为bps）、吞吐量（单位时间内成功传输的实际数据量）、时延（从发送到接收的总时间，分为发送时延、传播时延、处理时延和排队时延）、时延带宽积（链路中正在传播的比特数，等于传播时延乘以带宽）、往返时间RTT（从发送到收到确认的总时间）和利用率（包括信道利用率和网络利用率）。这些指标相互关联，共同决定用户体验。

## 关键结论

- 总时延 = 发送时延（数据量/发送速率） + 传播时延（距离/信号传播速度） + 处理时延 + 排队时延。在高速网络中，发送时延很小，传播时延成为主导因素
- 带宽和速率在计算机网络中往往混用，但严格来说速率是实际的数据传输率，带宽是线路的最大传输容量
- 信道利用率与服务质量之间存在博弈关系：利用率趋近于1时，排队时延急剧增大（类似公路上的交通拥堵），通常建议骨干链路利用率不超过70%
- 吞吐量受限于端到端路径中速率最小的链路（瓶颈链路），在共享式网络中还受到并发用户数量的影响
- RTT通常由传播时延主导，卫星通信的RTT可达540ms（地球同步轨道卫星高度约36000km），对交互式应用有显著影响

## 易错点

1. **发送时延与传播时延的混淆**：发送时延 = 数据帧长度 / 发送速率，与传输距离无关；传播时延 = 链路长度 / 信号传播速度，与数据帧长度无关。向长距离链路发送小数据时，传播时延远大于发送时延。例如同步卫星链路发送1KB数据，发送时延约8μs@1Gbps，但传播时延约270ms。

2. **带宽的两个含义不分**：通信领域中的"带宽"原指信号频率范围（Hz），计算机网络中借用了这个词表示最大数据传输速率（bps）。两者通过奈奎斯特定理和香农定理建立联系。在考试中需根据上下文区分具体含义。

3. **利用率不是越高越好**：直观上认为网络利用率越高越"物有所值"，但根据排队论，当利用率ρ→1时，平均排队时延T = T0/(1-ρ)趋于无穷大。这就是为什么需要流量工程和链路冗余来保证服务质量的数学依据。

4. **吞吐量与速率的区别**：速率是物理层的标称传输速度（如100Mbps以太网），吞吐量是上层实际可达的有效数据传输率。由于协议开销（帧头、校验等）、冲突重传、流量控制等因素，吞吐量总是低于标称速率。

## 例题

**例题1**：假设需要在一条传播时延为20ms的链路上以1Gbps的速率发送一个1000字节的IP数据报。计算总时延（忽略处理时延和排队时延）。

**解答**：发送时延 = (1000×8 bit) / (1×10^9 bps) = 8×10^(-6) s = 8μs。传播时延 = 20ms = 20000μs。总时延 ≈ 20008μs ≈ 20ms。可见在此场景下，传播时延占总时延的99.96%以上，发送时延可以忽略不计。而对于1MB的数据（8000倍），发送时延变为8ms，传播时延仍然是20ms，总时延 = 28ms，两者量级相当。

**例题2**：一条链路长度为1000km，信号的传播速度为2×10^8 m/s，链路带宽为10Mbps。计算该链路的时延带宽积，并说明其物理含义。

**解答**：传播时延 = 1000km / (2×10^8 m/s) = 1000×10^3 / (2×10^8) = 5×10^(-3) s = 5ms。时延带宽积 = 5×10^(-3) s × 10×10^6 bps = 5×10^4 bit = 50kbit。物理含义：在任意时刻，该链路上正在传输的比特数最多为50kbit。换言之，第一个比特到达接收端时，发送端已经发出了50kbit的数据。时延带宽积也称为以比特为单位的"管道长度"，可用于指导TCP窗口大小的设置。

## 代码示例

```bash
# 测量网络时延和RTT
ping -c 100 www.example.com  # 发送100个ICMP包，查看平均RTT和丢包率

# 测量路径上每一跳的时延
traceroute www.example.com
mtr www.example.com  # MTR综合了ping和traceroute

# 测量网络吞吐量（需要iperf3工具）
# 服务端
iperf3 -s
# 客户端（测试30秒TCP吞吐量）
iperf3 -c server_ip -t 30

# 查看网络接口统计信息（吞吐量相关）
cat /proc/net/dev
iftop -i eth0  # 实时查看流量
```

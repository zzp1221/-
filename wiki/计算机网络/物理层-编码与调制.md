## 核心定义

物理层的编码与调制是将数字数据或模拟数据转换为适合在物理信道上传输的信号的过程。编码（Encoding）是指将数字数据转换为数字信号的过程，常用于基带传输（即直接传输数字信号而不经过载波调制）。调制（Modulation）是指将数字数据转换为模拟信号，或将模拟信号搬移到更高频段的过程，用于带通传输。典型的数字编码技术包括：不归零编码（NRZ）、曼彻斯特编码、差分曼彻斯特编码和4B/5B编码。典型的数字调制技术包括：幅度键控（ASK）、频移键控（FSK）、相移键控（PSK）以及更复杂的正交幅度调制（QAM）。编码关注的是信号波形与比特序列之间的映射关系，调制关注的是如何用载波的参数（幅度、频率、相位）来表示数字信息。

## 关键结论

- NRZ编码实现简单但存在同步问题：连续的0或1会导致接收方无法区分每个比特的起止位置。NRZ-I（不归零逆转码，遇1翻转遇0不变）在一定程度上缓解了同步问题
- 曼彻斯特编码将时钟信号隐含于数据信号中（自同步编码）：每个比特周期内有一个跳变，从高到低表示0、从低到高表示1（IEEE 802.3以太网标准），缺点是编码后信号速率翻倍，带宽利用率仅50%
- 差分曼彻斯特编码的抗干扰性更好：每个比特周期起始处有跳变表示0、无跳变表示1，中间的跳变仅用于同步。常用于令牌环网络
- QAM（正交幅度调制）结合了幅度调制和相位调制，可以在有限带宽上实现更高的数据速率。常用的有QAM-16（每个码元4bit）、QAM-64（每个码元6bit）、QAM-256（每个码元8bit），广泛用于现代通信系统（ADSL、WiFi、4G/5G）

## 易错点

1. **编码与调制的区别**：编码是将数字数据→数字信号（数字基带传输），调制是将数字数据→模拟信号（数字频带/带通传输）。两者不可混为一谈，虽然经常在同一通信系统中组合使用。例如，ADSL调制解调器中同时存在QAM调制和信道编码。

2. **曼彻斯特编码的"跳变"不是越多越好**：曼彻斯特编码的每个比特周期都有跳变，虽然解决了同步问题，但信号变化的频率翻倍，即所需带宽翻倍。这使得它在10Mbps以太网中足够使用，但在高速网络中不再适用，现代千兆以太网采用4D-PAM5等更高效的编码方式。

3. **波特率与比特率的关系**：波特率是每秒传输的码元数，比特率是每秒传输的比特数。对于二元调制（一个码元1bit），两者相等。对于多进制调制（如QAM-64每个码元携带6bit信息），比特率 = 波特率 × log2(M)，其中M为调制阶数。波特率决定所需带宽，比特率决定数据传输速率。

4. **NRZ和NRZI的零电平问题**：不归零编码不一定是全零/全一。NRZ-L（电平编码）用高电平表示1、低电平表示0。当连续出现多个0或1时电平不变，接收方无法同步时钟。

## 例题

**例题1**：采用4D-PAM5编码的千兆以太网，已知数据传输速率为1Gbps，求其码元传输速率（波特率）。

**解答**：4D-PAM5编码同时使用4对双绞线，每对线上的调制方式为PAM5（脉冲幅度调制，5电平）。每条线上每个码元可以表示log2(5)≈2.32bit的信息。千兆以太网使用125MBaud的码元速率，4对线同时传输，每条线250Mbps。总比特率 = 125MBaud × 8bit/码元（4对×2bit等效）= 1000Mbps = 1Gbps。更准确的计算方式：每条线实际采用PAM-5，使用4种符号（余下1种用于纠错），故每符号2bit，4对线每符号8bit。波特率 = 1Gbps / 8bit/符号 = 125MBaud。

**例题2**：比较曼彻斯特编码和差分曼彻斯特编码的异同，并说明各自的适用场景。

**解答思路**：相同点：两种编码都是自同步编码，每个比特周期内都有跳变，接收方可以从中提取时钟信号，不需要额外的时钟线。不同点：（1）曼彻斯特编码用跳变的方向表示数据（高→低=0，低→高=1），每个比特中间跳变既用于同步又用于表示数据；（2）差分曼彻斯特编码用跳变的有无表示数据（比特开始处有跳变=0，无跳变=1），中间的跳变仅用于同步。适用场景：曼彻斯特编码用于10Mbps以太网（10Base-T）；差分曼彻斯特编码用于令牌环网。差分编码的优点在于对信号极性的反转不敏感（如果两根信号线接反了也不会出现误判）。

## 代码示例

```python
# Python演示基本编码方式（NRZ和曼彻斯特编码的简化实现）
def nrz_encode(bits):
    """不归零编码：1->高, 0->低"""
    signal = []
    for b in bits:
        signal.extend([1] if b == '1' else [-1])
    return signal

def manchester_encode(bits):
    """曼彻斯特编码：每个比特中间有跳变
    1 -> 低到高 (前半低, 后半高)
    0 -> 高到低 (前半高, 后半低)"""
    signal = []
    for b in bits:
        if b == '1':
            signal.extend([-1, 1])  # 低->高
        else:
            signal.extend([1, -1])  # 高->低
    return signal

# 测试
bits = "10110010"
print("NRZ:", nrz_encode(bits))
print("Manchester:", manchester_encode(bits))
```

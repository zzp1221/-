## 核心定义

CSMA/CD（Carrier Sense Multiple Access with Collision Detection，载波监听多点接入/碰撞检测）是以太网局域网使用的介质访问控制（MAC）协议，用于协调多个站点共享同一广播信道的访问。其工作原理可概括为"先听后说、边说边听、冲突即停、随机重试"。具体流程为：（1）发送前先监听信道（载波监听），若信道空闲则发送，若信道忙则持续监听直到信道空闲；（2）开始发送后持续监听信道（碰撞检测），若检测到其他站点也在发送产生了碰撞，则立即停止发送并发送一个强化碰撞信号（Jam Signal）；（3）碰撞后采用截断二进制指数退避算法（Truncated Binary Exponential Backoff）决定重传等待时间，重传次数越多竞争窗口越大。CSMA/CD仅适用于半双工通信模式，现代全双工交换式以太网中已不再使用CSMA/CD。

## 关键结论

- CSMA/CD不能完全避免碰撞，只能减少碰撞的持续时间和发生的概率：碰撞窗口（争用期）等于信号在两端之间传播的最大往返时间（2τ），只有在这个窗口期可能发生碰撞
- 以太网规定最小帧长为64字节，目的之一是保证发送方在发送完毕前能够检测到碰撞：64字节的发送时间 ≥ 争用期（2τ），确保如果发生碰撞，发送方一定还在发送帧，能够监听到
- 截断二进制指数退避算法的退避时间 = 基本争用期2τ × 随机整数r，其中r从[0, 2^k-1]中均匀随机选择，k = min(重传次数, 10)，最多重传16次后放弃
- CSMA/CD的网络长度受到碰撞检测的严格限制：网络端到端距离不能超过2τ × 信号传播速度，典型的10Mbps以太网最大网段长度为2500m
- CSMA/CD属于竞争型协议，信道利用率在理想条件下最高约为1/(1+5τ/T0)（T0为帧发送时间），当负载增大时碰撞概率上升，利用率下降

## 易错点

1. **争用期2τ是基带总线两端的往返传播时延**：争用期（collision window）是站点A到站点B的单程传播时延τ乘以2（往返），不是4τ。因为A在发出数据时，B在A的数据到达前的一瞬间（τ时刻之前）发出的数据也会与A的数据碰撞，A最晚在2τ时刻能检测到该碰撞。

2. **以太网最小帧长64字节的完整原因**：除了碰撞检测的要求（发送时延≥争用期2τ），还涉及：提供一个足够大的"碰撞检测窗口"以确保所有站点都能发现碰撞；配合IP最小数据包（46字节）加上18字节以太网帧头帧尾，自然得出最小帧长64字节。

3. **强化碰撞信号（Jam）是干扰信号不是确认/否认信号**：发送方检测到碰撞后，立即停止发送数据并发出32比特的强化碰撞信号（全1序列），目的是确保所有冲突站点都能可靠检测到碰撞，而不是为了传递任何信息。收到Jam的站点知道发生了碰撞并停止发送。

4. **全双工交换式以太网不工作于CSMA/CD**：现代以太网交换机每个端口都是独立的碰撞域（或没有碰撞域），全双工模式下发送和接收使用不同的线对，不可能发生碰撞，因此CSMA/CD完全不被使用。很多人误以为所有以太网络都使用CSMA/CD。

## 例题

**例题1**：某10Mbps以太网，信号的传播速率为2×10^8 m/s，争用期为51.2μs。求该网络允许的端到端最大距离。如果A和B是网络中的两端，A在t=0时刻发送数据，B在t=1024比特时间（1比特时间=0.1μs）也检测到信道空闲而发送数据，问是否会发生碰撞？A何时能检测到？

**解答**：端到端最大距离 = 信号传播速度 × τ = 2×10^8 × (51.2×10^-6)/2 = 5120m（τ=25.6μs）。t=0时A开始发送，数据在t=25.6μs时到达B。B在t=1024×0.1μs=102.4μs时才检测信道（认为空闲）发送数据，此时A的数据已到达B并被B接收。B检测信道时只能听到A的信号（信道忙），所以B应该检测到信道忙而推迟发送，不会发生碰撞。但如果题目说B在t=0时刻检测到信道空闲——此时A的数据还未到达B，B认为空闲发送，则B的数据在τ时刻与A的数据在中途碰撞，A最晚在2τ=51.2μs时刻检测到该碰撞。

**例题2**：简述截断二进制指数退避算法的工作原理，并说明为什么重传次数增加时竞争窗口也增加。

**解答思路**：算法流程为（1）碰撞后，设当前是第k次重传（k≤15），令m=min(k,10)；（2）从整数集合[0,1,2,...,2^m-1]中随机选取一个数r；（3）退避等待时间为r×2τ个比特时间；（4）若16次重传仍失败则放弃。窗口随重传次数增加而指数扩大的设计理念：碰撞意味着同时竞争站点的数量可能较多，扩大退避窗口的随机范围降低了再次碰撞的概率，这是一种自适应的负载调节机制。当重传次数较少时（少数站点冲突），小窗口足以分散时间线；当重传次数多时（可能大量站点同时竞争），大窗口才能有效均分重试时间点。

## 代码示例

```python
import random

class CSMACD:
    def __init__(self, tau=25.6e-6):
        self.tau = tau          # 信号传播时延
        self.retry_count = 0    # 当前重传次数
        self.max_retry = 16     # 最大重传次数
    
    def truncated_binary_exponential_backoff(self):
        """截断二进制指数退避"""
        if self.retry_count >= self.max_retry:
            return None  # 放弃发送
        
        k = min(self.retry_count, 10)
        r = random.randint(0, 2**k - 1)
        wait_time = r * 2 * self.tau  # 退避时间
        self.retry_count += 1
        return wait_time
    
    def attempt_send(self, channel):
        """尝试发送"""
        if channel.is_idle():
            # 开始发送
            if not channel.collision_detected():
                # 发送成功
                self.retry_count = 0
                return True
            else:
                # 碰撞，发送Jam，执行退避
                channel.send_jam()
                wait = self.truncated_binary_exponential_backoff()
                if wait is None:
                    return False
                time.sleep(wait)
                return self.attempt_send(channel)
        return False
```

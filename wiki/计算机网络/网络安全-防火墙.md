## 核心定义

防火墙（Firewall）是部署在网络边界上的网络安全设备（硬件或软件），用于监控和控制进出网络的数据流量，根据预设的安全策略（规则集）决定是允许还是阻止特定的数据包通过。防火墙是实现网络安全的第一道防线，它将内部可信任网络与外部不可信任网络（如互联网）隔离，以此防范外部网络威胁。防火墙经历了四代演进：（1）第一代——包过滤防火墙（Packet Filter），基于IP和传输层头部信息（源/目的IP、协议、端口）静态过滤，工作在OSI第3和第4层，如iptables和传统ACL路由器；（2）第二代——状态检测防火墙（Stateful Inspection），维护连接状态表跟踪连接的完整生命周期，只允许经过验证的已建立连接的回包通过或拒绝尚未建立合法连接方向的新请求进入内网；（3）第三代——应用层网关/代理防火墙（Application Gateway / Proxy），工作在应用层，理解HTTP/FTP/DNS等协议的具体内容，可以阻止嵌入在应用数据中的恶意内容（如SQL注入攻击签名的早期检测）；（4）第四代——下一代防火墙（NGFW，Next-Generation Firewall），集成了入侵防御系统（IPS）、应用识别与控制、用户身份关联（基于AD/LDAP）、威胁智能情报和深度包检测（DPI）于一体。

## 关键结论

- 包过滤防火墙根据五元组（源IP、目的IP、源端口、目的端口、协议号）做第一层过滤，优点是速度快且透明，缺点是无法理解应用层内容和连接状态
- 状态检测防火墙维护连接状态表（跟踪每条TCP/UDP/ICMP"会话"的建立状态），自动允许"已建立"相关回包通过而不需要额外规则。它比纯包过滤更智能和更安全，是当今最常见的防火墙形态
- DMZ（De-Militarized Zone，隔离区）是将对外公开服务（Web/邮件/DNS服务器）放置在与内部网络和外部网络都隔离的单独区域中。防火墙对DMZ设置严格策略——外网可访问DMZ但不可直接访问内网，DMZ服务器被入侵后内网仍受保护
- 访问控制列表（ACL）是防火墙规则的基本表达形式：按序排列的规则列表（通常遵循"先匹配先执行"模式），每条规则由匹配条件（五元组 + 时间/用户等）和动作（允许/拒绝/日志记录）组成。规则末尾一般有默认拒绝（Deny All）原则
- 防火墙的局限性：防火墙是不可缺少但并非充分的安全措施。它无法防御来自内部的威胁、经过加密隧道的恶意流量（如VPN/HTTPS内嵌恶意载荷）、应用层漏洞利用和USB/物理层面的攻击。纵深防御（Defense in Depth）要求防火墙配合IDS/IPS、WAF（Web应用防火墙）、端点安全EDR等多层次安全体系

## 易错点

1. **防火墙不能防御内部网络威胁**：防火墙在边界上工作，来自内网内部的攻击（已入侵的内部主机、恶意USB、员工绕过策略）不在防火墙视线范围内。内部网络也需要内部防火墙（Micro-Segmentation）或零信任原则。

2. **状态检测防火墙不等于自动阻挡所有的外部连接**：状态检测跟踪的是出站连接的返回流量。一个外部发起的SYN包（新建连接）如果是向内网发的，状态表中没有对应的转换项，默认被阻止。但许多家庭路由器的防火墙默认允许所有出站流量和对应的回包——这意味着如果内网主机已被植入后门木马（如C2反弹shell），防火墙不会阻止出站的数据。

3. **NAT不是防火墙**：NAT有时被误称为防火墙因为其隐藏了内部网络拓扑。但NAT的安全性是偶然的不完全安全效应——它仅阻截外部主动进来的连接但不对出站恶意流量、应用层攻击设防。状态检测功能是以防火墙的身份工作的，不是NAT。

4. **下一代防火墙的应用识别不等于解密**：NGFW能识别HTTP、FTP、SMB等应用协议并对其载荷做特征匹配，但对于TLS加密的应用流量（HTTPS等），NGFW必须在做TLS解密代理后才能看到实际载荷——这带来了隐私问题和额外的性能开销。有些NGFW部署做TLS INSPECTION（即MITM解密检查再加密），但这需要配置用户设备信任防火墙的CA证书。

## 例题

**例题1**：公司网络由外网→防火墙→DMZ（Web Server）→防火墙→内部网络三层结构组成。请制定防火墙规则以允许全球用户访问Web（TCP 80/443），并阻止外部直接访问内部数据库（TCP 3306），同时内部网络可以访问外部互联网。

**解答**：防火墙1规则（外部→DMZ）：(1)允许源任意/目的Web服务器IP/端口80，动作允许（HTTP）；(2)允许源任意/目的Web服务器IP/端口443，动作允许（HTTPS）；(3)拒绝所有其他入站连接（Implicit Deny All）。防火墙2规则（DMZ→内部）：(1)允许源Web服务器IP/目的内网数据库IP/端口3306（如果Web服务器需要读写数据库）；(2)允许源内网任意/目的任意/端口任意（内网出站请求自由流出）；(3)拒绝所有其他入站连接。这是分层防御的典型配置，确保即使Web服务器被攻破也限制了攻击者对内部网络的进一步侵害。

**例题2**：描述防火墙对TCP连接的状态跟踪机制（简化为SYN、SYN-ACK、ACK三阶段），说明如何防止SYN Flood和无效ACK攻击。

**解答思路**：状态防火墙从客户端的SYN包开始创建连接状态表条目（标记为"SYN_SENT"状态）。如果状态表中没有匹配的转发规则允许这个SYN，它被直接拒绝。收到回复的SYN-ACK包后，状态表更新为"ESTABLISHED"并在允许列表中验证此数据包属于该有效连接。随后该条目的流量（ACK包、FIN包等）都被允许。防止SYN Flood: 一是限制状态表的大小（超SYN队列，触发SYN Cookie保护）；二是对异常多的SYN进行速率限制或自动阻断。防止无效ACK攻击：没有对应状态的随机ACK包（不匹配表中任何条目）在状态防火墙中被当作无效/盲目ACK直接丢弃，不会被传递到内部服务器。

## 代码示例

```bash
# Linux iptables 防火墙规则示例
# 默认策略
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# 允许已建立和相关连接的回包
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 允许来自内部回环接口的流量
iptables -A INPUT -i lo -j ACCEPT

# 允许SSH访问（从特定管理IP）
iptables -A INPUT -p tcp -s 192.168.0.0/24 --dport 22 -j ACCEPT

# 允许HTTP/HTTPS访问
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# 保存规则
iptables-save > /etc/iptables/rules.v4

# 查看防火墙规则
iptables -L -n -v --line-numbers
```

## 核心定义

以太网（Ethernet）是目前应用最广泛的局域网技术，由IEEE 802.3标准所定义。以太网最初由Xerox公司的Robert Metcalfe于1973年发明，从最初的10Mbps同轴电缆总线型拓扑，发展到今天支持10Gbps/40Gbps/100Gbps的星型交换式拓扑。以太网采用CSMA/CD（早期半双工模式下）或全双工交换机制作为介质访问控制方法，使用MAC地址（48位物理地址）进行帧的寻址。以太网的帧格式有两种版本：DIX Ethernet V2（最常用）和IEEE 802.3标准帧。以太网的核心优势在于其简单性、低成本、可扩展性和向后兼容性，使其在40多年的演进中始终保持主导地位。现代以太网已从局域网扩展至城域网和广域网（如运营商级以太网），成为端到端通信的统一承载技术。

## 关键结论

- 以太网V2帧格式（最广泛使用）：目的MAC(6B) + 源MAC(6B) + 类型(2B，标识上层协议如0x0800=IPv4, 0x0806=ARP) + 数据(46-1500B) + FCS/CRC(4B)。IEEE 802.3帧将类型字段替换为长度/类型字段，并在其后增加了LLC和SNAP子层
- MAC地址（48位=6字节）是全球唯一标识：前24位是组织唯一标识符（OUI，由IEEE分配），后24位是厂商自行分配的扩展标识符。MAC地址的最高字节最低位为组播标志位（1=组播），次低位为本地管理标志位
- 以太网从共享式到交换式的演进是革命性的：共享式以太网（Hub）所有端口共享一个碰撞域和广播域；桥接式以太网（Bridge/二层交换机）隔离了碰撞域但共享广播域；VLAN技术进一步隔离了广播域
- 千兆以太网采用8B/10B编码或4D-PAM5编码（1000BASE-T使用4对5类线缆的全双工传输），万兆以太网仅支持全双工模式，彻底抛弃了CSMA/CD
- 以太网最小帧长64字节，最大帧长1518字节（不含VLAN标签）或1522字节（含4字节VLAN标签）。巨型帧（Jumbo Frame）可达9000字节，但属于非标准扩展

## 易错点

1. **DIX V2帧和IEEE 802.3帧的兼容性问题**：两种帧的"类型/长度"字段是区分点。当该字段值≤1500（0x05DC）时，表示IEEE 802.3的长度字段；当值≥1536（0x0600）时，表示DIX V2的类型字段。1501-1535之间的值未定义。现代网络设备通常都能自动识别两种帧格式。

2. **MAC地址全球唯一性并非绝对**：虽然有IEEE分配OUI保证理论上全球唯一，但实际中存在MAC地址克隆（虚拟机、高可用集群）、私有MAC地址（本地管理位=1的地址）和MAC地址耗尽重用等情况。

3. **以太网"类型"字段和IP首部"协议"字段的区别**：以太网帧的类型字段(2字节)标识上层协议（如0x0800=IPv4）；IP数据包首部的协议字段(1字节)标识IP层之上的传输层协议（如6=TCP, 17=UDP）。两者分级递进，共同实现多层协议的复用和分用。

4. **快速以太网（100Mbps）并非千兆以太网的直接10倍比例缩小**：100Mbps以太网的争用期仍然是51.2μs，但最小帧长仍为64字节。因为速率提升10倍时，最大网段距离缩减到约1/10（约200m），保证了碰撞检测的可行性。

## 例题

**例题1**：一个以太网帧携带了46字节的填充数据、20字节的IP头部和20字节的TCP头部以及40字节的实际数据。求该帧的总长度和效率。
求以太网帧的总长度。

**解答**：该帧的MAC层数据字段 = 20(IP头) + 20(TCP头) + 40(数据) = 80字节。最小帧数据字段为46字节，80>46故无需填充。总帧长 = 6(目的MAC) + 6(源MAC) + 2(类型) + 80(数据) + 4(FCS) = 98字节。传输效率 = 40/98 ≈ 40.8%。可见协议头部开销占据了一半以上的帧长，这是小数据包的固有劣势。

**例题2**：简述交换机（Switch）与集线器（Hub）的工作原理差异，并说明为什么交换机取代了集线器。

**解答思路**：Hub工作在物理层，收到一个端口的数据后向所有其他端口转发（广播），所有端口共享带宽且属于同一个碰撞域，传输效率低。Switch工作在数据链路层，具有MAC地址学习功能（自学习算法），能够维护一张"MAC地址→端口"的转发表（CAM表）。Switch收到帧后根据目的MAC查找转发表，仅向对应的端口转发（未知单播帧则泛洪），实现了：（1）每个端口独享带宽；（2）端口间的并发通信（不同端口对可同时传输而不互相干扰）；（3）碰撞域的隔离。这些优势使交换机在性能和可扩展性上远超集线器，从而完全取代了后者的市场地位。

```python
# Python解析以太网帧结构（使用struct模块）
import struct

def parse_ethernet_frame(frame_bytes):
    """解析以太网V2帧"""
    # 前14字节：6+6+2
    eth_header = struct.unpack('!6s6sH', frame_bytes[:14])
    dst_mac = ':'.join(f'{b:02x}' for b in eth_header[0])
    src_mac = ':'.join(f'{b:02x}' for b in eth_header[1])
    ethertype = eth_header[2]
    
    # 数据部分（去掉4字节FCS）
    data = frame_bytes[14:-4]
    fcs = frame_bytes[-4:]
    
    return {
        'dst_mac': dst_mac, 'src_mac': src_mac,
        'ethertype': hex(ethertype), 'data_len': len(data)
    }
```

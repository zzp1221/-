## 核心定义

IP协议（Internet Protocol，网际协议）是TCP/IP协议栈中最核心的网络层协议，负责将数据包从源主机传输到目的主机。IP协议提供的是无连接、不可靠、尽力而为（Best-Effort）的数据报传输服务。IPv4数据报首部标准长度为20字节（不含选项字段），包含版本、首部长度、区分服务、总长度、标识、标志、片偏移、生存时间（TTL）、协议、首部校验和、源地址和目的地址等关键字段。IP协议的核心功能包括：路由选择——根据目的IP地址和路由表决定数据包的下一跳；分片与重组——当数据包大小超过链路MTU时进行分片，到达目的地后进行重组；差错报告——配合ICMP协议报告传输过程中的错误。IP不负责数据的可靠传输、不保证顺序到达、不进行流量控制和拥塞控制，这些功能由上层（尤其是TCP）提供。

## 关键结论

- IPv4首部主要字段：版本(4bit, 值为4) | 首部长度(4bit, 以4B为单位，最小5=20B) | 总长度(16bit, 以B为单位，最大65535B) | 标识(16bit, 同源同标识) | 标志(3bit, DF禁止分片/MF还有分片) | 片偏移(13bit, 以8B为单位) | TTL(8bit, 跳数限制) | 协议(8bit, 6=TCP,17=UDP,1=ICMP) | 首部校验和(16bit, 仅校验首部)
- IP分片与重组：当数据报大于链路的MTU时，路由器将数据报分片。同一数据报的各分片具有相同的标识字段值。MF标志为1表示后面还有分片（最后一个分片MF=0）。片偏移字段表示该分片数据在原数据报中的位置（8字节为单位）。只有目的主机才进行重组，中间路由器只分片不重组
- TTL（Time To Live）实际按跳数工作：每经过一个路由器TTL减1，减到0时路由器丢弃该数据包并向源端发送ICMP时间超时报文。TTL防止数据包无限制地在网络中循环
- IP首部校验和：仅校验IP首部（不含数据部分），计算方法为将首部以16位字为单位进行二进制反码求和。每经过一个路由器，因TTL减1，校验和必须重新计算
- IP是无连接的、不可靠的、尽力而为的。无连接指发送前无需建立连接；不可靠指不保证一定能将数据包送达目的地（丢包、重复、乱序都可能发生）

## 易错点

1. **首部长度字段单位是4字节**：首部长度字段共4bit，最大值15，单位是4字节，所以IP首部最大长度=15×4=60字节。最小值为5（20字节标准首部）。当首部长度字段=5时，没有选项字段。很多人误算成字节直接相乘。

2. **片偏移以8字节为单位**：片偏移共13bit，单位是8字节（64bit）。因此除最后一个分片外，每个分片的数据长度必须是8字节的整数倍。如果一份1460字节的TCP数据需要分片，第一个分片数据长度可能是1480-20=1460（其中前8字节对齐），但这与"8字节倍数"不矛盾——片偏移字段的8字节单位指的是偏移量而非分片数据长度。

3. **DF标志和MTU路径发现**：当DF=1（Don't Fragment，禁止分片）且数据包超过下一跳MTU时，路由器不进行分片而是直接丢弃并发送ICMP"需要分片但DF已设置"差错报告。这是PMTUD（路径MTU发现）的工作基础。

4. **首部校验和不是CRC**：IP校验和采用16位反码求和，检测能力远弱于CRC。被检查的只有首部，不保护数据部分（数据完整性由上层协议负责）。每次路由都需重新计算（因为TTL变了），这也是IPv6完全取消了校验和字段的原因之一。

## 例题

**例题1**：一个IPv4数据报总长度为3000字节，标识为0x1234，需要在一个MTU为1500字节的链路上分片传输。请写出各个分片的：总长度、标识、MF标志、片偏移字段的值。

**解答**：原数据报除去20字节标准首部后数据部分为2980字节。MTU=1500字节，每个分片的最大数据载荷=1500-20=1480字节，且必须是8字节的倍数。1480是8的倍数（1480÷8=185），可用。分片1：总长=1500(20+1480)，标识=0x1234，MF=1，片偏移=0；分片2：总长=1500(20+1480)，标识=0x1234，MF=1，片偏移=1480÷8=185；分片3：总长=60(20+40)，标识=0x1234，MF=0，片偏移=(1480+1480)÷8=370。

**例题2**：说明为什么IPv6取消了IP首部校验和字段，这一决策的理由是什么？

**解答思路**：（1）每次路由都要重新计算校验和，增加处理时延；（2）数据链路层（以太网CRC）和传输层（TCP/UDP校验和）已经提供了差错检测，IP层再校验是冗余的；（3）校验和的检错能力有限（仅16位反码求和），对于高可靠性的现代光纤链路收益不大。但这一决策也意味着IPv6路由器对损坏的首部没有检测能力——坏的首部可能导致数据包被错误路由和丢弃。

## 代码示例

```python
import struct
import socket

def checksum(data):
    """计算IP校验和（16位反码求和）"""
    if len(data) % 2:
        data += b'\x00'
    total = 0
    for i in range(0, len(data), 2):
        word = struct.unpack('!H', data[i:i+2])[0]
        total += word
    while total >> 16:
        total = (total & 0xFFFF) + (total >> 16)
    return ~total & 0xFFFF

def build_ip_header(src_ip, dst_ip, protocol=socket.IPPROTO_TCP, data_len=0):
    """构造IPv4首部（简化版）"""
    version_ihl = 0x45  # 版本4 + 首部长度5×4B=20B
    total_length = 20 + data_len
    ip_id = 0x1234
    flags_offset = 0     # 无分片
    ttl = 64
    header = struct.pack('!BBHHHBBH',
        version_ihl, 0, total_length, ip_id,
        flags_offset, ttl, protocol, 0)  # 校验和先填0
    header += socket.inet_aton(src_ip)
    header += socket.inet_aton(dst_ip)
    # 计算校验和
    cksum = checksum(header)
    header = header[:10] + struct.pack('!H', cksum) + header[12:]
    return header

print(f"IP首部长度: {len(build_ip_header('192.168.1.1', '10.0.0.1'))}字节")
```

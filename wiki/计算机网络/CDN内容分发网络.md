## 核心定义

CDN（Content Delivery Network，内容分发网络）是一种基于互联网的分布式服务器系统，通过在全球范围内部署大量的边缘缓存节点（Edge Node / POP节点），将网站的内容（静态网页、图片、视频、下载文件等）缓存到距离用户最近的边缘节点上，当用户请求内容时由最近的边缘节点直接提供响应，从而显著减少内容传输的时延、降低源服务器负载并提升用户体验。CDN的核心技术包括：（1）DNS调度/智能DNS——根据用户的DNS服务器IP地址选择最近的或负载最低的边缘节点；（2）内容缓存与分发——将源服务器的内容通过拉取（Pull）或推送（Push）方式分发到各边缘节点的缓存中，并根据缓存策略（TTL、热度、文件大小等）决定缓存的有效期和替换策略；（3）源站回源——当边缘节点缓存未命中时向源服务器请求内容并缓存副本供后续请求使用；（4）动态加速（DSA/DCDN）——对无法缓存的动态内容（如API响应、个性化页面），通过CDN智能路由网络传输优化（选择最优网络路径、TCP/TLS连接优化、数据压缩等）加速交付。CDN是现代互联网不可或缺的基础设施，全球型CDN提供商如Akamai、Cloudflare、Fastly等支撑着WWW绝大多数高流量网站和流媒体服务的分发。

## 关键结论

- CDN的工作原理——缓存命中和回源：当用户请求www.example.com的首页时，本地DNS将其解析到CDN服务商提供的CNAME域名→CDN的DNS调度系统基于用户DNS服务器的地理位置、边缘节点负载等因素选择最优的边缘节点IP返回→用户浏览器连接该边缘节点→如果边缘节点已缓存了首页HTML，直接返回（命中）→如果未缓存，边缘节点向源站发起请求获取内容并缓存，然后响应给用户（回源/未命中）
- CDN的缓存策略：静态内容（图片、CSS/JS、字体文件、视频）长期缓存（几小时到几天TTL）；半静态内容（HTML首页）短期缓存（几分钟到几小时TTL）；动态内容（API响应）不缓存或者短时TTL（秒级）。现代CDN还支持高级缓存机制——如果源站返回新版本，缓存刷新（Purge）机制或版本化URL保证内容更新即时传播
- CDN对安全性的提升：CDN在边缘侧提供了额外的安全性层——其巨大的全球分布式网络容量可吸收并缓解DDoS攻击（DNS Flood、HTTP Flood等大容量攻击被分散在各节点上而非集中冲击源站）；WAF（Web应用防火墙）在CDN边缘识别并拦截SQL注入、XSS、CSRF等攻击流量；BOT管理区分真实用户和爬虫流量；TLS终止在CDN边缘加速加解密
- CDN加速HTTPS和优化连接：边缘节点提前完成TLS握手（TLS连接复用、Session Resumption），减少用户侧因为远距离而导致的高时延握手尝试；CDN支持TCP优化（BBR拥塞控制、TCP Fast Open）、连接池化（保持到源\边缘服务器的长连接）使得用户看到的内容加载时间大幅缩短
- 视频CDN的特殊需求：大文件分片传输、自适应码率（ABR、HLS/DASH协议）、高并发直播/点播需求下需要专门优化的大流量分发和P2P辅助分发

## 易错点

1. **CDN不是"把内容复制到所有边缘节点"**：CDN的缓存是按需Pull或预Push的方式。大部分CDN采用Pull模式——边缘节点首次未命中时回源获取并缓存，形成"热点"逐渐扩散。对整个库的全量复制的分发只有在大规模文件分发前才会执行Push预加载。误解CDN即自动全网拷贝会导致对缓存行为和成本管理的错误理解。

2. **DNS调度与Anycast的异同**：智能DNS是基于请求DNS服务器的网络位置（EDNS-Client-Subnet可选支持更精准的客户端IP子网）选择边缘节点IP返回，而后客户端用该IP建立连接。Anycast则是在IP路由层面——同一IP在不同地理位置上由不同的服务器对外声明，IP包通过BGP路由自动走向拓扑上最近的节点。两者常结合使用。

3. **CDN缓存不能代替动态加速**：缓存仅对可缓存的资源有效。动态请求（如针对用户ID的个性化搜索结果、交易操作）每个响应都不同——缓存无法加速。CDN通过高速骨干传输、智能路由选择、连接池化和协议优化来动态加速这些动态请求——DSA(Dynamic Site Acceleration)是CDN的高级付费功能集。

4. **CDN不等于对象存储**：CDN是缓存层而非持久存储层。虽然许多云服务商捆绑销售，但原理上CDN缓存可能被清除/被淘汰（TTL过期、缓存满策略驱逐），不能当作文件的唯一存放地。源站必须可靠保存所有原始内容的完整副本。

## 例题

**例题1**：大型视频网站使用CDN分发热门视频和非热门视频。说明CDN如何处理这两种场景，以及如何实现缓存对冷门内容的逐步覆盖。

**解答**：热门视频已经有大量边缘节点的缓存副本——用户请求被DNS调度到最近的节点，直接命中缓存返回高质量视频流；若视频已被数十万人观看，缓存副本覆盖了各大区域的CDN节点。非热门/长尾视频——用户第一次请求时边缘节点缓存不命中，实时从源站回源拉取视频数据（可能有首包延迟），同时节点在拉取过程中将视频逐段缓存下来。如果此后同一区域其他用户请求相同的冷门视频，第二用户就可以从缓存中获得（回源只发生一次）。冷门内容随着用户需求的增长会升温并逐渐在更多的节点缓存。为降低回源压力，CDN通常有"父节点"（中间缓存层）——先向区域父节点回源，父节点再从源站回源，层层缓存使得源站的请求量高度收敛。

**例题2**：CDN在网络安全方面发挥的作用是什么？为什么将CDN置于攻击面的前线有利于防护DDoS攻击？

**解答思路**：CDN的分布式全球网络规模极大（Tbps级）远超过单一源站的数据中心防御能力。攻击流量被CDN在全球的多个节点分散接收，不至于形成拥塞拥堵（任何单一链路被冲垮前CDN体系通过DNS重定向将流量切换到其他节点）。CDN边缘侧部署了WAF规则扫描恶意请求——在流量到达源站之前被清理和阻断（如大量不同源IP的GET请求可能全是HTTP Flood攻击，CDN用JS Challenge / CAPTCHA人机验证过滤）。CDN还使用Anycast网络隔断不良流量——同一IP掩蔽下的被DDoS攻击流量直接被丢弃在边缘。三层联防组成了现代Web安全方案的核心防线。

## 代码示例

```bash
# 使用curl查看CDN响应头
curl -I https://cdn.example.com/image.jpg
# 常见CDN响应头：
# X-Cache: HIT (缓存命中) / MISS (未命中回源)
# X-Cache-Key: 缓存键
# Via: CDN节点信息
# CF-Cache-Status: HIT (Cloudflare CDN)
# X-Served-By: 服务节点

# DNS解析查看CDN调度结果
dig www.cloudflare.com
# 可看到解析到多个不同区域的IP以体现调度

# 清除CDN缓存刷新（不同提供商的API）
# Cloudflare示例
curl -X POST "https://api.cloudflare.com/client/v4/zones/{zone_id}/purge_cache" \
     -H "Authorization: Bearer {api_token}" \
     -H "Content-Type: application/json" \
     --data '{"purge_everything":true}'
```

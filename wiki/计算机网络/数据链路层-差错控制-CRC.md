## 核心定义

循环冗余校验（CRC，Cyclic Redundancy Check）是数据链路层最广泛使用的差错检测技术，其核心思想是基于多项式除法（模二运算）为待发送的数据生成一个校验序列（FCS，帧检验序列），附加在数据帧尾部一起发送，接收方通过执行相同的多项式除法来判断数据在传输过程中是否发生了错误。发送方将k位数据M(x)左移r位（r为生成多项式G(x)的阶数），除以r+1位的生成多项式G(x)，得到的r位余数R(x)即为FCS。接收方将收到的完整帧（数据+FCS）除以相同的G(x)，若余数为0则表示传输正确，不为0则表示存在传输错误。标准化的生成多项式包括CRC-8（x^8+x^2+x+1）、CRC-16（x^16+x^15+x^2+1）、CRC-32（以太网标准：x^32+x^26+x^23+x^22+x^16+x^12+x^11+x^10+x^8+x^7+x^5+x^4+x^2+x+1）。CRC只能检测错误，不能纠正错误。

## 关键结论

- CRC的理论基础是有限域GF(2)上的多项式运算，所有运算采用模二加（异或/不进位加法）和模二除（不借位除法）
- 生成多项式G(x)的选择决定CRC的检错能力：若G(x)含有因子(x+1)，可检测所有奇数个比特差错；适当的G(x)可检测所有长度≤r的突发差错和绝大部分更长的突发差错
- CRC-32（以太网生成多项式）可以检测所有长度≤32比特的突发错误、所有奇数个比特错误，对更长的突发错误的漏检概率约为2^(-32)
- CRC计算步骤：（1）M(x)左移r位得到M(x)·x^r；（2）M(x)·x^r除以G(x)得到商Q(x)和余数R(x)；（3）发送T(x) = M(x)·x^r + R(x)；（4）接收方计算T(x)/G(x)，余数为0则无差错
- CRC是一个系统性码——原始数据在编码后保持不变（k + r个比特中前k位就是原数据），只在其后附加了校验序列

## 易错点

1. **CRC的模二除法运算**：模二除法与普通算术除法的区别在于，每一步的减法操作替换为异或（XOR，不考虑借位）。判断商位的规则是：若当前被除数的最高位是1，则商为1并做异或；若最高位是0，则商为0并继续使用0去异或（即实际不做任何操作而移位）。很多初学者在此处算术运算习惯导致错误。

2. **误以为CRC可以纠正错误**：CRC是纯粹的检错码（Error Detecting Code），不具备纠错能力（Error Correcting Code）。接收方检测到CRC错误后，唯一的处理方式是请求发送方重传（ARQ机制），而不能自行纠正。

3. **生成多项式阶数r与校验序列长度的关系**：校验序列FCS长度等于生成多项式G(x)的阶数r。如果G(x)是r阶多项式（最高次项为x^r），那么产生的余数R(x)的阶数≤r-1，即FCS长度为r比特。CRC-32的G(x)是32阶的，FCS为32比特（4字节）。

4. **对"所有错误都能检测"的误解**：CRC并不能检测出所有传输错误。当差错模式恰为G(x)的整数倍时（即E(x)能被G(x)整除），接收方计算的余数为0，会错误地认为传输无误。这种情况的概率就是漏检概率，对于良好设计的G(x)，漏检概率非常低。

## 例题

**例题1**：已知待发送数据M=101001，生成多项式G(x)=x^3+x^2+1（即1101），求CRC校验序列和实际发送的帧。

**解答**：G(x)的阶数r=3，所以将M左移3位得到M·x^3 = 101001000。用101001000除以1101（模二除法）：
101001000 ÷ 1101 进行模二除法，详细过程（采用异或运算）：第一步商1，异或得01110；第二步移一位为11100，商1，异或00110；第三步商0，移一位01100，商0；第四步移一倍11000，商1，异或00010；第五步商0，移一位00100，商0，再移一位01000，商0，最后余数010。所以FCS=R(x)=010（3位），实际发送帧为101001 + 010 = 101001010。验证：接收方计算101001010 ÷ 1101的模二除法余数应为0。

**例题2**：说明为什么CRC-32能够检测所有长度不超过32比特的突发错误。

**解答思路**：突发错误是指在连续的比特段内发生的错误。突发错误可以用错误多项式E(x)表示（出错位对应E(x)中该项系数为1，否则为0）。长度为L的突发错误可以表示为E(x) = x^i · (x^(L-1) + ... + 1) 的形式。CRC漏检的条件是E(x)能被G(x)整除。当L ≤ r = 32时，E(x)的最高次项不超过x^(i+L-1)≤ x^(i+31)＜ x^(i+32)，而G(x)是32阶多项式，除非E(x)=0（没有错误），否则G(x)不可能整除E(x)。因此所有长度≤32比特的突发错误都能被检测。

## 代码示例

```python
def crc_remainder(data_bits, generator_bits):
    """计算CRC余数（模二除法）"""
    # data_bits: 原始数据（已左移r位，尾部补r个0）
    # generator_bits: 生成多项式（r+1位，最高位必为1）
    data = list(data_bits)
    gen = list(generator_bits)
    r = len(gen) - 1  # 生成多项式的阶数
    n = len(data)
    
    for i in range(n - r):
        if data[i] == '1':
            # 异或操作
            for j in range(len(gen)):
                data[i + j] = '0' if data[i + j] == gen[j] else '1'
    return ''.join(data[-(r):])

def crc_encode(data, generator):
    """CRC编码：返回附加FCS后的完整帧"""
    r = len(generator) - 1
    shifted = data + '0' * r
    remainder = crc_remainder(shifted, generator)
    return data + remainder

def crc_verify(received, generator):
    """CRC验证：返回True（传输正确）或False（有错误）"""
    remainder = crc_remainder(received, generator)
    return all(b == '0' for b in remainder)

# 测试：M=101001, G(x)=x^3+x^2+1 -> 1101
data = "101001"
generator = "1101"
frame = crc_encode(data, generator)
print(f"发送帧: {frame}")
print(f"验证结果: {'正确' if crc_verify(frame, generator) else '有错误'}")
print(f"错误验证: {'正确' if crc_verify('101001011', generator) else '有错误'}")
```

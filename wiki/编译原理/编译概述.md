## 核心定义

编译原理（Compiler Design）研究如何将高级程序设计语言编写的源程序翻译为等价的目标语言（通常是机器代码或汇编代码）程序的理论和技术。编译过程分为六个主要阶段：(1) **词法分析**（Lexical Analysis）：扫描源程序字符流，识别出有意义的词法单元（Token），如关键字、标识符、常数、运算符等，由词法分析器（Lexer）完成；(2) **语法分析**（Syntax Analysis / Parsing）：根据语言的文法规则，将词法单元序列构建为语法树（Parse Tree）或抽象语法树（AST），检查程序的结构合法性；(3) **语义分析**（Semantic Analysis）：检查程序的语义合法性，包括类型检查、作用域分析等，生成带语义标注的中间表示；(4) **中间代码生成**（Intermediate Code Generation）：将源程序翻译为与机器无关的中间表示（如三地址码、四元式）；(5) **代码优化**（Code Optimization）：对中间代码进行等价变换以提高效率（减少指令数、提高并行度等）；(6) **目标代码生成**（Target Code Generation）：将优化后的中间代码映射到目标机器的指令集，完成寄存器分配和指令选择。此外，符号表管理（Symbol Table）和错误处理（Error Handling）贯穿所有阶段。

## 关键结论

- 编译过程是分阶段的流水线（Pipeline），各阶段之间通过明确的接口传递中间表示
- 前端（词法、语法、语义、中间代码生成）与机器无关；后端（优化、目标代码生成）与目标机器相关——这种分离提高了编译器的可移植性
- 词法分析和语法分析理论基础完善（正则表达式与有限自动机、上下文无关文法与下推自动机）
- 编译程序的结构通常包括：驱动模块、词法分析器、语法分析器、语义处理模块、符号表、错误处理器
- 现代编译器（如GCC、LLVM）采用多前端—统一优化器—多后端架构，LLVM的核心创新在于其统一中间表示IR
- 解释执行（Interpretation）与编译执行的关键区别：解释器逐条分析并执行源代码（不生成独立目标代码），编译器先翻译后执行

## 易错点

1. 混淆编译各阶段的输出：词法分析输出Token流（而非字符流），语法分析输出语法树（而非Token流），各阶段的输入输出需要清晰区分
2. 将"语义分析"与"中间代码生成"混为一谈：语义分析负责检查合法性（如类型匹配），中间代码生成负责翻译为IR——在语法制导翻译中可以合并进行
3. 认为编译就是完全转换：现代JIT（即时编译）混合了编译和解释，在运行时将热点代码编译为本地机器码

## 例题

**例题1**：描述源程序 `if (a > 3) b = a + 1;` 经过编译各阶段的处理结果。

**解答**：
- 词法分析：输出 Token 流 `[IF, LPAREN, ID(a), GT, NUM(3), RPAREN, ID(b), ASSIGN, ID(a), PLUS, NUM(1), SEMICOLON]`
- 语法分析：构建 if 语句的语法树，条件子树为 > 运算（a, 3），体子树为赋值（b, +（a, 1））
- 语义分析：检查 a 和 b 的类型是否兼容、> 运算的操作数是否合法
- 中间代码（三地址码）：`t1 = a > 3; if t1 goto L1; goto L2; L1: t2 = a + 1; b = t2; L2:`
- 优化：若 a 的值已知可进行常量折叠；复写传播等
- 目标代码：生成对应架构的机器指令序列

**例题2**：说明编译程序结构中"遍"（Pass）的概念及其与阶段的关系。

**解答**："遍"是指对源程序的整趟扫描。一个阶段可能需要多遍完成，多个阶段也可能合并在一遍中。例如：词法分析和语法分析常合并为一遍（由语法分析器调用词法分析器取下一个Token）；代码优化可能由多遍组成（每遍完成一种特定优化）。

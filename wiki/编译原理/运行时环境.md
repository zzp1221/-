## 核心定义

运行时环境（Runtime Environment）是支持编译后目标程序执行的基础设施，主要包括**存储组织**（运行时内存布局）和**过程调用机制**（如何管理函数调用的执行上下文）。它由编译器和操作系统共同支撑。

典型的内存布局（从低地址到高地址）为：代码区（Code/Text Segment）→ 静态数据区（Data Segment，存放全局/静态变量）→ 堆区（Heap，动态分配，向上增长）→ 空闲内存 → 栈区（Stack，向下增长，存放活动记录）。代码区和静态数据区在程序加载时确定大小；堆区由 `malloc/free`（C）或 GC（Java）管理；栈区管理过程的调用和返回。

**活动记录**（Activation Record / Stack Frame）是栈中为每次函数调用分配的连续内存块，包含：(1) 返回地址（Return Address）；(2) 调用者的栈帧指针（Dynamic Link / Control Link）；(3) 参数（Parameters）；(4) 局部变量（Local Variables）；(5) 临时变量（Temporaries）；(6) 保存的寄存器值（Saved Registers）。不同处理器和调用约定的活动记录布局不同。

**调用序列**（Calling Sequence）是过程调用的步骤：
- 调用者（Caller）：(1) 保存调用者保存寄存器；(2) 压入参数；(3) 调用 `call` 指令（压入返回地址并跳转）
- 被调用者（Callee）：(4) 保存旧帧指针；设置新帧指针；(5) 分配局部变量空间；(6) 保存被调用者保存寄存器

**返回序列**（Return Sequence）：(1) 恢复被调用者保存寄存器；(2) 将返回值放入寄存器；(3) 恢复栈和帧指针；(4) `ret` 指令（弹出返回地址并跳转）

**栈式分配**（Stack Allocation）利用 LIFO（后进先出）特性，天然匹配过程调用的嵌套激活模式。

## 关键结论

- 活动记录的帧结构是编译器中目标代码生成部分的直接操作对象
- 栈式分配使得递归成为可能——每次递归调用获得独立的栈帧
- 寄存器窗口（Register Window）技术（如 SPARC 架构）通过重叠寄存器组加速过程调用
- 静态链（Static Link）用于支持嵌套过程的非局部变量访问（如 Pascal 的嵌套函数），与动态链（Dynamic Link）不同
- 变长数据（如 C99 VLA、alloca）不能在编译期确定栈帧大小——需运行时动态调整或分配到堆
- Display 表是静态链的优化：用一个寄存器数组缓存各级静态链，加速非局部变量访问

## 易错点

1. 调用者与被调用者责任划分不清：哪些寄存器由调用者保存、哪些由被调用者保存由 ABI（应用程序二进制接口）规定，不同平台不同
2. 混淆静态链和动态链：动态链 = 前一个栈帧指针（调用顺序）；静态链 = 词法作用域的外层帧指针（仅嵌套过程语言需要）
3. 栈溢出与栈的大小：过多或过深的递归可导致栈溢出——编译期只能检测尾递归优化机会（Tail Recursion Optimization）

## 例题

**例题1**：画出以下 C 代码在递归调用 `fact(3)` 时的栈帧结构。
```c
int fact(int n) {
    if (n <= 1) return 1;
    return n * fact(n - 1);
}
```

**解答**：每次递归调用产生一个新栈帧。fact(3) → fact(2) → fact(1)。每个帧含：返回地址（返回到乘法语句后）、动态链（调用者帧指针）、参数 n、局部变量（可能为空）、返回值暂存位置。

**例题2**：解释为什么尾递归可以被优化为循环。

**解答**：若函数的最后操作是调用自身（尾递归），且返回值直接传递，则当前栈帧在调用后不再使用——可以被新调用的栈帧覆盖（复用同一帧）。这避免了 O(n) 的栈空间消耗，等价于循环。

## 代码示例

```python
class ActivationRecord:
    def __init__(self, name, level):
        self.locals = {}  # 局部变量偏移量
        self.offset = 0   # 当前偏移
        self.name = name
    
    def allocate(self, var_name, size=4):
        self.locals[var_name] = self.offset
        self.offset += size

class RuntimeStack:
    def __init__(self):
        self.stack = []
    
    def push_frame(self, name):
        frame = ActivationRecord(name, len(self.stack))
        self.stack.append(frame)
        return frame
    
    def pop_frame(self):
        return self.stack.pop()
    
    def current_frame(self):
        return self.stack[-1] if self.stack else None

runtime = RuntimeStack()
frame = runtime.push_frame('fact(3)')
frame.allocate('n', 4)
print(f"栈帧 '{frame.name}': 局部变量偏移 {frame.locals}")
```

## 核心定义

上下文无关文法（Context-Free Grammar, CFG）是 Chomsky 文法谱系中的 2 型文法，是描述程序设计语言语法结构的主要形式工具。CFG G 是一个四元组 (Vₙ, Vₜ, P, S)，其中 Vₙ 是非终结符有限集（表示语法范畴，如"表达式""语句"）、Vₜ 是终结符有限集（语言的字母表）、P 是产生式（Production）有限集、S ∈ Vₙ 是开始符号。每个产生式形如 A → α，其中 A ∈ Vₙ（单个非终结符），α ∈ (Vₙ ∪ Vₜ)*（任意终结符和非终结符组成的串）。

CFG 的关键特征是：非终结符的**替换不依赖于上下文**——无论 A 出现在什么上下文中，都可以按照产生式 A → α 进行替换。这区别于上下文有关文法（Type-1）的 βAγ → βαγ（A 仅在上下文 β...γ 中才可替换为 α）。

BNF（Backus-Naur Form）是描述 CFG 的标准记号：用 `::=` 替代 `→`，用 `< >` 括起非终结符，用 `|` 表示选择。EBNF 扩展了重复（`{}`表示零次或多次）、可选（`[]`）等语法糖。

## 关键结论

- CFG 的产生式 A → α 限制左边为单个非终结符，这使其具有足够表达力描述嵌套结构（而正则文法做不到）
- CFG 描述的语言称为上下文无关语言（CFL），正则语言是 CFL 的真子集
- 语法分析（Parsing）的核心任务：给定 CFG G 和输入串 w，判断 w ∈ L(G) 并构造语法树
- 任何 CFG 都可以等价地转化为 Chomsky 范式（CNF）：A → BC 或 A → a，未增加语言但极大简化了算法设计（如 CYK 算法）
- CFG 的歧义性（二义性）判定问题是不可判定的——不存在通用算法判断任意 CFG 是否有二义
- 上下文无关语言对并、连接、Kleene 闭包封闭，但对交和补不封闭

## 易错点

1. 混淆正规文法（右线性/左线性）与 CFG：CFG 允许产生式右边包含任意顺序的非终结符和终结符，而正规文法有严格的方向限制
2. 认为所有 CFG 都可以写为算符文法（Operator Grammar）：算符文法要求产生式右边不含两个连续的非终结符，这不是 CFG 的普遍特征
3. 空产生式 A → ε 的处理：消除 ε-产生式需要谨慎计算可空非终结符集合

## 例题

**例题1**：为 C 语言的 if-else 语句设计 CFG 产生式。

**解答**：
```
stmt → if_stmt | other_stmt
if_stmt → IF '(' expr ')' stmt
        | IF '(' expr ')' stmt ELSE stmt
```
该文法描述了 if 语句和 if-else 语句。注意此文法具有**二义性**（悬空 else 问题）：`if (x) if (y) a=1; else b=2;` 中的 else 可以与内层或外层 if 匹配，文法未明确指定。

**例题2**：证明 L = {aⁿbⁿcⁿ | n ≥ 1} 不是上下文无关语言。

**解答**：使用上下文无关语言的泵引理（Pumping Lemma for CFLs）。假设 L 是 CFL，泵长度为 p。取 s = aᵖbᵖcᵖ ∈ L。将 s 分解为 uvxyz（满足 |vxy| ≤ p, |vy| ≥ 1）。由于 |vxy| ≤ p，vxy 不能同时包含 a 和 c（否则跨过 p 个 b）。所以 vy 至多包含两类字符。泵后两类字符的数量不均衡，破坏 a:b:c = 1:1:1 的比例，矛盾。故 L 不是 CFL。

## 代码示例

```python
class CFG:
    def __init__(self, nonterminals, terminals, productions, start):
        self.Vn = set(nonterminals)
        self.Vt = set(terminals)
        self.P = productions  # dict: A -> [right_side1, right_side2, ...]
        self.S = start
    
    def derive(self, sentential_form, production_applied):
        """对句型应用产生式，返回推导出的新句型"""
        A, beta = production_applied
        return sentential_form.replace(A, beta, 1)

# 描述简单算术表达式的CFG
# E → E + T | T
# T → T * F | F
# F → ( E ) | id
cfg_expr = CFG(
    ['E', 'T', 'F'],
    ['+', '*', '(', ')', 'id'],
    {
        'E': [['E', '+', 'T'], ['T']],
        'T': [['T', '*', 'F'], ['F']],
        'F': [['(', 'E', ')'], ['id']]
    },
    'E'
)
print(f"CFG开始符号: {cfg_expr.S}, 非终结符: {cfg_expr.Vn}")
```

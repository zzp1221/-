## 核心定义

数据流分析（Data Flow Analysis, DFA）是编译器优化和程序分析的基础技术，通过对程序抽象状态的数学建模，在控制流图（CFG）上计算每个程序点的数据流信息。数据流分析的核心是一个方程系统：每个基本块有一个输入状态 IN[B] 和一个输出状态 OUT[B]，这些状态通过转移函数（Transfer Function）和交汇操作（Meet / Join Operator）关联。

四种经典数据流问题：

| 问题 | 域 | 方向 | 交汇操作 | 转移函数 |
|------|-----|------|----------|----------|
| 到达定值 | defs集合 | 前向 | ∪ (union) | OUT = GEN ∪ (IN - KILL) |
| 活跃变量 | vars集合 | 后向 | ∪ | IN = USE ∪ (OUT - DEF) |
| 可用表达式 | expr集合 | 前向 | ∩ (intersect) | OUT = GEN ∪ (IN - KILL) |
| 非常忙表达式 | expr集合 | 后向 | ∩ | IN = USE ∪ (OUT - DEF) |

**到达定值**（Reaching Definitions）：分析每个变量赋值能到达程序的哪些点而不被覆盖——用于常量传播和复写传播。

**活跃变量**（Live Variables）：分析在 CFG 某个点之后变量值是否还会被使用——用于寄存器分配（决定何时释放寄存器）。

**可用表达式**（Available Expressions）：分析某表达式是否已在所有到达路径上计算过且操作数未变——用于公共子表达式消除。

**迭代算法**是求解数据流方程的经典方法：(1) 初始化所有 IN/OUT 为适当值（前向流中 OUT[entry]=∅，其他为全集或空集）；(2) 对每个基本块应用转移函数，更新 OUT（前向）或 IN（后向）；(3) 重复直到所有 IN/OUT 不再变化（收敛）。算法保证收敛因为转移函数是单调的且格的有限链性。

## 关键结论

- 数据流分析的数学基础是格理论（Lattice Theory）——数据流值构成半格，转移函数是单调函数，不动点定理保证迭代收敛
- 前向分析（Forward Analysis）：信息沿控制流方向传播（IN → OUT）；后向分析（Backward Analysis）：信息逆控制流方向传播（OUT → IN）
- SSA 形式（Static Single Assignment）极大地简化了数据流分析——到达定值信息直接编码在变量名中
- 交汇操作 ∪（may analysis，保守分析：信息可能为真）vs ∩（must analysis，精确分析：信息一定为真）
- 数据流分析在 IDE（集成开发环境）中支撑了智能重构、死代码检测、未初始化变量警告等功能

## 易错点

1. 混淆前向和后向分析的初始值：前向分析中 entry 块的 OUT 设为初始值；后向分析中 exit 块的 IN 设为初始值
2. 转移函数 GEN/KILL 的顺序：正确顺序是 `OUT = (IN - KILL[B]) ∪ GEN[B]`，KILL 必须在 IN 之后实施
3. 交汇操作的混淆：May 问题用 ∪（如到达定值、活跃变量），Must 问题用 ∩（如可用表达式）

## 例题

**例题1**：计算以下 CFG 的到达定值。

```
B1: d1: a=1; d2: b=2;
B2: d3: c=a+b; d4: a=a+1;
B3: d5: b=a-b;
```

流图边：B1→B2, B2→B2（回边）, B2→B3。

**解答**：GEN[B1]={d1,d2}, GEN[B2]={d3,d4}, GEN[B3]={d5}。KILL[B1]={}（a,b均新定义）, KILL[B2]={d1}（a 被 d4 覆盖）, KILL[B3]={d2}（b 被 d5 覆盖）。

迭代求解：
OUT[B1] = {d1,d2}
IN[B2] = OUT[B1] ∪ OUT[B2] = {d1,d2} ∪ OUT[B2]（需迭代）
...（反复计算直到收敛）

**例题2**：活跃变量分析与寄存器分配的关系是什么？

**解答**：若两个变量在程序任何点都不同时活跃（即它们的活跃区间不重叠），它们可以共享同一个寄存器。这就是基于图着色的寄存器分配的核心思想——构造冲突图（Interference Graph），节点为变量，边表示两变量同时活跃（不能共享寄存器），然后对冲突图进行 k-着色（k 为可用寄存器数）。

## 代码示例

```python
def reaching_definitions(cfg, gen, kill):
    """前向数据流分析: 到达定值"""
    blocks = list(cfg.keys())
    IN = {b: set() for b in blocks}
    OUT = {b: set() for b in blocks}
    
    changed = True
    while changed:
        changed = False
        for b in blocks:
            # IN[B] = ∪ OUT[predecessors]
            new_IN = set()
            for pred in cfg.pred(b):
                new_IN |= OUT[pred]
            IN[b] = new_IN
            
            # OUT[B] = GEN[B] ∪ (IN[B] - KILL[B])
            new_OUT = gen[b] | (IN[b] - kill[b])
            if new_OUT != OUT[b]:
                OUT[b] = new_OUT
                changed = True
    return IN, OUT

# 简化的活跃变量分析 (后向)
def live_variables(cfg, use, defn):
    blocks = list(cfg.keys())
    IN = {b: set() for b in blocks}
    OUT = {b: set() for b in blocks}
    
    changed = True
    while changed:
        changed = False
        for b in reversed(blocks):
            # OUT[B] = ∪ IN[successors]
            new_OUT = set()
            for succ in cfg.succ(b):
                new_OUT |= IN[succ]
            OUT[b] = new_OUT
            # IN[B] = USE[B] ∪ (OUT[B] - DEF[B])
            new_IN = use[b] | (OUT[b] - defn[b])
            if new_IN != IN[b]:
                IN[b] = new_IN
                changed = True
    return IN, OUT

print("数据流分析: 迭代算法在格上求解不动点")
```

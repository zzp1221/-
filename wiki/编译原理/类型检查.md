## 核心定义

类型检查（Type Checking）是语义分析的核心任务之一，确保程序中所有操作的操作数类型符合语言的类型规则。类型检查器（Type Checker）遍历语法树（或 AST），对每个表达式和语句验证其类型合法性，并推断表达式的类型。类型检查可在编译时（静态类型检查）或运行时（动态类型检查）进行。

类型系统的关键概念：
- **类型等价**（Type Equivalence）：判断两个类型是否相同。两种主要方式：(a) **名称等价**（Name Equivalence）：两个类型相同当且仅当它们有相同的名称（如 `typedef int A; typedef int B;` 中 A 和 B 是不同类型）；(b) **结构等价**（Structure Equivalence）：两个类型相同当且仅当它们有相同的内部结构。
- **类型兼容**（Type Compatibility）：比等价宽松的概念，允许在某些语境下"相容"的类型进行交互。如 C 语言中 int 可隐式转换为 float。
- **强制类型转换**（Type Coercion）：编译器自动插入类型转换（如 int → float 的隐式提升），也称为**隐式类型转换**。
- **多态类型**（Polymorphic Type）：允许类型参数的类型，如泛型函数 `List<T>`。

类型推断（Type Inference）：从表达式上下文自动推导类型而不需要显式声明（如 ML/Haskell 的 Hindley-Milner 类型系统、C++11 的 auto）。

## 关键结论

- 静态类型检查在编译时捕获类型错误，提高程序安全性但增加开发时约束；动态类型检查推迟到运行时（如 Python）
- 强类型（Strong Typing）：严格限制不同类型间的隐式转换（如 Haskell、Rust）；弱类型（Weak Typing）：允许较多隐式转换（如 C）
- 类型检查基于类型规则（Typing Rules / Judgments），常用推理规则形式 `Γ ⊢ e : τ`（在类型上下文 Γ 下，表达式 e 有类型 τ）
- 类型安全和内存安全不完全等价：类型安全保证不存在类型错误，内存安全还涉及越界访问等
- 多态类型的实现：参数多态（通过类型变量统一化/Unification）、特设多态（通过重载解析 Overloading Resolution）

## 易错点

1. 混淆名称等价与结构等价：C 语言中结构体按名称等价（两个 `struct { int x; }` 即使结构相同，无名则不等价）；而数组类型基本上按结构等价
2. 忘记处理类型提升（Type Promotion）：如 `int + float` 中 int 应先提升为 float——若跳过此步则结果类型错误
3. 多态函数的实例化：泛型函数的每个实例化可能需要独立类型检查——忽略类型变量的替换导致类型错误漏报

## 例题

**例题1**：设计类型检查规则，检查表达式 a + b * c 的类型，其中 a:int, b:float, c:int。

**解答**：b * c：b 为 float，c 为 int → c 隐式提升为 float → 结果类型 float。a + (b*c)：a 为 int，(b*c) 为 float → a 提升为 float → 结果类型 float。最终表达式类型为 float。若语言不允许 int 到 float 的隐式转换，则需报错。

**例题2**：用推理规则形式表达加法运算的类型检查。

**解答**：
Γ ⊢ e₁ : int   Γ ⊢ e₂ : int
───────────────────────── (规则 INT-ADD)
     Γ ⊢ e₁ + e₂ : int

Γ ⊢ e₁ : float   Γ ⊢ e₂ : float
─────────────────────────────── (规则 FLOAT-ADD)
     Γ ⊢ e₁ + e₂ : float

如果需要混合类型（int + float），则添加：
Γ ⊢ e₁ : int   Γ ⊢ e₂ : float
────────────────────────────── (规则 MIXED-ADD-L)
   Γ ⊢ e₁ + e₂ : float
   (此处需隐式插入 int→float 转换)

## 代码示例

```python
class TypeChecker:
    def __init__(self):
        self.type_env = {}  # 变量 -> 类型
    
    def check_expr(self, expr):
        """返回表达式的类型"""
        if expr.kind == 'int_literal':
            return 'int'
        elif expr.kind == 'float_literal':
            return 'float'
        elif expr.kind == 'var':
            return self.type_env.get(expr.name, 'error')
        elif expr.kind == 'binop':
            left_type = self.check_expr(expr.left)
            right_type = self.check_expr(expr.right)
            if left_type == 'error' or right_type == 'error':
                return 'error'
            if expr.op == '+':
                if left_type == right_type:
                    return left_type
                elif 'int' in (left_type, right_type) and 'float' in (left_type, right_type):
                    return 'float'  # 隐式提升
                else:
                    return 'error'
            return 'error'
        return 'unknown'

# 测试: a(int) + b(float)
checker = TypeChecker()
checker.type_env = {'a': 'int', 'b': 'float'}
print(f"类型检查完成")
```

## 核心定义

寄存器分配（Register Allocation）是目标代码生成中最关键也最具挑战性的任务——决定哪些变量在何时驻留在（数量有限的）寄存器中，哪些变量需要溢出（Spill）到内存。由于寄存器访问速度远快于内存（通常快 10-100 倍），好的寄存器分配对性能影响巨大。

**图着色寄存器分配**（Graph-Coloring Register Allocation）是最经典和最广泛使用的方法，由 Chaitin 等人于 1981 年提出。核心思想：将寄存器分配问题归约为图的 k-着色问题（k = 可用寄存器数）。步骤如下：

(1) **构造冲突图**（Interference Graph）：节点 = 变量（活着的值），边 = 两变量在程序某点同时活跃（故不能共享同一寄存器）。冲突图的数据来源是活跃变量分析结果。

(2) **简化**（Simplify）：反复从图中移除度数 < k 的节点（压入栈中）。这些节点一定能被着色（因为它们邻居数量少于颜色数）。若所有节点度数均 ≥ k，则选择一个节点溢出（从图中移除并标记为溢出），继续简化。

(3) **选择/着色**（Select / Color）：从栈中弹出节点，分配一个与邻居不同的颜色（寄存器）。若无法分配颜色（所有 k 个颜色都被邻居占用），则该节点必须溢出。

(4) **溢出代码生成**（Spill Code）：为溢出变量在内存中分配栈槽，在每次使用前 Load、每次定义后 Store。溢出后重新构造冲突图并重复着色过程。

**线性扫描寄存器分配**（Linear Scan, Poletto 1999）：更简单和快速的方法——按变量的活跃区间排序，贪心地分配寄存器。虽有较低的分配质量，但因其速度快而常用于 JIT 编译器（如 HotSpot Client Compiler、V8）。

## 关键结论

- 寄存器分配质量直接影响编译器生成代码的性能——现代编译器通常有多个寄存器分配策略
- Chaitin-Briggs 算法是图着色分配的经典改进：Briggs 采用乐观着色（Optimistic Coloring），推迟溢出决策
- 对冲突图进行 k-着色的判定问题是 NP 完全的——但实践中启发式算法效果优秀
- 寄存器分配与指令调度之间存在相位排序问题（Phase Ordering Problem）——先分配后调度 vs 先调度后分配各有优劣
- SSA 形式的解构（Destruction）——从 SSA 形式转换回常规代码时会产生大量 `move` 指令，寄存器分配器负责消除冗余 move
- 函数调用 ABI（Application Binary Interface）对寄存器分配施加约束：某些寄存器必须由调用者或被调用者保留

## 易错点

1. 精确计算冲突图：若对变量的活跃区间保守估计过大，会产生多余的冲突边——导致不必要的溢出
2. 溢出代价估计失衡：应优先溢出在循环外使用或有低频率执行路径的变量——使用静态预测的加权溢出代价
3. SSA 解构时机：寄存器分配通常在退出 SSA 后进行，但 Chaitin 更早在 SSA 构建时就设计了 allocator

## 例题

**例题1**：有 2 个可用寄存器（R1, R2），变量 a, b, c, d 的活跃区间为：a[1,5], b[2,4], c[3,7], d[6,8]。画出冲突图并分配寄存器。

**解答**：冲突：a 与 b（区间重叠[2,4]）、a 与 c（[3,5]）、b 与 c（[3,4]）、c 与 d（[6,7]）。冲突图：a-b-c 构成三角形，d 仅与 c 相连。

图着色（k=2）：度为 2 的 a, c 都是关键节点。移除 d（度=1）→ 移除 b（与 a,c 相邻，但 d 已移除后 b 度和）。试试不同顺序... 实际上这是一个 3-团加上悬挂 d。三角形需要 3 种颜色，所以需要溢出至少一个变量。假设溢出 c：a→R1, b→R2, d→R1（c 在内存）。

**例题2**：解释为什么"Load/Store 溢出代码要尽量放在循环外"。

**解答**：若一个变量需要在循环中溢出，则每次迭代都要执行相关的 Load/Store——循环通常有高执行频率，溢出代价会被放大。寄存器分配器应优先溢出循环外活跃的变量而非循环内频繁使用的变量。

## 代码示例

```python
def greedy_coloring(interference_graph, k_colors):
    """简化的贪心图着色寄存器分配"""
    color_of = {}
    
    # 按度数降序排序（启发式）
    nodes = sorted(interference_graph.keys(), 
                   key=lambda v: len(interference_graph[v]), reverse=True)
    
    for node in nodes:
        # 邻居已用的颜色
        used = {color_of[n] for n in interference_graph[node] 
                if n in color_of}
        # 找第一个可用颜色
        for c in range(k_colors):
            if c not in used:
                color_of[node] = c
                break
        else:
            # 溢出
            color_of[node] = 'spill'
    
    return color_of

# 构造冲突图: a-b, a-c, b-c, c-d
graph = {
    'a': {'b', 'c'},
    'b': {'a', 'c'},
    'c': {'a', 'b', 'd'},
    'd': {'c'}
}
allocation = greedy_coloring(graph, 2)
print(f"2个寄存器的分配: {allocation}")
```

## 核心定义

**指令选择**（Instruction Selection）是将中间表示（IR）操作映射为目标机器指令序列的过程。目标机器的指令集架构（ISA）通常支持多种实现同一操作的方式，指令选择器需要在候选指令模式中选择最优组合。主要方法包括：

(1) **树覆盖法**（Tree Covering / Tree Pattern Matching）：将 IR 表达式表示为一棵树，目标机器的每条指令表示为树的重写规则（Tile），指令选择转化为用最小代价的 tile 覆盖整棵树的问题。动态规划（DP）或最大匹配（Maximal Munch，贪心最大匹配）是两种常用求解策略。IBURG/TWIG 等工具基于此法自动生成指令选择器。

(2) **窥孔优化**（Peephole Optimization）：在已生成的代码序列上滑动一个小窗口（如 2-3 条指令），识别可优化的模式并替换为更高效的指令序列。

**指令调度**（Instruction Scheduling）是重新排列指令的执行顺序以提高处理器流水线效率的技术。现代处理器使用流水线（Pipeline）、超标量（Superscalar）、乱序执行（Out-of-Order Execution）等微架构技术，指令调度旨在：(a) 减少流水线停顿（Pipeline Stall / Bubble）；(b) 隐藏指令延迟（如 load 指令的延迟槽）；(c) 提高指令级并行度（ILP）。

基本方法：构造**数据依赖图**（Data Dependence Graph）和**资源约束**，用**列表调度**（List Scheduling）算法——维护就绪指令队列（所有操作数已就绪的指令），按优先级（如关键路径长度）选出就绪指令并在可用功能单元上发射。

## 关键结论

- 指令选择是 NP 完全的（在一般情况下），但动态规划在树形表达式上可在多项式时间内找到最优解
- 常见的 tile：寄存器-寄存器操作、立即数-寄存器操作、内存-寄存器操作（CISC 中）、load/store（RISC 中）
- 指令调度的核心挑战：平衡数据依赖（必须遵守）和资源约束（功能单元种类和数量有限）
- 静态调度（编译器完成）vs 动态调度（硬件运行时完成）：现代处理器两者兼有
- 软件流水（Software Pipelining）是循环调度的高级技术——交错不同迭代的指令以填充延迟槽
- 指令调度与寄存器分配相互影响——先分配后调度可能导致"反依赖"（anti-dependence），需要适当的处理

## 易错点

1. 指令选择中忽略寻址模式的约束：CISC 机器的复杂寻址模式（如基址+变址+偏移）有特定组合限制
2. 调度破坏数据依赖：不能将 `x = a + b; y = x + c;` 调度为 `y = x + c; x = a + b;`——这违反了 RAW（Read-After-Write）依赖
3. 延迟槽的误填：延迟槽（Delay Slot）中的指令在分支生效前执行——必须确保填槽的指令在所有情况下都合法

## 例题

**例题1**：将 IR 指令 `d = (a + b) * c` 用 RISC 风格（三地址 load/store）和 CISC 风格（内存操作数）分别进行指令选择。

**解答**：
RISC（MIPS-like）：
```
lw r1, a       // load a
lw r2, b       // load b
add r3, r1, r2 // r3 = a + b
lw r4, c       // load c
mul r5, r3, r4 // r5 = r3 * c
sw r5, d       // store d
```

CISC（x86-like）：
```
mov eax, [a]   // load a
add eax, [b]   // eax = a + b
imul eax, [c]  // eax = eax * c
mov [d], eax   // store d
```

**例题2**：解释为什么 `load r2, [addr]` 和 `add r3, r2, r4` 之间可能需要调度独立的指令。

**解答**：load 指令通常有较大延迟（缓存命中需数个周期，缓存缺失需数百周期）。若直接使用 r2 会流水线停顿。在 load 和使用之间插入不依赖 r2 的独立指令可以隐藏 load 延迟。
```
load r2, [addr]   // 延迟 3 周期
add r5, r6, r7    // 独立指令1
sub r8, r9, r10   // 独立指令2
add r3, r2, r4    // 使用 r2（此时 load 已完成）
```

## 代码示例

```python
def list_scheduling(dep_graph, func_units, latency):
    """列表调度算法"""
    ready = [n for n in dep_graph if dep_graph[n].deps == 0]
    scheduled = []
    cycle = 0
    
    while ready:
        # 选优先级最高的就绪指令
        best = max(ready, key=lambda n: dep_graph[n].priority)
        if func_units.available(best, cycle):
            scheduled.append((best, cycle))
            func_units.reserve(best, cycle, latency[best.op])
            ready.remove(best)
            # 释放依赖
            for succ in dep_graph[best].successors:
                dep_graph[succ].deps -= 1
                if dep_graph[succ].deps == 0:
                    ready.append(succ)
        cycle += 1
    return scheduled

# 示例: a=load, b=add(a,c), d=add(b,e)
print("列表调度: 就绪指令按优先级调度到功能单元")
```

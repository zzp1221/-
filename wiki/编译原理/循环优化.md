## 核心定义

循环优化（Loop Optimization）是代码优化中最有效的类别——因为程序的大部分执行时间通常花在循环上（"10%的代码占据了90%的执行时间"）。循环优化的核心是识别和利用循环的结构性质来减少循环体内的计算量。

**循环不变量代码外提**（Loop-Invariant Code Motion / Hoisting）：若循环中某表达式的值在每次迭代中都不变（其操作数在整个循环过程中值不变），则可将其移到循环之前仅计算一次。例如：
```c
while (i < n) {
    x = y * z;  // y 和 z 在循环中不变 → 可外提
    a[i] = x + i;
}
```

**强度削弱**（Strength Reduction）：用低代价操作替代循环中每次迭代重复进行的高代价操作，特别是将乘法转化为加法。经典模式：循环变量 i 每次增加 1，表达式 `i * k` 可以用累加（每次都加 k）替代乘法。

**归纳变量优化**（Induction Variable Optimization）：归纳变量是每次迭代按固定步长（通常为 ±1）变化的变量。包括：(a) 基本归纳变量：`i = i ± c`（如循环计数器 i）；(b) 派生归纳变量：与基本归纳变量成线性关系的变量（如 `j = 4*i`）。优化包括：强度削弱（派生归纳变量的乘改为加）和归纳变量消除（用派生归纳变量替代基本归纳变量以减少变量数目）。

**其他循环优化**：(d) 循环展开（Loop Unrolling）：复制循环体以减少迭代次数和分支开销；(e) 循环合并/分裂；(f) 循环交换（Loop Interchange）：改变嵌套循环的顺序以改善缓存局部性。

## 关键结论

- 循环优化的前提是正确识别循环（自然循环通过支配关系和返回边识别）
- 代码外提的充分条件：表达式在循环的所有出口处都产生相同的值 — 这要求到达-定值分析（Reaching Definitions）
- 强度削弱将乘法/除法转化为加法，尤其对数组元素访问（地址计算 `base + i*stride`）效果显著
- 归纳变量的线性关系由定义点和使用点的数据流方程推导
- 循环展开的"度"（展开因子）需权衡代码膨胀和性能提升——过度展开导致指令缓存（I-Cache）频繁缺失
- LLVM 中使用 `opt -licm`, `opt -loop-reduce`, `opt -indvars` 等 Pass 执行循环优化

## 易错点

1. 代码外提时对非纯函数的误判：若循环不变量涉及函数调用，需确认函数是纯函数（无副作用）才能外提
2. 强度削弱后忘记更新使用点：将 `t = 4*i` 削弱为 `t += 4` 后，原代码中所有使用 `t` 的地方需引用新的削弱变量
3. 循环展开在边界处需处理剩余迭代：当迭代次数不是展开因子的整数倍时，需额外代码处理"尾数"

## 例题

**例题1**：优化以下循环。
```c
for (i = 0; i < n; i++) {
    a[i] = b[i] + k * m;
}
```

**解答**：(1) 循环不变量外提：`temp = k * m` 移到循环前。(2) 强度削弱：`a[i]` 的地址为 `a + i*sizeof(int)`，可用指针累加 `pa++` 替代；同理 `pb++`。优化后：
```c
temp = k * m;
for (i = 0, pa = a, pb = b; i < n; i++, pa++, pb++) {
    *pa = *pb + temp;
}
```

**例题2**：识别以下循环中的归纳变量并消去多余归纳变量。
```c
i = 0; j = 0;
while (i < n) {
    a[j] = i;
    i = i + 1;
    j = j + 1;
}
```

**解答**：i 和 j 都是归纳变量，步长相同为 1，且满足 j = i。可以消除 j，用 i 替代：
```c
i = 0;
while (i < n) {
    a[i] = i;
    i = i + 1;
}
```

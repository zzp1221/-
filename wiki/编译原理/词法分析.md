## 核心定义

词法分析（Lexical Analysis）是编译过程的第一阶段，负责将源程序的字符序列转换为有意义的**词法单元**（Token）序列。词法分析器（Lexer / Scanner）扫描输入字符流，按照语言的词法规则（以正则表达式 / 正规式描述）识别出关键字（Keyword）、标识符（Identifier）、常量（Literal / Constant）、运算符（Operator）、界符（Delimiter）等 Token，并过滤掉空白字符和注释。Token 通常表示为一个二元组（种别码，属性值），其中种别码标识 Token 的种类（如 KW_INT, ID, NUM, OP_PLUS），属性值记录具体信息（如标识符的名字、常数的数值）。

词法规则的核心理论基础是**正则表达式与有限自动机的等价性**：正规式 → NFA（Thompson 构造法）→ DFA（子集构造法）→ 最小化 DFA（Hopcroft 算法）。此流程构成了词法分析器自动生成工具（如 Lex/Flex）的理论基础。Lex 接受以正则表达式书写的词法规则，自动生成 C 语言实现的 DFA 词法分析器。词法分析中的关键实现技术包括：最大匹配原则（最长匹配，识别尽可能长的词法单元）、前瞻技术（Lookahead，向前看若干字符以消除歧义）。

## 关键结论

- 词法分析的数学基础是正则语言理论：正则表达式 ⇔ NFA ⇔ DFA ⇔ 正则文法，四种描述等价
- Token 的词法规则用正规式定义；但正规式无法描述嵌套结构（如匹配括号）——这需要上下文无关文法
- 最大匹配原则（Longest Match）：当多个模式匹配前缀时，取匹配字符最多的模式
- 关键字通常作为保留字处理：先识别为标识符，再查保留字表来判定是否为关键字
- 缓冲区管理技术（双缓冲区 / 哨兵标记）使词法分析器高效读取字符而无需逐字符调用 I/O
- Lex/Flex 工具生成代码的原理是：将多条正规式合并为一个 NFA → 转化为 DFA → 最小化 → 输出驱动代码

## 易错点

1. 正规式与上下文无关文法的混淆：aⁿbⁿ 不能用正规式描述，因为正规语言无法"记住"任意数量的计数——需要栈来匹配
2. 关键字与标识符的优先匹配：必须优先匹配关键字模式，否则 `int` 会被错误地识别为普通标识符
3. NFA 到 DFA 转换时遗漏 ε-闭包：子集构造法的第一步是对初始状态求 ε-闭包，忘记此步会导致接受状态错误

## 例题

**例题1**：为 C 语言的整型常量（十进制和八进制）设计正则表达式。

**解答**：十进制整型常量：`0 | [1-9][0-9]*`。八进制常量（以 0 开头）：`0[0-7]*`。合并：`0[0-7]* | [1-9][0-9]*`。注意：单独的 "0" 匹配第一个分支，非零开头的数字串匹配第二个分支。

**例题2**：构造正规式 (a|b)*abb 的 NFA，然后转化为 DFA。

**解答**：
NFA（Thompson构造法）：从开始状态经 ε 转移到 (a|b)* 的子自动机和 abb 的子自动机。(a|b)* 含 ε-自环，连接 a, b, b 的顺序转移。

DFA（子集构造）状态集合：
A = ε-closure({0}) = {0,1,2,4,7}
B = ε-closure(move(A,a)) = {1,2,3,4,6,7,8}  (读到a后)
C = ε-closure(move(A,b)) = {1,2,4,5,6,7}    (读到b后)
D = ε-closure(move(B,a)) = B
E = ε-closure(move(B,b)) = {1,2,4,5,6,7,9}  (读到ab后)
F = ε-closure(move(E,b)) = {1,2,4,5,6,7,10} (读到abb后，接受状态)
...最终得到最小 DFA。

## 代码示例

```python
class Token:
    def __init__(self, type, value):
        self.type = type
        self.value = value
    def __repr__(self): return f"Token({self.type},{self.value})"

class SimpleLexer:
    def __init__(self, text):
        self.text = text
        self.pos = 0
    
    def scan(self):
        tokens = []
        while self.pos < len(self.text):
            ch = self.text[self.pos]
            if ch.isspace():
                self.pos += 1
            elif ch.isdigit():
                num = ''
                while self.pos < len(self.text) and self.text[self.pos].isdigit():
                    num += self.text[self.pos]; self.pos += 1
                tokens.append(Token('NUM', int(num)))
            elif ch.isalpha():
                ident = ''
                while self.pos < len(self.text) and self.text[self.pos].isalnum():
                    ident += self.text[self.pos]; self.pos += 1
                keywords = {'if', 'else', 'while', 'int', 'return'}
                t = 'KEYWORD' if ident in keywords else 'ID'
                tokens.append(Token(t, ident))
            elif ch in '+-*/()':
                tokens.append(Token('OP', ch)); self.pos += 1
            else:
                self.pos += 1
        return tokens

lexer = SimpleLexer("int x = 123 + y")
print(lexer.scan())
```

## 核心定义

预测分析表（Predictive Parsing Table）是表驱动 LL(1) 分析器的核心数据结构，是一个二维表 M[A, a]，其中 A 为非终结符，a 为终结符（包括 $），表项 M[A, a] 包含当栈顶为 A 且当前输入符号为 a 时应使用的产生式（或为空表示语法错误）。预测分析表通过 FIRST 和 FOLLOW 集构造：对文法的每条产生式 A → α，(1) 对每个 a ∈ FIRST(α) − {ε}，将 A → α 填入 M[A, a]；(2) 若 ε ∈ FIRST(α)，则对每个 b ∈ FOLLOW(A)，将 A → α 填入 M[A, b]。

表驱动的 LL(1) 分析器使用一个**分析栈**和一个**输入缓冲区**。初始栈为 `$ S`（S 为开始符号，$ 为栈底标记），输入缓冲为 `w$`（w 为输入串）。每一步分析：(a) 若栈顶 X 为终结符且 X 等于当前输入符号 a，则弹栈并前进输入指针；(b) 若栈顶 X 为非终结符，查预测分析表 M[X, a]，若存在产生式 X → Y₁Y₂...Yₖ，则弹 X 并将 Yₖ,...,Y₁ 依次压栈（注意顺序）；(c) 若栈顶与输入不匹配且无产生式可用，报告语法错误。

预测分析表出现**多重入口**（M[A, a] 有多个产生式）说明文法不是 LL(1)（此时叫**分析冲突**）。重复分析直到栈和输入同时到达 $，则分析成功。

## 关键结论

- 预测分析表的构造完全由 FIRST 和 FOLLOW 集确定——两者计算的正确性直接决定分析表的正确性
- 预测分析表的尺寸为 |Vₙ| × (|Vₜ| + 1)，对典型的编程语言文法，此表尺寸适中
- 表驱动的 LL(1) 分析器等价于递归下降分析器（一个使用栈模拟递归，一个使用函数调用栈）
- 多重入口的出现意味着需改写文法（消除二义、消除左递归、提取左公因子）或使用更强的分析法（如 LL(k), k>1）
- 预测分析表也可用于错误恢复：在空白表项处填入错误处理例程的标记

## 易错点

1. 产生式右部入栈顺序错误：压栈时应从右到左压入（使最左符号在栈顶），初学者容易反向
2. 栈底标记与输入结束标记混淆：栈底用 $，输入结束也用 $——两者语义不同但值相同
3. 将 ε-产生式处理为不压入任何东西：A → ε 当选中时只需弹出 A，不压入新符号

## 例题

**例题1**：为文法 E → TE', E' → +TE' | ε, T → FT', T' → *FT' | ε, F → (E) | id 构造预测分析表。

**解答**：由 FIRST 和 FOLLOW 集计算结果可得：

|     |  id  |  +   |  *   |  (   |  )   |  $   |
|-----|------|------|------|------|------|------|
|  E  | TE'  |      |      | TE'  |      |      |
| E'  |      | +TE' |      |      |  ε   |  ε   |
|  T  | FT'  |      |      | FT'  |      |      |
| T'  |      |  ε   | *FT' |      |  ε   |  ε   |
|  F  | id   |      |      | (E)  |      |      |

空白表项表示语法错误。

**例题2**：用上述分析表模拟分析字符串 `id + id * id`。

**解答**（部分步骤）：
栈: $ E       输入: id+id*id$ → M[E,id]=TE', 弹出E压入E'T
栈: $ E'T     输入: id+id*id$ → M[T,id]=FT', 弹出T压入T'F
栈: $ E'T'F   输入: id+id*id$ → M[F,id]=id, 弹出F压入id
栈: $ E'T'id  输入: id+id*id$ → 匹配id, 弹栈前进
栈: $ E'T'    输入: +id*id$  → M[T',+]=ε, 弹出T'
栈: $ E'      输入: +id*id$  → M[E',+]=+TE', 弹出E'压入E'T+
... 继续直到栈和输入都只剩 $，分析成功。

## 代码示例

```python
def ll1_parse(table, start, input_tokens):
    """表驱动LL(1)分析器"""
    stack = ['$', start]
    input_tokens = list(input_tokens) + ['$']
    ip = 0
    
    while stack:
        top = stack.pop()
        a = input_tokens[ip]
        
        if top == '$' and a == '$':
            return True  # 接受
        if top == a:
            ip += 1      # 匹配终结符
        elif top in table and a in table[top]:
            rhs = table[top][a]
            if rhs != ['ε']:
                # 从右到左压栈
                for sym in reversed(rhs):
                    stack.append(sym)
        else:
            print(f"Error: top={top}, lookahead={a}")
            return False
    return False

# 简化分析表
table = {
    'E':  {'id': ['T', 'E\''], '(': ['T', 'E\'']},
    'E\'':{'+': ['+', 'T', 'E\''], ')': ['ε'], '$': ['ε']},
    'T':  {'id': ['F', 'T\''], '(': ['F', 'T\'']},
}
print(ll1_parse(table, 'E', 'id+id'))
```

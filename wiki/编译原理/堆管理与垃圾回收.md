## 核心定义

堆（Heap）是内存中用于动态分配的区域——在编译期无法确定大小或生命周期的数据在此分配。堆的管理是运行时系统的关键责任，包括分配（Allocation）和回收（Deallocation / Reclamation）两大部分。C 语言依赖手动管理（`malloc/free`），而 Java/C#/Python/Go 等语言配备了**垃圾回收**（Garbage Collection, GC）机制自动回收不再使用的内存。

**标记-清除**（Mark-and-Sweep）是最基础的 GC 算法：(1) **标记阶段**：从根集（Root Set，包括全局变量、栈上的局部变量、寄存器中的指针）出发，DFS/BFS 遍历所有可达对象，将可达对象标记为"存活"；(2) **清除阶段**：线性扫描整个堆，回收所有未标记对象的内存（加入空闲链表）。标记-清除的缺点是产生内存碎片且回收时需要暂停（Stop-the-World, STW）。

**引用计数**（Reference Counting）是另一种策略：每个对象维护一个被引用次数的计数器。当计数器归零时对象可被立即回收。优点：实时性（无需 STW）、实现简单；缺点：(1) 无法回收循环引用（circular references）——A 引 B, B 引 A，计数永不为零；(2) 每次引用赋值都需更新计数，开销较大。

**分代收集**（Generational GC）利用了"大多数对象很快死亡"（Weak Generational Hypothesis）和"存活越久的对象越可能继续存活"（Strong Generational Hypothesis）两个经验假设。将堆分为新生代（Young Generation，使用复制收集的 Minor GC）和老年代（Old Generation，使用标记-清除/标记-整理的 Major GC），新生代使用复制算法（Copying GC：将存活对象复制到新空间）。

## 关键结论

- 三色标记法（Tri-Color Marking）：白（未访问）、灰（已访问但子节点未遍历）、黑（子节点已遍历）——是增量式 GC 的基础
- 写屏障（Write Barrier）和读屏障（Read Barrier）用于在并发/增量 GC 中追踪运行中的程序对对象图的变化
- 标记-整理（Mark-and-Compact）是标记-清除的改进：标记后将存活对象移动到堆的一端，消除碎片
- Stop-the-World 时间：标记-清除和复制 GC 通常在 STW 下执行；增量 GC 和并发 GC 旨在减少 STW 时间
- Java HotSpot 的默认 GC（G1）是区域性、并发、分代的 GC；低延迟应用选用 ZGC 或 Shenandoah
- 引用计数的循环引用问题可通过"弱引用"或结合追踪式 GC（Trial Deletion 算法）部分解决

## 易错点

1. 引用计数算法中忘记处理循环引用：导致内存泄漏——这是引用计数最著名的缺陷
2. 标记-清除中的悬空指针（Dangling Pointer）：回收后若不将原指针置空，会导致 use-after-free 安全漏洞
3. 分代 GC 中的跨代引用：老年代对象引用新生代对象（Rare but possible）——需通过记忆集（Remembered Set）记录

## 例题

**例题1**：模拟标记-清除算法，给定对象图 A→B→C→D, B→C, E→F, G（独立）。根集 = {A, E}。标记存活对象。

**解答**：从 A 出发：标记 A,B,C,D；从 E 出发：标记 E,F。未标记对象：G。清除后回收 G 的内存。

**例题2**：引用计数收集器能否回收以下结构？
```python
a = [1, 2]   # refcount=1
b = a        # refcount=2
a = None     # refcount=1
b = [3, 4]   # 原来b指向的[1,2] refcount=0，可以回收
a, b = b, a  # 交换，各自refcount不变
```

**解答**：能。每个对象的引用计数在所有引用消失后归零即被回收。此例中没有循环引用，引用计数能正确处理。若代码变为 `a.next=b; b.next=a; a=b=None` 则两个对象相互引用，引用计数无法回收。

## 代码示例

```python
class RefCountObj:
    def __init__(self, value):
        self.value = value
        self.ref_count = 0
    
    def inc_ref(self): self.ref_count += 1
    def dec_ref(self):
        self.ref_count -= 1
        return self.ref_count == 0  # 可回收

def mark_sweep_sim(objects, roots):
    """简化的标记-清除模拟"""
    marked = set()
    def dfs(obj):
        if obj in marked: return
        marked.add(obj)
        for ref in obj.get('refs', []):
            dfs(ref)
    
    for root in roots:
        dfs(root)
    
    alive = marked
    garbage = set(objects.values()) - alive
    return alive, garbage

# 示例对象图
obj_a = {'id': 'A', 'refs': []}; obj_b = {'id': 'B', 'refs': []}
obj_a['refs'] = [obj_b]; obj_b['refs'] = [obj_a]  # 循环引用
objects = {'A': obj_a, 'B': obj_b}
alive, garbage = mark_sweep_sim(objects, [])
print(f"根为空时，循环引用是否可达: {obj_a in alive}")
```

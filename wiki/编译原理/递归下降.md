## 核心定义

递归下降分析（Recursive Descent Parsing）是最直观的自顶向下语法分析方法——为文法的每个非终结符编写一个分析函数（Procedure），函数之间通过相互调用来递归地解析输入。这些函数根据当前输入符号（前瞻一个 Token）决定使用哪个产生式，并依次调用产生式右部各符号对应的分析函数：对于终结符则期望当前 Token 与之匹配并前进；对于非终结符则递归调用其分析函数。

递归下降分析器本质上是一个手写的 LL(1) 预测分析器。其正确性前提是文法为 LL(1)（无回溯），或配合回溯技术（Backtracking）来处理非 LL(1) 情况。在生产实践中，大多数工业级编译器（如 GCC、Clang、Go 编译器）都采用**手写递归下降**来实现语法分析器，因其：(1) 代码直观，易于理解和调试；(2) 错误恢复策略灵活，可给出高质量的错误信息；(3) 不受自动生成工具文法形式的限制。

递归下降分析器的一般结构：
```c
void parse_S() {
    if (lookahead in FIRST(α1)) { parse_α1(); }
    else if (lookahead in FIRST(α2)) { parse_α2(); }
    ...
    else { error("unexpected token"); }
}
```

## 关键结论

- 递归下降分析器每个非终结符对应一个函数，函数的控制流直接映射文法的产生式结构
- 针对二义性文法需通过手动消歧：如为运算表达式设计多层函数（parse_expr → parse_term → parse_factor）来编码优先级
- EBNF 记号的递归下降实现：{X} 对应 while 循环，[X] 对应 if 语句
- 左递归文法的递归下降实现会引发**无限递归**——必须先消除左递归
- 预测分析的核心：对每个产生式右部计算"开始符号集"（FIRST），以此为分支条件
- 错误恢复策略可灵活嵌入：如 panic mode（跳过 Token 直到同步符号）、短语级恢复等

## 易错点

1. 在含有左递归的文法中直接编写递归下降函数——这将导致无限递归，必须先将文法改为右递归形式
2. 忘记在匹配终结符后前进 Token：若 match(T) 内部未调用 next_token()，则分析器会陷入死循环
3. 错误恢复策略考虑不足：简单的"发现错误即退出"策略使一次只能报告一个错误

## 例题

**例题1**：为文法 E → TE', E' → +TE' | ε, T → FT', T' → *FT' | ε, F → (E) | id 编写递归下降分析器伪代码。

**解答**：
```
parse_E():  parse_T(); parse_E'()

parse_E'():
    if lookahead == '+':  match('+'); parse_T(); parse_E'()
    // else: ε，什么都不做

parse_T():  parse_F(); parse_T'()

parse_T'():
    if lookahead == '*':  match('*'); parse_F(); parse_T'()
    // else: ε

parse_F():
    if lookahead == '(':  match('('); parse_E(); match(')')
    else if lookahead == 'id':  match('id')
    else:  error()
```

**例题2**：为什么 `if-else` 语句在递归下降中能正确处理悬空 else 问题？

**解答**：在递归下降分析器中，`if_stmt` 函数的实现通常为：`parse_if(); parse_stmt(); if (lookahead == ELSE) { match(ELSE); parse_stmt(); }`。这种实现天然地将 else 与最近的 if 匹配——因为第一次出现的 else 立即被最内层正在等待可选 else 的 if_stmt 函数消费。

## 代码示例

```python
class RecursiveDescentParser:
    def __init__(self, tokens):
        self.tokens = tokens
        self.pos = 0
    
    def peek(self):
        return self.tokens[self.pos] if self.pos < len(self.tokens) else '$'
    
    def match(self, expected):
        if self.peek() == expected:
            self.pos += 1
            return True
        raise SyntaxError(f"Expected '{expected}', got '{self.peek()}'")
    
    # E → T E'
    def parse_E(self):
        self.parse_T()
        self.parse_E_prime()
    
    # E' → + T E' | ε
    def parse_E_prime(self):
        if self.peek() == '+':
            self.match('+')
            self.parse_T()
            self.parse_E_prime()
    
    # T → F T'
    def parse_T(self):
        self.parse_F()
        self.parse_T_prime()
    
    # T' → * F T' | ε
    def parse_T_prime(self):
        if self.peek() == '*':
            self.match('*')
            self.parse_F()
            self.parse_T_prime()
    
    # F → ( E ) | id
    def parse_F(self):
        if self.peek() == '(':
            self.match('(')
            self.parse_E()
            self.match(')')
        elif self.peek() == 'id':
            self.match('id')
        else:
            raise SyntaxError(f"Unexpected token: {self.peek()}")

# 测试
parser = RecursiveDescentParser(['id', '+', 'id', '*', 'id'])
parser.parse_E()
print("Parsing successful!")
```

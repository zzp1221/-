## 核心定义

二分图最大匹配（Maximum Bipartite Matching）问题要求在二分图 G=(L,R,E) 中找到一组边，使得没有两条边共享同一个顶点，且边数尽可能多。匹配（Matching）是两两无公共顶点的边子集；最大匹配（Maximum Matching）即大小最大的匹配；完美匹配（Perfect Matching）覆盖了所有顶点。匈牙利算法（Hungarian Algorithm / Kuhn-Munkres）是求解该问题的经典方法，基于增广路径（Augmenting Path）的概念：交替通过非匹配边和匹配边的路径，若起点和终点都是未匹配的，则翻转该路径上所有边的匹配状态可使匹配数增加 1。匈牙利算法每次寻找一条增广路径并翻转，直到不存在增广路径，时间复杂度 O(VE)。Hopcroft-Karp 算法通过 BFS 寻找多条不相交的最短增广路径并一次性处理，复杂度优化至 O(√V * E)。

## 关键结论

- 增广路径定理：匹配 M 是最大匹配当且仅当不存在关于 M 的增广路径
- 匈牙利算法 O(VE)：每次增广路径查找 O(E)，最多增广 |V| 次
- Hopcroft-Karp 算法 O(√V * E)：使用分层图 + 多路增广，是二分图匹配的最优算法
- Konig 定理：二分图中最大匹配的大小等于最小顶点覆盖的大小
- 最大流模型：二分图匹配可以转化为网络流（源点->左部->右部->汇点，容量均 1），最大流 = 最大匹配

## 易错点

1. 增广路径的翻转后原匹配边变为非匹配，非匹配边变为匹配，总匹配数精确增加 1。
2. 匈牙利算法的 DFS 中 visited 数组每轮需重置：因为每轮寻找关于当前匹配的新增广路径。
3. 二分图最大匹配不等同于最大权匹配（后者需 Kuhn-Munkres 算法，O(n^3)），这也是作业分配问题的核心。

## 例题

**例题1：** 二分图 L={1,2,3,4}, R={5,6,7,8}，边 (1,5),(1,6),(2,5),(2,7),(3,6),(3,8),(4,7)。用匈牙利求最大匹配。

**解答：** 从 L 左侧遍历：1 匹配 5；2 尝试 5 冲突，5 尝试换给 1->6 可行，2-5 匹配；最终匹配：1-6,2-5,3-8,4-7，共 4 对。

**例题2：** 说明最大流求二分图匹配的构造方法。

**解答：** 添加超级源 s 连所有 L 顶点（容量 1），超级汇 t 连所有 R 顶点（容量 1），原二分图边方向 L->R（容量 1）。运行最大流算法（如 Dinic），流值为 1 的 L->R 边构成最大匹配。

## 代码示例

```python
def hungarian_dfs(n_left, n_right, edges):
    """
    n_left: 左部顶点数
    n_right: 右部顶点数
    edges: [(l, r), ...]
    """
    graph = [[] for _ in range(n_left)]
    for l, r in edges:
        graph[l].append(r)
    
    matchR = [-1] * n_right  # matchR[r] = 匹配到的左部顶点
    matchL = [-1] * n_left   # matchL[l] = 匹配到的右部顶点
    
    def dfs(u, visited):
        for v in graph[u]:
            if visited[v]:
                continue
            visited[v] = True
            # 若 v 未匹配或 v 当前的匹配者能换到别人
            if matchR[v] == -1 or dfs(matchR[v], visited):
                matchR[v] = u
                matchL[u] = v
                return True
        return False
    
    result = 0
    for u in range(n_left):
        visited = [False] * n_right  # 每轮重置
        if dfs(u, visited):
            result += 1
    
    # 返回匹配边列表
    matches = [(u, matchL[u]) for u in range(n_left) if matchL[u] != -1]
    return result, matches

# Hopcroft-Karp 算法 O(√V * E)
from collections import deque

def hopcroft_karp(n_left, n_right, edges):
    graph = [[] for _ in range(n_left)]
    for l, r in edges:
        graph[l].append(r)
    
    pairU = [-1] * n_left
    pairV = [-1] * n_right
    dist = [0] * n_left
    
    def bfs():
        q = deque()
        for u in range(n_left):
            if pairU[u] == -1:
                dist[u] = 0
                q.append(u)
            else:
                dist[u] = float('inf')
        INF = float('inf')
        d = INF
        while q:
            u = q.popleft()
            if dist[u] < d:
                for v in graph[u]:
                    if pairV[v] == -1:
                        d = dist[u] + 1
                    elif dist[pairV[v]] == INF:
                        dist[pairV[v]] = dist[u] + 1
                        q.append(pairV[v])
        return d != INF
    
    def dfs(u):
        for v in graph[u]:
            if pairV[v] == -1 or (dist[pairV[v]] == dist[u] + 1 and dfs(pairV[v])):
                pairU[u] = v
                pairV[v] = u
                return True
        dist[u] = float('inf')
        return False
    
    matching = 0
    while bfs():
        for u in range(n_left):
            if pairU[u] == -1 and dfs(u):
                matching += 1
    return matching

print(hungarian_dfs(4, 4, [(0,0),(0,1),(1,0),(1,2),(2,1),(2,3),(3,2)]))
```

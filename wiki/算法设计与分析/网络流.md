## 核心定义

网络流（Network Flow）理论研究在带容量限制的有向图中，从源点 s 到汇点 t 的最大流量传输问题。流网络 G=(V,E) 中每条边 (u,v) 有容量 c(u,v) >= 0，流 f(u,v) 需满足三个约束：容量约束（0 <= f(u,v) <= c(u,v)）、斜对称性（f(u,v) = -f(v,u)）、流量守恒（除 s,t 外每个顶点的净流量为 0）。最大流问题求最大可能的 |f|（从 s 流出的净流量）。

Ford-Fulkerson 方法是求解最大流的通用框架：在残余网络（Residual Network）中寻找增广路径（Augmenting Path）——从 s 到 t 且每条边上剩余容量 >0 的路径，沿该路径增加流量。Edmonds-Karp 算法指定使用 BFS 寻找最短增广路径，将时间复杂度限定为 O(VE^2)。网络流的核心定理是最大流最小割定理（Max-Flow Min-Cut Theorem）：最大流的值等于最小割的容量。

## 关键结论

- 最大流最小割定理是网络流理论的基石：|f_max| = min{c(S,T) | S为s-t割}，可通过增广路径终止时 s 在残余网络中能到达的顶点集构造最小割
- Ford-Fulkerson 在无理数容量下可能不终止；Edmonds-Karp 保证了多项式时间复杂度 O(VE^2)
- 残余网络：给定流 f，定义残余容量 c_f(u,v) = c(u,v) - f(u,v)，增广路径是残余网络中的 s-t 路径
- Dinic 算法更进一步优化到 O(V^2 E)，使用分层图 + 阻塞流，实际效率优于 Edmonds-Karp
- 网络流应用广泛：二分图匹配、最小路径覆盖、图像分割（Graph Cut）、航空调度、棒球淘汰问题等

## 易错点

1. 残余网络中反向边的容量随时间变化：增广后需更新正向边和反向边的残余容量。反向边的容量初始为 0，增广后增加对应的流量值。
2. 浮点容量可能引发 Ford-Fulkerson 不终止：在无理数容量下可能无限逼近但不收敛。使用 Edmonds-Karp 或 Dinic 可避免此问题。
3. s-t 割是有序的：割 (S,T) 中 S 包含 s，T 包含 t。割的容量只计从 S 到 T 的边，从 T 到 S 的边不计入容量。

## 例题

**例题1：** 网络含 4 个节点：s->1(容量 10), s->2(5), 1->t(5), 1->2(15), 2->t(10)。求最大流。

**解答：** 增广路径1: s->1->t, 流量 5；增广路径2: s->2->t, 流量 5；增广路径3: s->1->2->t, 流量 5（利用反向边 1->s）。总流量 = 15。最小割：S={s,1,2}, T={t}，c(S,T)=5+10=15。

**例题2：** 用最大流求二分图最大匹配。

**解答：** 构建流网络：源点 s 连接所有左部顶点（容量 1）；右部顶点连接汇点 t（容量 1）；图中原有边方向从左到右（容量 1）。最大流的值等于最大匹配的大小，流量为 1 的边即为匹配边。

## 代码示例

```python
from collections import deque

def edmonds_karp(n, edges, source, sink):
    """Edmonds-Karp O(VE^2)"""
    # 邻接矩阵存储容量
    cap = [[0] * n for _ in range(n)]
    adj = [[] for _ in range(n)]
    
    for u, v, c in edges:
        cap[u][v] += c  # 支持重边
        cap[v][u] += 0  # 反向边初始 0
        adj[u].append(v)
        adj[v].append(u)
    
    max_flow = 0
    parent = [-1] * n
    
    def bfs():
        """BFS 找增广路径，返回瓶颈容量"""
        parent[:] = [-1] * n
        parent[source] = -2
        q = deque([(source, float('inf'))])
        
        while q:
            u, flow = q.popleft()
            for v in adj[u]:
                if parent[v] == -1 and cap[u][v] > 0:
                    parent[v] = u
                    new_flow = min(flow, cap[u][v])
                    if v == sink:
                        return new_flow
                    q.append((v, new_flow))
        return 0
    
    while True:
        flow = bfs()
        if flow == 0:
            break
        max_flow += flow
        # 更新残余容量
        v = sink
        while v != source:
            u = parent[v]
            cap[u][v] -= flow
            cap[v][u] += flow
            v = u
    
    # 找最小割: 从 source 在残余网络中可达的顶点
    visited = [False] * n
    q = deque([source])
    visited[source] = True
    while q:
        u = q.popleft()
        for v in adj[u]:
            if not visited[v] and cap[u][v] > 0:
                visited[v] = True
                q.append(v)
    
    min_cut_S = [i for i in range(n) if visited[i]]
    return max_flow, min_cut_S
```

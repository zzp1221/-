## 核心定义

最优二叉搜索树（Optimal Binary Search Tree, OBST）问题：给定 n 个有序关键字 k1 < k2 < ... < kn 及其搜索概率 p_i，以及 n+1 个伪关键字 d0, d1, ..., dn（表示搜索值落在关键字区间之外的情况）及其概率 q_i。所有这些概率之和为 1。目标是构建一棵二叉搜索树，使得期望搜索代价 E = sum(p_i * (depth(k_i)+1)) + sum(q_i * depth(d_i)) 最小。该问题具有最优子结构：若一棵 BST 是最优的，则其任意子树 BST 也必是对应子区间的最优方案。动态规划解法的时间复杂度为 O(n^3)，Knuth 提出四边形不等式优化可降至 O(n^2)。OBST 是动态规划在树结构优化中的经典应用，也与哈夫曼树有着本质区别（哈夫曼树仅最小化加权外部路径长度，不要求保持搜索树性质）。

## 关键结论

- 状态定义：cost[i][j] 为包含关键字 k_i 到 k_j 的最优 BST 的期望搜索代价
- 递推公式：cost[i][j] = min_{r=i..j} { cost[i][r-1] + cost[r+1][j] + w(i,j) }，其中 w(i,j) 为区间内所有概率之和
- w(i,j) 可使用前缀和数组 O(1) 计算：w[i][j] = w[i][j-1] + p_j + q_j
- 状态转移中 root[i][j] 记录了区间的最优根 r，用于重构树结构
- Knuth 优化：利用 root[i][j-1] <= root[i][j] <= root[i+1][j] 单调性，将内层循环从 O(n^3) 降至 O(n^2)

## 易错点

1. OBST 与哈夫曼树的区别：哈夫曼树不要求保持 BST 性质（左小右大），仅最小化加权路径长度。OBST 必须保持 BST 性质。
2. 混淆 p 和 q 的索引：p_i 对应 k_i 的搜索概率，q_i 对应 d_i（伪关键字落在区间）的概率。d_i 的数量比 k_i 多一个。
3. 理解 w(i,j) 的物理意义：它表示在区间 [i,j] 内搜索的总概率。每次递归加深一层时，区间内所有节点的深度增加 1，代价增加 w(i,j)。

## 例题

**例题1：** 给定 n=3, p=[0.15, 0.10, 0.05], q=[0.05, 0.10, 0.20, 0.05]，求最优 BST 结构和期望代价。

**解答：** 概率总和检验 = 0.05+0.15+0.10+0.10+0.20+0.05+0.05 = 0.70（需归一化）。期望代价计算：选 k1, k2, k3 分别作为根，取最小。求得 cost[1][3] ≈ 2.00，最优根为 k2。

**例题2：** 说明 w(i,j) 的递推关系和 Knuth 优化原理。

**解答：** w[i][j] = w[i][j-1] + p[j] + q[j]，可用前缀和 O(1) 求得。Knuth 优化基于单调性定理 root[i][j-1] <= root[i][j] <= root[i+1][j]，使 r 的搜索范围大大缩小，均摊 O(n^2)。

## 代码示例

```python
def optimal_bst(p, q, n):
    """
    p[1..n]: 关键字的搜索概率
    q[0..n]: 伪关键字的搜索概率
    """
    # cost[i][j]: 包含 k_i..k_j 的最优BST期望代价
    cost = [[0.0] * (n + 2) for _ in range(n + 2)]
    # w[i][j]: 区间概率和(使用前缀和优化)
    w = [[0.0] * (n + 2) for _ in range(n + 2)]
    # root[i][j]: 记录最优根
    root = [[0] * (n + 2) for _ in range(n + 2)]
    
    # 初始化单节点
    for i in range(1, n + 2):
        cost[i][i - 1] = q[i - 1]
        w[i][i - 1] = q[i - 1]
    
    # 按子树大小 L 从 1 到 n
    for L in range(1, n + 1):
        for i in range(1, n - L + 2):
            j = i + L - 1
            cost[i][j] = float('inf')
            w[i][j] = w[i][j - 1] + p[j] + q[j]
            
            # Knuth优化：限制 r 的范围
            r_start = root[i][j - 1] if j > i else i
            r_end = root[i + 1][j] if i < j else j
            
            for r in range(r_start, r_end + 1):
                t = cost[i][r - 1] + cost[r + 1][j] + w[i][j]
                if t < cost[i][j]:
                    cost[i][j] = t
                    root[i][j] = r
    
    return cost, root

def build_tree(root, i, j, keys):
    """重构最优BST结构"""
    if i > j:
        return None
    r = root[i][j]
    return {
        'key': keys[r - 1],
        'left': build_tree(root, i, r - 1, keys),
        'right': build_tree(root, r + 1, j, keys)
    }
```

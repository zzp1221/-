## 核心定义

算法（Algorithm）是指求解特定问题的一系列有限、确定、可执行的步骤序列。一个合格的算法必须具备以下五个基本特性：有穷性（Finiteness），即算法必须在执行有限步后终止，不能陷入无限循环；确定性（Definiteness），每条指令必须有明确的含义，在相同输入下产生唯一确定的输出；可行性（Effectiveness），每条指令都必须是可执行的基本操作；输入（Input），算法可以有零个或多个输入；输出（Output），算法至少产生一个输出，反映问题的求解结果。算法是计算机程序的核心灵魂——程序 = 算法 + 数据结构，这一经典公式揭示了算法在计算机科学中的基础地位。算法的好坏直接影响程序的效率和可用性，因此算法设计与分析成为计算机科学最重要的核心课程之一。

## 关键结论

- 算法复杂度包括时间复杂度和空间复杂度两个维度，分别衡量算法运行所需的时间资源和空间资源
- 大O记号（Big-O Notation）描述的是增长率的渐近上界，关注输入规模趋近无穷大时算法性能的增长趋势
- 时间复杂度分析通常关注最坏情况（Worst-Case），因为它给出了算法性能的保证上界
- 常见的复杂度阶从低到高：O(1) < O(log n) < O(n) < O(n log n) < O(n^2) < O(2^n) < O(n!)
- 算法设计的基本策略包括分治法、动态规划、贪心法、回溯法、分支限界法等

## 易错点

1. 大O记号不等于最坏情况：大O是一个数学概念，描述函数的渐近上界。可以讨论最好情况的O记号，也可以讨论最坏情况的O记号。三者（大O、大Omega、大Theta）各司其职，不要混淆。
2. 常数因子在大O中会被忽略但不能在实际工程中忽略：O(1000000 * n)和O(n)在理论上是同一阶，但常数因子在工程实践中可能决定一个算法的可用性。
3. 时间复杂度不是衡量算法好坏的唯一标准：对小规模输入，运行常数大的O(n^2)算法可能比O(n log n)算法更快；硬件特性（如缓存友好度）也会影响实际性能。

## 例题

**例题1：** 分析以下代码的时间复杂度。
```cpp
int sum = 0;
for (int i = 0; i < n; i++)
    for (int j = i; j < n; j++)
        sum++;
```
**解答：** 内层循环执行 n - i 次。总执行次数 = n + (n-1) + (n-2) + ... + 1 = n(n+1)/2。因此时间复杂度为 O(n^2)。

**例题2：** 分析以下代码的时间复杂度。
```cpp
int i = 1, s = 0;
while (i <= n) {
    s += i;
    i *= 2;
}
```
**解答：** 每轮迭代 i 翻倍（1, 2, 4, ...），设迭代 t 次后 2^t <= n，即 t <= log_2 n。时间复杂度为 O(log n)。

## 代码示例

```python
# 常见的复杂度示例

# O(1) - 常数时间
def constant_time(arr):
    return arr[0] if arr else None

# O(log n) - 对数时间（二分查找）
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1

# O(n) - 线性时间
def linear_time(arr):
    total = 0
    for x in arr:
        total += x
    return total

# O(n log n) - 归并排序
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

## 核心定义

算法的复杂度分析（Complexity Analysis）是对算法所需资源进行量化研究的系统方法，主要包括时间复杂度和空间复杂度两个维度。时间复杂度（Time Complexity）衡量算法从开始执行到结束运行所需的时间量，通常以基本操作（如赋值、比较、算术运算）的执行次数来计量；空间复杂度（Space Complexity）衡量算法运行过程中占用的存储空间，包括算法代码自身占用的空间、输入数据占用的空间以及临时变量和递归调用栈等辅助空间。根据不同情况，复杂度分析分为最好情况（Best-Case）、最坏情况（Worst-Case）和平均情况（Average-Case）三种。最坏情况分析最为常用，因为它为算法性能提供了保证上界。复杂度分析的目的是预测算法在不同输入规模下的行为和性能趋势。

## 关键结论

- 时间复杂度的计算规则：顺序结构取加法（max规则）、循环结构取乘法、条件结构取最差分支
- 递归算法的复杂度分析需要建立递归方程（Recurrence），再使用代入法、递归树法或主定理求解
- 均摊分析（Amortized Analysis）适用于操作序列的复杂度分析，常用的方法有聚集法、记账法和势能法
- 空间复杂度中的辅助空间（Auxiliary Space）是除输入外额外使用的空间，是衡量算法"就地性"的关键指标
- 同一问题的不同算法可以有相同的时间复杂度但不同的空间复杂度（以空间换时间）、或者相反（以时间换空间）

## 易错点

1. 将程序运行时间（秒）等同于时间复杂度：时间复杂度是基本操作次数，与硬件速度无关。一个 O(n^2) 的C程序可能比 O(n log n) 的Python程序跑得快。
2. 递归空间复杂度估算遗漏调用栈深度：递归算法的空间复杂度必须计入递归调用栈的深度，如斐波那契递归的空间复杂度为 O(n)（栈深度），而非直觉中的 O(1)。
3. 更优的渐近复杂度不能保证对任何输入规模都更快：O(n^2) 和 O(n log n)算法的交叉点可能在 n 很大之后才出现，对小规模数据需要实际测试。

## 例题

**例题1：** 分析在n个元素的已排序数组中二分查找的时间复杂度（最好、最坏、平均）。

**解答：** 每轮比较将搜索范围减半，直到找到目标或区间为空。执行次数 = 二叉决策树的深度。最好情况（第一次命中）= O(1)。最坏情况 = ceil(log_2 n)，约 O(log n)。平均情况 = O(log n)，具体为 log_2 n + O(1)。

**例题2：** 分析原地反转数组 O(n) 时间和 O(1) 空间复杂度的算法。

**解答：** 使用双指针从两端向中间交换元素。时间复杂度 O(n)（n/2 次交换）。空间复杂度 O(1)（仅用两个指针变量）。

## 代码示例

```cpp
// 复杂度的系统分析示例
#include <iostream>
#include <vector>
using namespace std;

// 时间复杂度 O(n^2)，空间复杂度 O(1)
// 分析：外层循环 n 次，内层循环 (n - i - 1) 次 -> Σ ≈ n^2/2
void bubble_sort(vector<int>& arr) {
    int n = arr.size();
    for (int i = 0; i < n - 1; i++) {
        bool swapped = false;              // O(1) 空间
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr[j], arr[j + 1]);  // 基本操作: O(1)
                swapped = true;
            }
        }
        if (!swapped) break;               // 优化：最好情况 O(n)
    }
}

// 时间复杂度 O(n + k)，空间复杂度 O(n + k)
// k 为数据范围 (max - min + 1)
vector<int> counting_sort(const vector<int>& arr) {
    int n = arr.size();
    int lo = *min_element(arr.begin(), arr.end());  // O(n)
    int hi = *max_element(arr.begin(), arr.end());  // O(n)
    int k = hi - lo + 1;                            // O(1)
    
    vector<int> count(k, 0);                        // O(k) 空间
    vector<int> output(n);                          // O(n) 空间
    
    for (int x : arr) count[x - lo]++;              // O(n)
    for (int i = 1; i < k; i++)
        count[i] += count[i - 1];                   // O(k)
    for (int i = n - 1; i >= 0; i--)
        output[--count[arr[i] - lo]] = arr[i];      // O(n)
    
    return output;
}
// 总时间: O(n + k)，总空间: O(n + k)
```

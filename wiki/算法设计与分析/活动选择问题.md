## 核心定义

活动选择问题（Activity Selection Problem）是贪心算法的经典例题。给定 n 个活动，每个活动 i 有开始时间 s_i 和结束时间 f_i（s_i < f_i），要求选出最大数量的互不冲突的活动（即任何两个选中的活动不能在时间上重叠：一个活动结束之前另一个不能开始）。该问题具有贪心选择性质：将活动按结束时间升序排序后，每次选择结束时间最早且与已选活动不冲突的活动，即可得到最优解。最优子结构体现在：选择了结束时间最早的活动 a_1 后，原问题 S 归约为子问题 S' = {i in S: s_i >= f_1}（即 a_1 结束后才开始的活动），S' 的最优解与 {a_1} 的并集构成 S 的最优解。该算法时间复杂度为 O(n log n)（排序主导），排序后选择过程为 O(n)。

## 关键结论

- 贪心策略的多种等价形式：选结束时间最早、选开始时间最晚、选持续时间最短（注意：选开始时间最早是错误的策略）
- 活动选择问题的贪心选择性质可通过"剪切-粘贴"法严格证明
- 该问题是区间调度（Interval Scheduling）的最简单形式：资源（教室/会议室）只有一份，最大化使用次数
- 扩展：加权活动选择（每个活动有权重，目标是总权重最大）不满足贪心性质，需用 DP O(n log n) 求解
- 带权版本的状态转移：dp[i] = max(dp[i-1], dp[pred[i]] + w_i)，其中 pred[i] 是第 i 个活动之前最后一个不冲突的活动

## 易错点

1. 错误的贪心策略：选择持续时间最短的或开始时间最早的，这两个策略都不能保证最优。例如活动(1,5)(4,6)(5,10)，开始最早选(1,5)得1个，而最优选(4,6)+(5,10)不冲突因为(4,6)结束于6>5不能与(5,10)共存，最优选择(1,5)+(5,10)这2个。
2. 忽略活动结束时间的定义：假设结束时间和开始时间相等不算冲突（因为一个活动结束后另一个立刻开始）。
3. 排序时没有排除不可行活动：选择过程中只需检查 start >= last_finish，无需额外操作。如果所有活动都在已选活动之前结束，自然无法再选。

## 例题

**例题1：** 证明"选择结束时间最早的活动"的贪心策略正确。

**解答：** 设 S 为活动集合，a_1 为 S 中结束最早的活动。设 A 为 S 的任一最优解，设 a_k 为 A 中结束最早的活动。若 a_k = a_1，则 A 包含 a_1，证毕。若 a_k != a_1，则 f_1 <= f_k（a_1 结束最早）。构造 A' = A - {a_k} + {a_1}。因为 f_1 <= f_k，a_1 不与 A-{a_k} 中任何活动冲突。|A'| = |A|，故 A' 亦为最优解且包含 a_1。因此存在最优解包含 a_1。

**例题2：** 给定活动 (1,4), (3,5), (0,6), (5,7), (3,9), (5,9), (6,10), (8,11), (8,12), (2,14)，求最大活动子集。

**解答：** 排序结束时间：(1,4),(3,5),(0,6),(5,7),(3,9),(5,9),(6,10),(8,11),(8,12),(2,14)。选择 (1,4) -> last=4; (5,7) -> last=7; (8,11) -> last=11; (12,16) 如果给定的话会选。最终4个活动。

## 代码示例

```python
def activity_selection_greedy(activities):
    """activities: list of (start, finish)"""
    n = len(activities)
    if n == 0:
        return []
    # 按结束时间排序
    activities.sort(key=lambda x: x[1])
    
    selected = [activities[0]]
    last_finish = activities[0][1]
    
    for i in range(1, n):
        start, finish = activities[i]
        if start >= last_finish:
            selected.append((start, finish))
            last_finish = finish
    
    return selected

def weighted_activity_selection(activities):
    """带权版本 O(n log n) DP"""
    # activities: [(start, finish, weight), ...]
    n = len(activities)
    activities.sort(key=lambda x: x[1])  # 按结束时间排序
    
    # 预计算 pred[i]: 第i个活动前最后一个不冲突的活动
    pred = [-1] * n
    for i in range(n):
        lo, hi = 0, i - 1
        while lo <= hi:
            mid = (lo + hi) // 2
            if activities[mid][1] <= activities[i][0]:
                pred[i] = mid
                lo = mid + 1
            else:
                hi = mid - 1
    
    dp = [0] * n
    dp[0] = activities[0][2]
    for i in range(1, n):
        include = activities[i][2]
        if pred[i] != -1:
            include += dp[pred[i]]
        dp[i] = max(dp[i - 1], include)
    
    return dp[-1]

# 示例
acts = [(1,4), (3,5), (0,6), (5,7), (3,9), (5,9), (6,10), (8,11), (8,12), (2,14)]
print("贪心选择:", activity_selection_greedy(acts))
print("共", len(activity_selection_greedy(acts)), "个活动")
```

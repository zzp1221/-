## 核心定义

贪心算法（Greedy Algorithm）是一种在每一步选择中都采取在当前状态下最优（局部最优）的决策，期望最终达到全局最优解的问题求解策略。与动态规划"先求解子问题再做决策"不同，贪心算法"先做决策再求解子问题"。贪心算法能够获得全局最优解的前提是问题具有两个核心性质：贪心选择性质（Greedy-Choice Property）——全局最优解可以通过一系列局部最优选择来构造，即每一步的贪心选择都包含在某个全局最优解中；最优子结构（Optimal Substructure）——在做出贪心选择后，剩下的子问题也必然具有最优解。贪心算法的典型应用包括活动选择、哈夫曼编码、最小生成树、单源最短路径等。

## 关键结论

- 贪心算法的核心步骤：建立数学模型描述问题，将问题分解为子问题，确定贪心策略并证明其正确性
- 贪心选择性质的证明常用"剪切-粘贴"法（Cut-and-Paste）：将任意最优解中非贪心选择的部分替换为贪心选择，证明不会变差
- 贪心算法通常比动态规划更高效（时间复杂度和空间复杂度都更低），但适用范围更窄
- 不能用贪心求解的典型问题：0-1背包（需DP）、旅行商问题（TSP）
- 贪心算法的正确性验证是关键步骤：错误的贪心策略可能导致非最优甚至错误的结果

## 易错点

1. 将贪心策略的存在性作为其正确性证明：仅仅因为某个贪心策略"看起来合理"不能保证其正确性，必须严格证明贪心选择性质。
2. 对不满足最优子结构的问题使用贪心：在某些问题中，局部最优的选择可能导致子问题无解或劣解。
3. 贪心和DP的边界混淆：分数背包是贪心（可分割），0-1背包是DP（不可分割）。关键在于物品是否可分割——可分割则贪心成立，不可分割则需要DP。

## 例题

**例题1：** 证明活动选择问题的贪心选择性质。

**解答：** 策略为每次选择结束时间最早的活动。证明（反证法）：设 A 是活动集 S 的最优解，将 A 中第一个活动（按结束时间排序）替换为 S 中结束时间最早的活动 a_1，得到 A'。因为 a_1 的结束时间最早，它不会与 A 中其他活动冲突，A' 仍是合法解且规模相同，故 a_1 属于某个最优解。

**例题2：** 判断：对币值 {1, 5, 10, 25} 的硬币找零问题（求最少硬币数），贪心是否正确？币值 {1, 3, 4} 呢？

**解答：** 对前者正确：因为每个大面值都是小面值的倍数，贪心取最大面值不会阻塞后续最优。对后者：找零 6，贪心选 4+1+1=3 枚，但最优为 3+3=2 枚。不满足贪心选择性质，需用 DP。

# 活动选择问题
def activity_selection(activities):
    """
    activities: [(start, finish), ...]
    贪心策略：优先选结束时间最早的活动
    """
    activities.sort(key=lambda x: x[1])  # 按结束时间排序
    selected = []
    last_finish = 0
    
    for start, finish in activities:
        if start >= last_finish:  # 贪心选择：不冲突即可选
            selected.append((start, finish))
            last_finish = finish
    
    return selected

# 分数背包
def fractional_knapsack(W, items):
    """
    items: [(weight, value), ...]
    贪心策略：按价值密度 v/w 降序选择
    """
    items.sort(key=lambda x: x[1] / x[0], reverse=True)
    total_value = 0.0
    for w, v in items:
        if W >= w:
            W -= w
            total_value += v
        else:
            total_value += v * (W / w)
            break
    return total_value

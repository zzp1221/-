## 核心定义

最小生成树（Minimum Spanning Tree, MST）问题：给定一个带权连通无向图 G=(V,E)，找到一棵包含所有顶点且边权总和最小的树（无环连通子图）。MST 具有贪心选择性质：每次选择当前权重最小且不构成环的边，最终构成 MST。两种经典算法体现了这一性质：Prim 算法（1957年，Jarnik 1930年）从任意起点出发，每次选取与当前已选顶点集距离最近的外部顶点加入，类似于 Dijkstra，使用优先队列实现 O(E log V)；Kruskal 算法（1956年）从小到大遍历所有边，用并查集（Union-Find）判断加入该边是否形成环，实现 O(E log E)。两种算法都基于 MST 的一个关键性质——割性质（Cut Property）：对图的任意一个割，跨越该割的最小权边必属于某棵 MST。

## 关键结论

- 最小生成树不唯一（当存在等权边时），但 MST 的权重之和是唯一的
- Prim 使用二叉堆 O(E log V)，使用斐波那契堆可优化至 O(E + V log V)
- Kruskal 使用并查集 + 路径压缩，时间复杂度 O(E log E) 近似 O(E log V)
- 割性质（Cut Property）和回路性质（Cycle Property）是 MST 的两个基础定理
- 在稠密图（E ~ V^2）中 Prim 更优（O(V^2)），稀疏图（E ~ V）中 Kruskal 更优
- MST 应用：网络布线、电路设计、图像分割、聚类分析中的单连接聚类

## 易错点

1. MST 要求无向图：有向图中的最小树形图（Minimum Arborescence）使用 Chu-Liu/Edmonds 算法，与 MST 不同。
2. 图可以不连通：若图不连通则不存在生成树（MST 无解），算法需要检测连通性。
3. 负权边在 MST 中合法且不影响 Prim/Kruskal：而 Dijkstra 不能处理负权边。

## 例题

**例题1：** 用 Prim 和 Kruskal 分别求下图的 MST（边: AB:4, AC:2, BC:1, BD:5, CD:8）。

**解答：** Kruskal 排序边：BC(1), AC(2), AB(4), BD(5), CD(8)。依次加入 BC, AC（不形成环）, BD(5) 加入后 V=4 条边=3，结束。MST 总权重=1+2+5=8。Prim 从 A 出发：A 加入{C(2), B(4)}，选 C；A-C 集邻接 {B(1), D(8)}，选 B(1)；A-C-B 集邻接 {D(5)}，选 D(5)。结果同。

**例题2：** 证明割性质：对任意割 (S, V-S)，设 e 是跨越该割的最小权边，则存在一棵 MST 包含 e。

**解答：** 反证法。设 T 是一棵 MST 不含 e。将 e 加入 T 必形成环，该环至少包含一条跨越割的边 e'（e' 不在 T 中）。因为 e 是最小跨割边，w(e) <= w(e')。移除 e' 得到 T' = T - {e'} + {e}，w(T') <= w(T)，T' 仍为生成树。故存在包含 e 的 MST。

## 代码示例

```python
import heapq

# Prim 算法 O(E log V)
def prim(graph, n):
    """graph: 邻接表 [(neighbor, weight), ...]"""
    visited = [False] * n
    min_heap = [(0, 0, -1)]  # (weight, vertex, parent)
    mst_weight = 0
    mst_edges = []
    
    while min_heap:
        w, u, p = heapq.heappop(min_heap)
        if visited[u]:
            continue
        visited[u] = True
        mst_weight += w
        if p != -1:
            mst_edges.append((p, u, w))
        
        for v, weight in graph[u]:
            if not visited[v]:
                heapq.heappush(min_heap, (weight, v, u))
    
    return mst_weight, mst_edges

# Kruskal 算法 O(E log E)
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
    
    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # 路径压缩
        return self.parent[x]
    
    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        if self.rank[rx] < self.rank[ry]:
            self.parent[rx] = ry
        elif self.rank[rx] > self.rank[ry]:
            self.parent[ry] = rx
        else:
            self.parent[ry] = rx
            self.rank[rx] += 1
        return True

def kruskal(edges, n):
    """edges: [(u, v, weight), ...]"""
    edges.sort(key=lambda x: x[2])
    uf = UnionFind(n)
    mst_weight = 0
    mst_edges = []
    
    for u, v, w in edges:
        if uf.union(u, v):
            mst_weight += w
            mst_edges.append((u, v, w))
            if len(mst_edges) == n - 1:
                break
    
    return mst_weight, mst_edges
```

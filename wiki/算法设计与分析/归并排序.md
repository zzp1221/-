## 核心定义

归并排序（Merge Sort）是基于分治法的经典排序算法，由约翰·冯·诺依曼于1945年提出。其核心思想是将待排序序列递归地一分为二，直到每个子序列仅包含单个元素（自然有序），然后将有序子序列两两合并，最终得到一个完整的有序序列。归并排序包含两个核心操作：递归分解（Divide），将规模为n的数组不断对半分割，递归深度为 O(log n)；有序合并（Merge），将两个有序子序列合并为一个有序序列，使用双指针技术，时间复杂度为 O(n)。归并排序是稳定排序算法，最坏、最好和平均时间复杂度均为 O(n log n)，空间复杂度为 O(n)（非原地排序）。

## 关键结论

- 归并排序的时间复杂度在所有情况下均为 O(n log n)，这是基于比较的排序算法的最优渐近下界
- 归并排序的空间复杂度为 O(n)，需要辅助数组存储合并结果。优化的原地归并可将空间复杂度降至 O(1) 但会增加时间复杂度
- 归并排序是稳定排序：当两个元素相等时，它们在合并时保持相对顺序，这一特性在数据库多关键字排序中很有价值
- 归并排序特别适合外部排序：当数据无法全部放入内存时，可以采用多路归并将已排序的外部块逐步合并
- 归并排序在小数组上不如插入排序，因此实际实现中通常在小规模子数组（如 n < 10）时切换到插入排序

## 易错点

1. 合并后未将剩余元素复制到原数组：两个指针到达末尾后，需要将未遍历完的剩余元素全部追加到结果中。
2. 辅助数组大小不足：合并时需要大小为 n 的辅助数组。如果不复用同一个辅助数组而在每层递归都分配新数组，空间复杂度会升至 O(n log n)。
3. 混淆自顶向下和自底向上的归并排序：前者是递归实现（分治），后者是迭代实现（非递归）。两者复杂度一致，但迭代版本避免了递归栈开销，更适合嵌入式系统。

## 例题

**例题1：** 分析归并排序对逆序对数量的影响。给定数组 [2, 4, 1, 3, 5]，在归并排序的合并过程中如何统计逆序对？

**解答：** 在合并过程中，当左子数组的当前元素 > 右子数组的当前元素时，说明左子数组中该元素及其后面的所有元素都与右子数组当前元素构成逆序对。count += len(left) - i。该算法的时间复杂度仍为 O(n log n)，可以在排序的同时计算逆序对。

**例题2：** 归并排序递归式 T(n) = 2T(n/2) + n 如何求解？

**解答：** 使用主定理或递归树：log_2 2 = 1，f(n)=n=Θ(n^1 log^0 n)，属于 Case 2 (k=0)。T(n) = Θ(n log n)。递归树：每层工作量 n，共 log n 层，总计 O(n log n)。

## 代码示例

```python
def merge_sort(arr):
    """自顶向下递归归并排序 O(n log n)"""
    n = len(arr)
    if n <= 1:
        return arr
    
    mid = n // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    """合并两个有序数组 O(n)"""
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    # 追加剩余元素
    result.extend(left[i:])
    result.extend(right[j:])
    return result

def merge_sort_iterative(arr):
    """自底向上迭代归并排序, 无需递归 O(n log n)"""
    n = len(arr)
    width = 1
    while width < n:
        for i in range(0, n, 2 * width):
            lo, mid, hi = i, i + width, min(i + 2 * width, n)
            if mid < hi:
                left = arr[lo:mid]
                right = arr[mid:hi]
                merged = merge(left, right)
                arr[lo:hi] = merged
        width *= 2
    return arr

# 归并排序统计逆序对
def count_inversions(arr):
    """在归并排序过程中统计逆序对个数 O(n log n)"""
    def merge_count(left, right):
        result = []
        i = j = 0
        inv_count = 0
        while i < len(left) and j < len(right):
            if left[i] <= right[j]:
                result.append(left[i]); i += 1
            else:
                result.append(right[j]); j += 1
                inv_count += len(left) - i  # 关键
        result.extend(left[i:])
        result.extend(right[j:])
        return result, inv_count
    
    def sort_count(arr):
        if len(arr) <= 1:
            return arr, 0
        mid = len(arr) // 2
        left, left_inv = sort_count(arr[:mid])
        right, right_inv = sort_count(arr[mid:])
        merged, cross_inv = merge_count(left, right)
        return merged, left_inv + right_inv + cross_inv
    
    _, total = sort_count(arr)
    return total
```

## 核心定义

编辑距离（Edit Distance），又称 Levenshtein 距离，是衡量两个字符串相似度的经典度量。它定义为通过最少的编辑操作次数将一个字符串转换成另一个字符串。编辑操作通常包括三种：插入（Insertion）一个字符、删除（Deletion）一个字符、替换（Substitution）一个字符。例如将 "kitten" 转为 "sitting" 需要 3 步：k->s（替换）、e->i（替换）、插入 g。编辑距离具有典型的最优子结构，使用动态规划在 O(mn) 时间内求解。状态定义 dp[i][j] 为 word1[0..i-1] 转为 word2[0..j-1] 的最小编辑次数，转移方程：若 word1[i-1]==word2[j-1]，dp[i][j]=dp[i-1][j-1]；否则 dp[i][j]=1+min(dp[i-1][j]删除, dp[i][j-1]插入, dp[i-1][j-1]替换)。编辑距离在拼写检查、DNA序列比对、模糊搜索、机器翻译评估等领域被广泛使用。

## 关键结论

- 编辑距离是度量字符串相似性的最直观方法，距离越小越相似
- 三种编辑操作在 DP 转移中各有映射：dp[i-1][j] 对应删除 word1 当前字符，dp[i][j-1] 对应插入 word2 当前字符到 word1，dp[i-1][j-1] 对应替换（或匹配跳过）
- Damerau-Levenshtein 距离是扩展版本，额外允许相邻字符交换（Transposition），更贴合人工拼写错误场景
- 最长公共子序列 LCS 与编辑距离的关系：edit_dist = |X| + |Y| - 2*LCS（仅允许插入删除时）
- DNA比对使用的 Needleman-Wunsch 和 Smith-Waterman 算法与编辑距离同根同源

## 易错点

1. 初始化的边界条件：dp[i][0]=i（删除 i 个字符），dp[0][j]=j（插入 j 个字符）。
2. 混淆 dp[i][j] 中 i 和 j 与字符串索引的对应关系：dp[i][j] 对应 word1[:i] 和 word2[:j]。
3. 编辑操作的类型数量可变：经典版本 3 种操作（插入/删除/替换），但有些场景替换算 2 步（先删后插），需要根据具体定义调整转移方程。

## 例题

**例题1：** 计算 "horse" 到 "ros" 的编辑距离。

**解答：** dp[5][3] 计算过程：
 - dp[1][0]=1（h删除），dp[0][1]=1（r插入）
 - ...经过DP递推，最终 dp[5][3]=3
 - 操作序列：horse -> rorse (h->r) -> rose (删第二个r) -> ros (删e)

**例题2：** 证明 dp[i][j] 的转移方程穷尽了所有可能。

**解答：** 考虑 word1[:i] 转为 word2[:j] 的最后一步仅有三种可能：1) word1 末尾字符被删除，则之前需将 word1[:i-1] 转为 word2[:j]；2) word2 的末字符被新插入，则之前需将 word1[:i] 转为 word2[:j-1]；3) 替换或匹配 word1 末字符为 word2 末字符，之前需将 word1[:i-1] 转为 word2[:j-1]。三种情况覆盖所有可能，取 min 即最优。

## 代码示例

```python
def edit_distance(word1, word2):
    """O(mn) 时间, O(min(m,n)) 空间"""
    m, n = len(word1), len(word2)
    # 确保 word1 是较短的字符串以节省空间
    if m > n:
        word1, word2 = word2, word1
        m, n = n, m
    
    prev = list(range(n + 1))  # dp[0][j] = j
    
    for i in range(1, m + 1):
        curr = [i] + [0] * n   # curr[0] = dp[i][0] = i
        for j in range(1, n + 1):
            if word1[i - 1] == word2[j - 1]:
                curr[j] = prev[j - 1]
            else:
                curr[j] = 1 + min(
                    prev[j],       # 删除
                    curr[j - 1],   # 插入
                    prev[j - 1]    # 替换
                )
        prev = curr
    
    return prev[n]

def edit_distance_with_ops(word1, word2):
    """返回编辑距离和操作序列"""
    m, n = len(word1), len(word2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i - 1] == word2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = 1 + min(dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1])
    
    # 回溯操作序列
    ops = []
    i, j = m, n
    while i > 0 or j > 0:
        if i > 0 and dp[i][j] == dp[i - 1][j] + 1:
            ops.append(f"删除 '{word1[i-1]}' at pos {i-1}")
            i -= 1
        elif j > 0 and dp[i][j] == dp[i][j - 1] + 1:
            ops.append(f"插入 '{word2[j-1]}' at pos {i}")
            j -= 1
        elif i > 0 and j > 0:
            if word1[i - 1] != word2[j - 1]:
                ops.append(f"替换 '{word1[i-1]}' -> '{word2[j-1]}'")
            i -= 1; j -= 1
    
    return dp[m][n], list(reversed(ops))

print(edit_distance("horse", "ros"))  # 3
print(edit_distance("intention", "execution"))  # 5
```
